⚡ Métodos de Arreglos Imprescindibles

Para trabajar con datos reales que consumimos de servidores o APIs REST, necesitamos manipular arreglos de objetos. Imagina que recibimos este conjunto de datos:

// Datos simulados de una API (Base de datos de productos)
const productos = [
  { id: 1, nombre: 'Laptop Gamer', categoria: 'Tecnología', precio: 1200, stock: true },
  { id: 2, nombre: 'Teclado Mecánico', categoria: 'Tecnología', precio: 80, stock: false },
  { id: 3, nombre: 'Silla Ergonómica', categoria: 'Hogar', precio: 250, stock: true },
  { id: 4, nombre: 'Monitor 4K', categoria: 'Tecnología', precio: 400, stock: true }
];

Para manipular estos datos rápidamente sin usar bucles for complejos, usamos los tres métodos fundamentales:

.find()

Buscador exacto. Devuelve solo el primer elemento que cumpla con la condición.

Retorna: Un objeto o undefined

.filter()

El colador. Filtra el arreglo y devuelve todos los elementos que cumplan la condición.

Retorna: Un nuevo arreglo

.map()

El transformador. Toma los elementos y crea un nuevo arreglo modificando o extrayendo datos.

Retorna: Un nuevo arreglo (mismo tamaño)

Ejemplos de Código Práctico:

// 1. Encontrar el producto con id igual a 3 (.find)
const silla = productos.find(p => p.id === 3);
console.log(silla); // Resultado: { id: 3, nombre: 'Silla Ergonómica', ... }

// 2. Filtrar solo los productos que están disponibles en stock (.filter)
const disponibles = productos.filter(p => p.stock === true);
console.log(disponibles); // Devuelve un arreglo con 3 productos (excluye teclado)

// 3. Extraer solo los nombres de los productos (.map)
const nombres = productos.map(p => p.nombre);
console.log(nombres); // ['Laptop Gamer', 'Teclado Mecánico', 'Silla Ergonómica', 'Monitor 4K']

⏳ Próxima Clase: Entendiendo la Asincronía

En la programación tradicional, el código se ejecuta línea por línea (síncrono). Si una línea tarda mucho, bloquea todo. Pero al conectarnos a servidores, la respuesta tarda milisegundos que no podemos predecir. Aquí es donde entra la asincronía.

☕
La analogía de la Cafetería:

Si pides un café de forma síncrona, toda la fila se detiene hasta que te preparen el café. En cambio, de forma asíncrona (Promesa), pagas, te dan un "beeper" o recibo, la fila avanza, y cuando el café está listo, te avisan para que vayas a buscarlo.

El estándar moderno: Async / Await

Para no complicarnos con Promesas anidadas, JavaScript usa async/await para escribir código asíncrono que se lee casi como síncrono:

// Simulamos una llamada a una API real
async function obtenerProductos() {
  try {
    // Esperamos (await) a que el servidor responda
    const respuesta = await fetch('https://api.ejemplo.com/productos');
    const datos = await respuesta.json(); // Convertimos la respuesta a JSON
    
    console.log(datos);
  } catch (error) {
    console.error('¡Algo falló al cargar los datos!', error);
  }
}

💡 Tips de Código Limpio

  • Usa siempre const a menos que estés seguro de que reasignarás el valor de la variable (en ese caso usa let).
  • Usa las funciones flecha () => {} para tener un código de iteradores más compacto y legible.
  • Al usar fetch o async/await, nunca olvides envolver el código en bloques try / catch para capturar errores del servidor.
