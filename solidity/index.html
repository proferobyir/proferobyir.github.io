Nivel 2 · El poder de require()

⚖️ Guardián de la puerta

require(condición, "mensaje") valida una condición. Si es falsa, todo se detiene y revierte.

require(pago >= precioCafe, "Saldo insuficiente");
// Si falla: reversión total del estado + devolución del gas no consumido.

🔁 REVERT (Reversión total)

✅ Si la condición es FALSA: el segundo parámetro muestra el mensaje de error, pero NO permite que nada más continúe. Provoca un mecanismo llamado Revert:

➜ El programa se detiene en seco.
➜ La transacción se cancela por completo.
➜ Cualquier cambio que se haya intentado hacer en la memoria de la Blockchain se borra, devolviendo el estado exactamente a como estaba antes de pulsar el botón.
➜ Además, se le regresa al usuario el gas (dinero de la red) que no se gastó.

🎯 En conclusión: el segundo parámetro es el "mensaje de expulsión" cuando el guardián require te rebota.

💨 Gas no consumido → reembolsado. Solo pagas por la verificación fallida.

Un contrato es una clase con atributos (stock) y métodos (surtir). Encapsulamos lógica.

contract Maquina {

Nivel 4 · Smart Contract

⛓️ Despliegue en Blockchain

El contrato se despliega y ejecuta reglas inmutables. require(stock>0) protege la compra.

contract Tienda {
    uint256 public stock = 3;
    function comprar() public {
        require(stock > 0, "Sin existencias");
        stock--;
    }
}

🧠 Recuerda: Aquí el require también aplica la misma regla. Si stock=0, el mensaje "Sin existencias" aparece y la transacción revierte, devolviendo el estado anterior y reembolsando el gas no usado.
