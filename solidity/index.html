De Cero a Programador de Blockchain

Una guía interactiva secuencial para aprender a pensar en código y dominar Solidity paso a paso.

Nivel 1: Los Datos Básicos

¿Dónde se guarda la información?

Para programar, primero necesitas "cajas" donde guardar datos. En programación las llamamos Variables. En Solidity debes decirle a la caja qué tipo de dato va a guardar estrictamente. 

// Creamos una caja para números enteros positivos
uint256 public precioCafe = 2;
// Creamos una caja para texto (Strings)
string public mensaje = "¡Hola!";

Nivel 2: Tomar Decisiones

El control de flujo (If / Require)

Los programas no son lineales; toman decisiones. Si pasa "X", hacemos "Y". En Solidity usamos la palabra reservada require(), que funciona como un portero de discoteca: si no cumples la regla, traba la puerta y cancela todo. 

// Si el dinero enviado es menor al precio, el programa explota
require(dineroEnviado >= precioCafe, "Dinero insuficiente");

Nivel 3: El Molde (POO)

Estructurar un Objeto completo

En lugar de tener variables sueltas, en la Programación Orientada a Objetos juntamos los atributos (variables) y las acciones (funciones) dentro de un solo molde llamado Clase (o Contrato en Solidity). 

contract MaquinaExpendedora {
    uint256 public stock; // Atributo

    function surtir(uint256 _cantidad) public {
        stock += _cantidad; // Acción
    }
}

Atributo actual -> Stock: 0 unidades

Nivel 4: El Contrato Inteligente

Instanciación e Inmutabilidad en Blockchain

¡Felicidades, unimos todo! En Solidity, la clase se sube a la Blockchain dándole a Desplegar (Deploy). El contrato corre de forma autónoma, nadie lo puede modificar, hackear ni apagar. 

// Todo junto funcionando de forma autónoma
contract Final {
    // Nivel 1 (Variables) + Nivel 3 (Estructura de clase)
    uint256 public stock = 3;
    
    function comprar() public payable {
        // Nivel 2 (Condicionales / Reglas)
        require(stock > 0, "Agotado");
        stock -= 1;
    }
}
