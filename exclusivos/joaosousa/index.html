2. Conectar a MySQL · Explicación profunda

🔌 Archivo db_config.py con comentarios

# =======================================================
# 1. IMPORTAMOS EL CONECTOR DE MYSQL
#    Este módulo lo instalamos con: pip install mysql-connector-python
#    Contiene todas las clases y funciones para comunicarnos con MySQL.
# =======================================================
import mysql.connector

# =======================================================
# 2. FUNCIÓN QUE CREA Y DEVUELVE UNA CONEXIÓN
#    Cada vez que la llamamos, obtenemos un objeto conexión activo.
# =======================================================
def get_connection():
    # ---------------------------------------------------
    # 3. mysql.connector.connect() es una función del módulo importado.
    #    Crea un objeto de tipo 'MySQLConnection' que representa
    #    la sesión con el servidor MySQL.
    #
    #    PARÁMETROS (de dónde salen):
    #    - host: "localhost" → significa que MySQL está en la misma máquina.
    #    - user: "root" → usuario de MySQL (el que creaste al instalar).
    #    - password: "tu_contraseña" → la contraseña de ese usuario.
    #    - database: "crud_tkinter" → nombre de la base de datos que usaremos.
    #      Debes haberla creado antes con CREATE DATABASE crud_tkinter;
    # ---------------------------------------------------
    return mysql.connector.connect(
        host="localhost",          # ¿de dónde? Es el servidor local.
        user="root",               # usuario con permisos para la BD
        password="tu_contraseña",    # la clave que configuraste en MySQL
        database="crud_tkinter"      # la BD que contiene nuestra tabla
    )
    # ---------------------------------------------------
    # 4. LO QUE DEVUELVE: un objeto 'conn' (MySQLConnection) que tiene:
    #    - método .cursor() para crear cursores
    #    - método .commit() para guardar cambios
    #    - método .close() para cerrar la conexión
    #    - y muchos más.
    # ---------------------------------------------------
🧠 Entiende el objeto conexión: conn = mysql.connector.connect(...) → conn es un objeto que "habla" con MySQL. Sin él, no podemos ejecutar ni una sola consulta.

3. CRUD · Explicación de objetos y flujo

📝 Crear (INSERT)
def crear_usuario(nombre, email):
    # 1. Llamamos a get_connection() que nos devuelve un objeto conexión
    conn = get_connection()  
    #    ↑ conn es un objeto MySQLConnection

    # 2. Sobre conn, llamamos a .cursor() que crea un objeto cursor
    cursor = conn.cursor()   
    #    ↑ cursor es un objeto que ejecuta SQL y trae resultados

    # 3. Definimos la consulta SQL con marcadores %s (parametrizada)
    sql = "INSERT INTO usuarios (nombre, email) VALUES (%s, %s)"

    # 4. Ejecutamos la consulta en el cursor, pasando los datos
    #    cursor.execute() envía la instrucción al servidor MySQL
    cursor.execute(sql, (nombre, email))

    # 5. Confirmamos la transacción con .commit() sobre la conexión
    conn.commit()            
    #    ↑ sin commit(), los cambios no se guardan en la BD

    # 6. Cerramos la conexión (liberamos recursos)
    conn.close()             
    #    ↑ siempre cerrar para evitar fugas de conexiones

📤 viaje nombre/email → cursor.execute → commit → cierra

📖 Leer (SELECT)
def obtener_usuarios():
    conn = get_connection()       # objeto conexión
    # cursor con dictionary=True → devuelve filas como diccionarios
    cursor = conn.cursor(dictionary=True)  
    # ↑ así podemos acceder con u['nombre'] en lugar de u[0]

    cursor.execute("SELECT * FROM usuarios")  
    # ↑ ejecuta la consulta

    resultados = cursor.fetchall()  
    # ↑ .fetchall() obtiene TODAS las filas como lista de diccionarios

    conn.close()                 # cerramos conexión
    return resultados
✏️ Actualizar (UPDATE)
def actualizar_usuario(id, nombre, email):
    conn = get_connection()
    cursor = conn.cursor()
    sql = "UPDATE usuarios SET nombre=%s, email=%s WHERE id=%s"
    # %s protege contra inyección SQL
    cursor.execute(sql, (nombre, email, id))  # el orden importa
    conn.commit()   # confirmamos cambios
    conn.close()
🗑️ Eliminar (DELETE)
def eliminar_usuario(id):
    conn = get_connection()
    cursor = conn.cursor()
    cursor.execute("DELETE FROM usuarios WHERE id=%s", (id,))
    conn.commit()   # sin esto, no se elimina
    conn.close()

4. Interfaz Tkinter (main.py) con comentarios

🧪 Ventana con campos y botones

import tkinter as tk
from tkinter import messagebox
from crud_ops import crear_usuario, obtener_usuarios, actualizar_usuario, eliminar_usuario

# ============================================================
# Función que se ejecuta al hacer click en "Guardar"
# ============================================================
def guardar_datos():
    # 1. Obtenemos el texto de los campos Entry
    nombre = entry_nombre.get()   # .get() devuelve el string escrito
    email = entry_email.get()

    # 2. Validamos que no estén vacíos
    if nombre and email:
        # 3. Llamamos a la función del CRUD que hace INSERT
        #    AQUÍ OCURRE EL VIAJE: los datos pasan a crud_ops → db_config → MySQL
        crear_usuario(nombre, email)   
        messagebox.showinfo("Éxito", "Usuario guardado")
        limpiar_campos()   # borramos los Entry
    else:
        messagebox.showerror("Error", "Completa todos los campos")

# ... creación de widgets (labels, entries, botones) ...
# Los Entry se asocian a variables o se leen con .get()

🎯 Flujo completo: El usuario escribe → click en botón → guardar_datos() → crear_usuario() → conexión MySQL → INSERT → feedback.

🧭 El viaje de los datos (explicado)

1. Usuario escribe en Entry → entry_nombre.get() captura el texto.
2. Botón "Guardar" → ejecuta guardar_datos().
3. guardar_datos() → llama a crear_usuario(nombre, email) (de crud_ops).
4. crud_ops.crear_usuario() → llama a get_connection() (db_config).
5. get_connection() → retorna un objeto conn (MySQLConnection).
6. Sobre conn, creamos cursor → cursor = conn.cursor().
7. Ejecutamos SQL → cursor.execute("INSERT ...", (nombre, email)).
8. conn.commit() → confirma la transacción en la BD.
9. conn.close() → liberamos la conexión.
10. Mensaje de éxito → el usuario ve confirmación.

📋 Mostrar datos en Tkinter

def listar_usuarios():
    # Limpiamos el frame que contiene la lista
    for widget in frame_lista.winfo_children():
        widget.destroy()

    # Obtenemos los datos desde la BD
    usuarios = obtener_usuarios()   # SELECT *

    # Creamos una etiqueta por cada usuario
    for u in usuarios:
        lbl = tk.Label(frame_lista, text=f"{u['id']} - {u['nombre']} ({u['email']})")
        lbl.pack()

✅ Buenas prácticas en Windows

  • Usa entorno virtual → evita conflictos de paquetes.
  • Separa responsabilidades → interfaz (Tkinter) / lógica (CRUD) / conexión (db_config).
  • Maneja excepciones → try/except en operaciones de base de datos.
  • Cierra siempre la conexión → conn.close() para evitar fugas.
  • Usa parametrización → (%s) para evitar inyección SQL.
  • En Windows: si usas python y no funciona, prueba con py.

🧠 Recuerda: el viaje de datos es circular: formulario → procesamiento → BD → y de vuelta para mostrar.
