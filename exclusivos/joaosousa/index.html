🪟 TKINTER + MySQL en WINDOWS

✅ Preparación · Entorno virtual · CRUD · Viaje de datos Windows 10/11

🧩 Paso a paso, sin rodeos. Aprende a crear un CRUD con Tkinter y MySQL en Windows. Sigue el flujo de los datos: desde el formulario hasta la base de datos.

🎯 Formato claro — diseñado para mantener la atención.

1. Preparación en Windows

🖥️ Entorno virtual (cmd)
  • python -m venv entorno_tk → crea carpeta
  • entorno_tk\Scripts\activate → activar
  • Instala: pip install mysql-connector-python
  • Verifica: pip list → debe aparecer mysql-connector
  • 💡 Si no funciona, prueba con py -m venv entorno_tk
⚡ Recuerda: siempre activa el entorno antes de ejecutar.
📁 Estructura de archivos
📁 mi_crud_win/
 ├── 📄 main.py          # ventana Tkinter
 ├── 📄 db_config.py     # conexión MySQL
 ├── 📄 crud_ops.py      # funciones CRUD
 └── 📄 utils.py         # (opcional)

✅ Mantén separada la lógica de la interfaz.

2. Conectar a MySQL

🔌 Archivo db_config.py

# db_config.py
import mysql.connector

def get_connection():
    return mysql.connector.connect(
        host="localhost",
        user="root",           # tu usuario de MySQL
        password="tu_contraseña",  # tu password
        database="crud_tkinter"
    )

🔹 Viaje de datos: El formulario envía datos → se llama a get_connection() → se ejecutan las consultas.

🖥️ Formulario Tkinter ➡️ 📄 crud_ops.py (procesa) ➡️ 🐬 MySQL (INSERT/UPDATE/DELETE)
🛠️ En Windows: asegúrate que el servicio MySQL esté corriendo (Services → MySQL).

3. Operaciones CRUD

📝 Crear (INSERT)
def crear_usuario(nombre, email):
    conn = get_connection()
    cursor = conn.cursor()
    sql = "INSERT INTO usuarios (nombre, email) VALUES (%s, %s)"
    cursor.execute(sql, (nombre, email))
    conn.commit()
    conn.close()

📤 viaje nombre/email → cursor.execute → commit → cierra

📖 Leer (SELECT)
def obtener_usuarios():
    conn = get_connection()
    cursor = conn.cursor(dictionary=True)
    cursor.execute("SELECT * FROM usuarios")
    resultados = cursor.fetchall()
    conn.close()
    return resultados
✏️ Actualizar (UPDATE)
def actualizar_usuario(id, nombre, email):
    conn = get_connection()
    cursor = conn.cursor()
    sql = "UPDATE usuarios SET nombre=%s, email=%s WHERE id=%s"
    cursor.execute(sql, (nombre, email, id))
    conn.commit()
    conn.close()
🗑️ Eliminar (DELETE)
def eliminar_usuario(id):
    conn = get_connection()
    cursor = conn.cursor()
    cursor.execute("DELETE FROM usuarios WHERE id=%s", (id,))
    conn.commit()
    conn.close()

4. Interfaz Tkinter (main.py)

🧪 Ventana con campos y botones

import tkinter as tk
from tkinter import messagebox
from crud_ops import crear_usuario, obtener_usuarios, actualizar_usuario, eliminar_usuario

def guardar_datos():
    nombre = entry_nombre.get()
    email = entry_email.get()
    if nombre and email:
        crear_usuario(nombre, email)   # ← viaje de datos aquí
        messagebox.showinfo("Éxito", "Usuario guardado")
        limpiar_campos()
    else:
        messagebox.showerror("Error", "Completa todos los campos")

# ... widgets: labels, entries, botones ...

🎯 Flujo completo: El usuario escribe → click en botón → se llama a guardar_datos() → crear_usuario() → conexión MySQL → INSERT → feedback.

🧭 El viaje de los datos (paso a paso)

1. Usuario escribe en Entry → entry_nombre.get() obtiene el texto.
2. Botón "Guardar" → comando asociado guardar_datos().
3. Función guardar_datos() → valida, llama a crear_usuario(nombre, email) (desde crud_ops).
4. crud_ops.crear_usuario() → abre conexión con get_connection() (db_config).
5. Ejecuta SQL → cursor.execute("INSERT ...", (nombre, email)) → la instrucción viaja al servidor MySQL.
6. commit() → persiste el cambio en la base de datos.
7. Cierra conexión → conn.close() y retorna. El usuario ve el mensaje de éxito.

📋 Mostrar datos en Tkinter

def listar_usuarios():
    for widget in frame_lista.winfo_children():
        widget.destroy()
    usuarios = obtener_usuarios()   # SELECT *
    for u in usuarios:
        lbl = tk.Label(frame_lista, text=f"{u['id']} - {u['nombre']} ({u['email']})")
        lbl.pack()

🔁 Los datos viajan desde la base de datos → obtener_usuarios() → se renderizan en labels dentro de un Frame.

✅ Buenas prácticas en Windows

  • Usa entorno virtual → evita conflictos de paquetes.
  • Separa responsabilidades → interfaz (Tkinter) / lógica (CRUD) / conexión (db_config).
  • Maneja excepciones → try/except en operaciones de base de datos.
  • Cierra siempre la conexión → para evitar fugas de recursos.
  • Usa parametrización → (%s) para evitar inyección SQL.
  • En Windows: si usas python y no funciona, prueba con py.

🧠 Recuerda: el viaje de datos es circular: formulario → procesamiento → BD → y de vuelta para mostrar.
