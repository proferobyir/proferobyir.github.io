Conceptos Esenciales de Docker

Bienvenidos al laboratorio práctico de Docker. En este entorno estudiaremos cómo empaquetar, distribuir y ejecutar aplicaciones de forma aislada y eficiente utilizando contenedores independientes de la infraestructura física subyacente.

1. ¿Qué es un Contenedor?

Un contenedor es un proceso aislado en el espacio de usuario que ejecuta código sobre el kernel del sistema operativo anfitrión. A diferencia de las máquinas virtuales tradicionales, no requiere cargar un sistema operativo completo, lo que minimiza drásticamente el consumo de recursos de cómputo.

Nota Pedagógica

Las imágenes son las plantillas de lectura (los planos de construcción), mientras que los contenedores son la instancia en ejecución (el edificio ya construido y operativo).

2. Contenerizar una Aplicación (Paso a Paso)

Para encapsular un proyecto web o de backend, primero necesitamos declarar los pasos de construcción en un documento de texto plano llamado obligatoriamente Dockerfile.

Paso A: Crear el archivo Dockerfile

Define el entorno base, las dependencias y el comando de arranque de tu servicio:

Dockerfile 
# 1. Definir la imagen base del ecosistema oficial
FROM node:20-alpine

# 2. Configurar el directorio de trabajo del contenedor
WORKDIR /app

# 3. Copiar manifiestos e instalar dependencias de producción
COPY package*.json ./
RUN npm install --production

# 4. Traspasar el resto del código fuente del proyecto
COPY . .

# 5. Declarar el puerto de escucha lógica
EXPOSE 3000

# 6. Comando ejecutor por defecto
CMD ["npm", "start"]

Paso B: Compilar la Imagen

Ejecuta la construcción en tu terminal local (asegúrate de incluir el punto al final, que indica el contexto del directorio actual):

Terminal / Bash 
docker build -t mi-app-web:v1 .

Paso C: Desplegar el Contenedor

Inicializa la instancia en segundo plano asignando los puertos mapeados del host hacia el contenedor:

Terminal / Bash 
docker run -d -p 8080:3000 --name contenedor-web mi-app-web:v1

Verificación de Éxito

A partir de este momento, tu entorno está sirviendo la aplicación. Abre tu navegador web en la dirección http://localhost:8080 para comprobar la conectividad.
