Guía Inicial de Docker

Aprende los conceptos fundamentales y los comandos esenciales para empezar a contenedorizar tus aplicaciones.

1. ¿Qué es Docker? (Concepto Clave)

Docker nos permite empaquetar una aplicación con todas sus dependencias en una unidad estandarizada llamada contenedor, garantizando que funcione en cualquier computadora.

🧱 La Analogía del Plano y la Casa

Para entender Docker fácilmente sin complicarte con tecnicismos:

  • Imagen: Es el plano de arquitectura de una casa (o la receta de un pastel). Es estática, de solo lectura y no la puedes habitar; solo sirve para construir.
  • Contenedor: Es la casa ya construida en el mundo real utilizando ese plano. Puedes entrar, usarla, modificar cosas dentro y destruirla cuando quieras. ¡Y puedes levantar 10 casas idénticas usando el mismo plano!

3. Validación: Tu primer comando

Antes de descargar herramientas complejas, verifica que tu instalación de Docker funciona correctamente corriendo el "Hola Mundo" oficial:

docker run hello-world

Si la instalación es correcta, verás un mensaje de éxito en tu terminal confirmando que Docker descargó la imagen y levantó el contenedor correctamente.

4. Comandos Esenciales para Contenedores

A continuación se detallan los comandos fundamentales para gestionar contenedores en tu día a día:

# Levantar un servidor web Nginx de prueba
docker run -d -p 8080:80 --name mi-servidor nginx
🔍 ¿Qué significan los modificadores (-d, -p, --name)?

Cuando ejecutas un comando largo como el de arriba, desglosamos sus partes así:

  • -d (Detached): Corre el contenedor en segundo plano. Libera tu terminal para que puedas seguir escribiendo otros comandos.
  • -p 8080:80 (Port): Mapea los puertos. Conecta el puerto 8080 de tu computadora real con el puerto 80 interno del contenedor.
  • --name mi-servidor: Le asigna un nombre personalizado y fácil de recordar para que no tengas que usar IDs largos.
  • Listar contenedores activos: 
    docker ps
    Muestra los contenedores que se están ejecutando en este momento.
  • Listar todos los contenedores (activos e inactivos): 
    docker ps -a
    Útil para ver contenedores que se detuvieron o dieron algún error.
  • Detener un contenedor: 
    docker stop mi-servidor
    Apaga el contenedor de forma segura sin borrar sus archivos internos.
  • Iniciar un contenedor ya existente: 
    docker start mi-servidor
    Vuelve a encender un contenedor que previamente habías detenido.
  • Eliminar un contenedor: 
    docker rm mi-servidor
    Borra por completo el contenedor de tu sistema (debe estar apagado primero).

5. Gestión de Imágenes

Comandos para administrar los "moldes" o planos que descargas de Docker Hub:

  • Listar imágenes locales: 
    docker images
    Muestra los planos/imágenes guardados en el disco duro de tu PC.
  • Descargar una imagen sin ejecutarla: 
    docker pull ubuntu
    Trae la imagen oficial desde Docker Hub directo a tu máquina local.
  • Eliminar una imagen: 
    docker rmi ubuntu
    Borra la imagen para liberar espacio en disco (no puede haber contenedores usándola).
