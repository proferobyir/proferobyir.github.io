Conceptos Esenciales de Docker

Este recurso académico tiene como objetivo introducir las bases del empaquetamiento, distribución y ejecución de aplicaciones de forma aislada utilizando contenedores, abstrayendo la lógica del software de las configuraciones del sistema operativo físico.

1. ¿Qué es un Contenedor?

Un contenedor es un proceso aislado en el espacio de usuario que comparte el kernel del sistema operativo anfitrión. A diferencia de las máquinas virtuales de hipervisor, los contenedores no emulan hardware ni cargan un sistema operativo invitado completo, lo que los hace extremadamente ligeros y veloces en su arranque.

Nota de Concepto Básica

Una Imagen es un archivo estático, inmutable y de solo lectura que contiene el código y entorno necesario. El Contenedor es la instancia viva, portátil y editable en ejecución basada en dicha imagen.

2. Contenerizar una Aplicación Paso a Paso

Para empaquetar un desarrollo, se crea un manifiesto declarativo llamado formalmente Dockerfile dentro de la raíz del proyecto.

Paso A: Estructura del Dockerfile

Configuración típica para un entorno basado en microservicios:

Dockerfile 
# Definir la imagen oficial ligera de Node.js
FROM node:20-alpine

# Establecer directorio interno de trabajo
WORKDIR /usr/src/app

# Copiar manifiestos e instalar dependencias
COPY package*.json ./
RUN npm install --production

# Transferir el código fuente restante
COPY . .

# Declarar el puerto lógico expuesto
EXPOSE 3000

# Comando ejecutor por defecto
CMD ["node", "app.js"]

Paso B: Construcción de la Imagen

Utiliza la terminal de comandos para compilar el plano del contenedor asignándole una etiqueta identificadora:

Terminal / Console 
docker build -t mi-aplicacion:v1.0 .

Paso C: Despliegue en Ejecución

Crea e inicia el contenedor mapeando los puertos de tu máquina host hacia el entorno interno aislado:

Terminal / Console 
docker run -d -p 8080:3000 --name instancia-ejecucion mi-aplicacion:v1.0

Verificación de Despliegue

La aplicación está lista para pruebas. Los estudiantes pueden abrir el navegador e ingresar a la URL local http://localhost:8080 para verificar el acceso lógico directo.
