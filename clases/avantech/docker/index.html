Conceptos Esenciales de Docker

Este recurso académico tiene como objetivo introducir las bases del empaquetamiento, distribución y ejecución de aplicaciones de forma aislada utilizando contenedores, abstrayendo la lógica del software de las configuraciones del sistema operativo físico.

1. ¿Qué es un Contenedor?

Un contenedor es un proceso aislado en el espacio de usuario que comparte el kernel del sistema operativo anfitrión. A diferencia de las máquinas virtuales de hipervisor, los contenedores no emulan hardware ni cargan un sistema operativo invitado completo, lo que los hace extremadamente ligeros y veloces en su arranque.

Nota de Concepto Básica

Una Imagen es un archivo estático, inmutable y de solo lectura que contiene el código y entorno necesario. El Contenedor es la instancia viva, portátil y editable en ejecución basada en dicha imagen.

2. Contenerizar una Aplicación XAMPP Paso a Paso

Para empaquetar un desarrollo basado en PHP y MySQL (el entorno clásico de XAMPP), creamos un manifiesto declarativo llamado Dockerfile para levantar Apache con PHP, junto a una base de datos.

Paso A: Estructura del Dockerfile (Entorno Apache + PHP)

Configuración típica para simular los servicios web de XAMPP en Docker:

Dockerfile 
# Definir la imagen oficial de PHP con Apache incorporado
FROM php:8.2-apache

# Instalar extensiones de MySQL requeridas por XAMPP
RUN docker-php-ext-install mysqli pdo pdo_mysql

# Copiar los archivos locales de tu proyecto al directorio web de Apache
COPY ./src /var/www/html/

# Habilitar el módulo de reescritura de Apache si es necesario
RUN a2enmod rewrite

# Exponer el puerto clásico HTTP
EXPOSE 80

Paso B: Construcción de la Imagen

Utiliza la terminal de comandos para compilar el plano de tu servidor Apache/PHP simulando XAMPP:

Terminal / Console 
docker build -t mi-xampp-servidor:v1.0 .

Paso C: Despliegue en Ejecución

Crea e inicia el contenedor mapeando el puerto web local (8080) hacia el puerto 80 del servidor Apache interno:

Terminal / Console 
docker run -d -p 8080:80 --name contenedor-xampp mi-xampp-servidor:v1.0

Verificación de Despliegue

La simulación de XAMPP está lista. Los estudiantes pueden abrir el navegador e ingresar a la URL local http://localhost:8080 para procesar sus scripts PHP.
