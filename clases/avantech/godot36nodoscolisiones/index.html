Cuerpos Físicos, Detección de Colisiones y Sprites Animados

⏱️ Duración: 3 Horas Académicas 🎯 Nivel: Introductorio 🛠️ Motor: Godot 3.x

📌 Mapa Rápido de Nodos Físicos

💡 Regla de Oro: Todo nodo de física o área necesita un CollisionShape2D como hijo para tener un cuerpo tangible.
Nodo ¿Bloquea paso? ¿Se mueve por código? Casos de Uso Primarios
StaticBody2D ✅ Sí ❌ No Suelos, paredes, estructuras fijas.
KinematicBody2D ✅ Sí ✅ Sí Jugador, enemigos, plataformas móviles.
Area2D ❌ No (Detector) ⚡ Opcional Monedas, zonas de daño, portales.

⏱️ Planificación de la Sesión (3 Horas)

Bloque 1: Fundamentos de Cuerpos 2D (45 min)

  • Diferenciación práctica entre Kinematic, Static y Area.
  • Jerarquía de nodos y la importancia del CollisionShape2D.
  • Configuración del entorno inicial en el escenario.

Bloque 2: Sprites Animados y Capas (45 min)

  • Visual vs. Físico: El disfraz (Sprite) frente al cuerpo real (Colisión).
  • Configuración de AnimatedSprite y recurso SpriteFrames.
  • Creación de estados animados básicos: idle (reposo) y walk (caminar).

☕ Receso Corto (15 min)

Bloque 3: Integración de Código GDScript (45 min)

  • Manejo de entrada de teclado y movimiento vectorizado.
  • Uso de la función move_and_slide().
  • Lógica de recolección en el Area2D usando la señal body_entered.

Bloque 4: Práctica Integradora y Reto (30 min)

  • Montaje de mini-escena: Paredes + Jugador + Moneda.
  • Pruebas continuas de ejecución y ajustes visuales.
  • Desafío guiado para fijación de conceptos.

💻 Código GDScript: Jugador (KinematicBody2D)

Adjunta este script al nodo raíz KinematicBody2D de tu personaje.

Jugador.gd GDScript (Godot 3.x) 
extends KinematicBody2D

# Variables configurables desde el Inspector
export (int) var velocidad = 200

# Almacena la dirección del movimiento
var movimiento = Vector2.ZERO

# Referencia al nodo de animación hijo
onready var sprite_animado = $AnimatedSprite

func _physics_process(delta):
    obtener_input()
    actualizar_animaciones()
    
    # move_and_slide aplica la velocidad y maneja colisiones automáticas
    movimiento = move_and_slide(movimiento)

func obtener_input():
    movimiento = Vector2.ZERO
    
    # Lectura de teclas direccionales
    if Input.is_action_pressed("ui_right"):
        movimiento.x += 1
    if Input.is_action_pressed("ui_left"):
        movimiento.x -= 1
    if Input.is_action_pressed("ui_down"):
        movimiento.y += 1
    if Input.is_action_pressed("ui_up"):
        movimiento.y -= 1
        
    # Normalización para evitar velocidad superior en diagonales
    movimiento = movimiento.normalized() * velocidad

func actualizar_animaciones():
    if movimiento != Vector2.ZERO:
        sprite_animado.play("walk")
        # Orientación horizontal del sprite
        if movimiento.x != 0:
            sprite_animado.flip_h = (movimiento.x < 0)
    else:
        sprite_animado.play("idle")
Nota Técnica sobre Godot 3.x: En Godot 3.x, la función move_and_slide() ya multiplica la velocidad por el parámetro delta internamente. No debe multiplicarse de forma manual dentro de esa instrucción.

🪙 Código GDScript: Moneda / Recolectable (Area2D)

Adjunta este script al nodo raíz Area2D del elemento recolectable y conecta su señal body_entered.

Moneda.gd GDScript (Godot 3.x) 
extends Area2D

# Función conectada desde la pestaña 'Nodos / Señales' (body_entered)
func _on_Moneda_body_entered(body):
    # Validamos que el objeto con el que chocó sea el jugador
    if body is KinematicBody2D:
        print("¡Moneda recolectada!")
        # Elimina el nodo de la escena de forma segura
        queue_free()

⚙️ Claves Prácticas para la Clase

  • Centrado de Origen: Asegura que tanto el Sprite como el CollisionShape2D estén centrados en la posición (0,0) de su nodo padre.
  • Capas de Colisión (Collision Layers):
    • Layer: En qué capa existe el objeto.
    • Mask: Con qué capas decide colisionar el objeto.
