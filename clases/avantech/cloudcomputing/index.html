Cloud Computing

Conceptos esenciales · Modelos · Servicios · Arquitectura serverless

¿Qué es?

Cloud Computing es el modelo de acceso bajo demanda a recursos informáticos (servidores, almacenamiento, bases de datos, redes, software) a través de Internet, con facturación por consumo y sin necesidad de inversión inicial en infraestructura física.

Características esenciales (NIST):
  • Autoservicio bajo demanda
  • Acceso ubicuo a la red
  • Conjunto de recursos compartidos
  • Elasticidad rápida (escalar)
  • Servicio medido (pago por uso)

CapEx vs OpEx: la nube convierte gastos de capital (comprar servidores) en gastos operativos (pagar por hora).

Modelos SPI

Los tres modelos fundamentales de servicio en la nube:

IaaS

Infraestructura como Servicio. Alquilas VMs, almacenamiento, redes. Tú administras el SO y las aplicaciones.

Ej: VMs, discos, VPC

PaaS

Plataforma como Servicio. Alquilas el entorno para desplegar tu código sin preocuparte por el SO ni parches.

Ej: App Service, Elastic Beanstalk

SaaS

Software como Servicio. Usas aplicaciones listas en la nube (Gmail, Office 365, Salesforce).

Ej: Gmail, Netflix
Modelo de responsabilidad compartida: El proveedor protege la nube (hardware, red). Tú proteges lo que pones en la nube (datos, acceso, SO).

Regiones & AZ

Región: ubicación geográfica con centros de datos. Zonas de Disponibilidad (AZ): centros de datos independientes dentro de una región.

  • Alta disponibilidad = desplegar en ≥ 2 AZ
  • Resiliencia: si una AZ falla, las otras siguen funcionando
  • Latencia: elegir región cercana a los usuarios
Redes (VPC):
  • Subnet pública (con Internet) y privada (sin salida directa).
  • Security Groups (firewall de instancias) vs ACLs (firewall de subred).

Modelos de despliegue: Público · Privado · Híbrido · Multicloud.

Almacenamiento & BD

Objetos
S3, Blob → archivos, estático, escalable
Bloques
EBS, discos persistentes para VMs
Archivos
EFS, NFS → compartido entre VMs

Bases de datos:

  • SQL (relacional) → ACID, transacciones (RDS, Cloud SQL)
  • NoSQL → documental (DynamoDB), clave-valor, en memoria (Redis)
  • Data Warehouse → analítica masiva (BigQuery, Redshift)

Seguridad & IAM

IAM (Identity and Access Management) es el pilar de la seguridad. Define quién (usuario/rol) puede hacer qué (acción) sobre qué (recurso).

  • Principio de mínimo privilegio
  • Cifrado en reposo (discos) y en tránsito (TLS)
  • Auditoría (CloudTrail / Activity Logs)
  • Cumplimiento (GDPR, HIPAA, ISO)
Políticas: documentos JSON que definen permisos. Ejemplo: { "Effect": "Allow", "Action": "s3:GetObject", "Resource": "arn:aws:s3:::mi-bucket/*" }

Escalabilidad & Resiliencia

  • Escalado vertical: cambiar a una VM más grande.
  • Escalado horizontal: añadir más VMs (Auto Scaling).
  • Balanceadores de carga: distribuyen tráfico entre instancias (capa 7 HTTP o capa 4 TCP).
  • Patrones de resiliencia: alta disponibilidad (≥2 AZ), tolerancia a fallos, recuperación ante desastres (DR).
Mensajería asíncrona: Colas (SQS, Pub/Sub) para desacoplar servicios. Un servicio produce mensajes, otro los consume cuando puede.

FinOps: gestión financiera de la nube. Usar instancias spot (sobrante barato), reservadas (1-3 años) y Savings Plans para optimizar costos.

Automatización (IaC)

Infraestructura como Código (IaC): definir toda la infraestructura en archivos de configuración (Terraform, CloudFormation).

  • CI/CD: pipelines de integración y despliegue continuo.
  • Configuración como Código: Ansible, Chef, Puppet.
  • Observabilidad: métricas + logs + trazas (seguir una petición entre microservicios).

// Importar SDK const { DynamoDBClient, PutItemCommand } = require("@aws-sdk/client-dynamodb"); const db = new DynamoDBClient({ region: "us-east-1" }); exports.handler = async (event) => { const { cedula, nombre, email } = JSON.parse(event.body); const params = { TableName: "Usuarios", Item: { "Cedula": { S: cedula }, "Nombre": { S: nombre }, "Email": { S: email }, "Fecha": { N: Date.now().toString() } } }; await db.send(new PutItemCommand(params)); return { statusCode: 200, body: JSON.stringify({ mensaje: "✅ Registrado" }) }; };
