☁️ Cloud Computing & AWS

Aprende paso a paso cómo funciona la nube y cómo desplegar aplicaciones web

📚 PASO 1: ¿Qué es Cloud Computing?

Cloud Computing significa usar recursos de computación (servidores, almacenamiento, bases de datos) a través de Internet, pagando solo por lo que usas. Es como pagar por la electricidad: pagas lo que consumes, no tienes que construir tu propia planta eléctrica.

🏢 IaaS - Infraestructura

Alquilas servidores, almacenamiento y redes. Tú controlas el sistema operativo.

Ejemplo: Amazon EC2, Google Compute Engine

⚙️ PaaS - Plataforma

Te concentras solo en tu código. El proveedor maneja los servidores y el sistema operativo.

Ejemplo: AWS Elastic Beanstalk, Heroku

📱 SaaS - Software

Aplicaciones completas que usas por Internet. No instalas nada en tu computadora.

Ejemplo: Gmail, Netflix, Spotify
🎯 PARA RECORDAR: IaaS = alquilas la computadora, PaaS = alquilas el entorno para programar, SaaS = usas la aplicación directamente.

🌎 PASO 2: Regiones y Zonas de Disponibilidad

AWS tiene centros de datos en todo el mundo. Cada Región es una ubicación geográfica (ej: Norte de Virginia). Cada Región tiene múltiples Zonas de Disponibilidad (AZs) que son centros de datos separados físicamente.

📍 Regiones de AWS

  • us-east-1 - Norte de Virginia (EE.UU.)
  • us-west-2 - Oregón (EE.UU.)
  • eu-west-1 - Irlanda (Europa)
  • sa-east-1 - São Paulo (Brasil)
  • ap-northeast-1 - Tokio (Japón)

🎯 ¿Por qué elegir bien la región?

  • Latencia: Más cerca del usuario = más rápida la respuesta
  • Precios: Cada región tiene costos diferentes
  • Leyes: Algunos datos deben quedarse en un país específico
  • Disponibilidad: Si una AZ falla, las otras siguen funcionando

🏷️ PASO 3: ARN - El "Cédula de Identidad" de cada recurso

ARN (Amazon Resource Name) es un identificador único que AWS usa para cada recurso. Así como tú tienes una cédula, cada recurso tiene un ARN.

arn: aws: s3: : 123456789012: bucket/mi-bucket
🔴 arn: Prefijo fijo
🟠 aws: Partición (nube pública)
🟢 s3: Servicio AWS
🔵 (vacío): S3 es global, no lleva región
🟣 123456789012: Tu ID de cuenta
⚫ bucket/mi-bucket: Recurso específico

🎮 Prueba: Construye un ARN

arn:aws:s3:::bucket/fotos-usuarios

⚡ PASO 4: Servicios que usaremos

AWS IAM

Identity and Access Management - Controla quién puede hacer qué en tu nube.

🔐 Servicio GLOBAL

AWS Lambda

Serverless - Ejecuta código sin administrar servidores. Solo pagas cuando se ejecuta.

⚡ Por demanda - sin servidores

API Gateway

Puerta de entrada - Crea URLs públicas que activan tus funciones Lambda.

🌐 El "puente" entre el mundo exterior y Lambda

Amazon S3

Simple Storage Service - Almacena archivos, imágenes, videos. Ideal para fotos de perfil.

📦 Almacenamiento de objetos

Amazon DynamoDB

Base de datos NoSQL - Rápida, escalable y totalmente administrada.

💾 Guarda datos de forma segura

📝 PASO 5: Ejemplo Real - Formulario con Foto + Datos

Vamos a crear una aplicación completa que:

  • 📋 Muestra un formulario con Cédula, Nombre, Email y Foto de perfil
  • 🖼️ La foto se guarda en S3 Bucket (almacenamiento de archivos)
  • 📝 Los datos del usuario se guardan en DynamoDB
  • 🚀 Todo se procesa con AWS Lambda (sin servidores)

🎮 ¡Prueba el formulario completo aquí!

💡 1. Completa los campos
💡 2. Selecciona una foto (opcional)
💡 3. Haz clic en "Enviar" para ver el flujo completo

🖼️ ¿Para qué usamos S3 Bucket en este ejemplo?

S3 (Simple Storage Service) es donde AWS guarda archivos (imágenes, videos, documentos). Es como un "disco duro en la nube" pero infinito y muy económico.

  • La foto del usuario se guarda en S3 para que pueda ser accedida después
  • La URL de la foto (dónde está guardada) se guarda en DynamoDB junto con los datos del usuario
  • Ventajas: Las fotos no abruman la base de datos, S3 es más barato para archivos grandes

📄 5.1 El Formulario HTML con Foto (Frontend)

📁 frontend.html - Formulario con selector de archivo
<!-- El input de tipo "file" permite seleccionar una foto --> <input type="file" id="fotoPerfil" accept="image/*"> <!-- JavaScript que convierte la foto a Base64 y la envía a Lambda --> <script> const API_URL = "https://xxxxx.execute-api.us-east-1.amazonaws.com/prod/registrar"; async function enviarFormulario() { // 1. Obtener los datos del formulario const cedula = document.getElementById('cedula').value; const nombre = document.getElementById('nombre').value; const email = document.getElementById('email').value; // 2. Leer la foto seleccionada (si hay) const archivoFoto = document.getElementById('fotoPerfil').files[0]; let fotoBase64 = ""; if (archivoFoto) { // Convertir la foto a Base64 para enviarla en JSON fotoBase64 = await leerArchivoComoBase64(archivoFoto); } // 3. Armar el objeto con todos los datos (incluyendo la foto) const datosCompletos = { cedula: cedula, nombre: nombre, email: email, foto: fotoBase64, // La foto en formato Base64 nombreArchivo: archivoFoto ? archivoFoto.name : "", tipoArchivo: archivoFoto ? archivoFoto.type : "" }; // 4. Enviar a AWS Lambda const respuesta = await fetch(API_URL, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(datosCompletos) }); const resultado = await respuesta.json(); alert(resultado.mensaje); } // Función auxiliar para leer archivos como Base64 function leerArchivoComoBase64(archivo) { return new Promise(function(resolver, reject) { const lector = new FileReader(); lector.onload = () => resolver(lector.result.split(',')[1]); lector.onerror = reject; lector.readAsDataURL(archivo); }); } </script>

⚡ 5.2 La Función Lambda (Backend que guarda en S3 y DynamoDB)

📁 lambda.js - Guarda foto en S3 y datos en DynamoDB
// ============================================ // AWS LAMBDA - Guarda foto en S3 y datos en DynamoDB // Se ejecuta CADA VEZ que alguien envía el formulario // ============================================ // 1. IMPORTAR las bibliotecas de AWS const { S3Client, PutObjectCommand } = require("@aws-sdk/client-s3"); const { DynamoDBClient, PutItemCommand } = require("@aws-sdk/client-dynamodb"); // 2. CREAR los clientes const s3Cliente = new S3Client({ region: "us-east-1" }); const dynamoCliente = new DynamoDBClient({ region: "us-east-1" }); // 3. Nombres de los recursos const BUCKET_NOMBRE = "fotos-perfil-usuarios"; // S3 Bucket const TABLA_NOMBRE = "UsuariosRegistrados"; // DynamoDB Table // 4. FUNCIÓN PRINCIPAL exports.handler = async (evento) => { // 4.1 Extraer los datos que envió el formulario const { cedula, nombre, email, foto, nombreArchivo, tipoArchivo } = JSON.parse(evento.body); // 4.2 Si hay foto, guardarla en S3 let urlFoto = ""; if (foto && foto.length > 0) { // Crear un nombre único para la foto (evita colisiones) const timestamp = Date.now(); const llaveS3 = `${cedula}-${timestamp}-${nombreArchivo}`; // Parámetros para guardar en S3 const parametrosS3 = { Bucket: BUCKET_NOMBRE, Key: llaveS3, Body: Buffer.from(foto, 'base64'), // Convertir Base64 a binario ContentType: tipoArchivo, Metadata: { 'cedula': cedula, 'nombre': nombre } }; // Guardar en S3 await s3Cliente.send(new PutObjectCommand(parametrosS3)); // Construir la URL pública de la foto urlFoto = `https://${BUCKET_NOMBRE}.s3.amazonaws.com/${llaveS3}`; } // 4.3 Guardar los datos del usuario en DynamoDB (incluyendo la URL de la foto) const parametrosDynamo = { TableName: TABLA_NOMBRE, Item: { "Cedula": { S: cedula }, "Nombre": { S: nombre }, "Email": { S: email }, "UrlFoto": { S: urlFoto }, // URL de la foto en S3 "FechaRegistro": { N: Date.now().toString() } } }; await dynamoCliente.send(new PutItemCommand(parametrosDynamo)); // 4.4 Responder al formulario return { statusCode: 200, headers: { "Access-Control-Allow-Origin": "*" }, body: JSON.stringify({ mensaje: `✅ Usuario ${nombre} registrado exitosamente`, urlFoto: urlFoto, cedula: cedula }) }; };

🔍 ¿Qué hace S3 en este código?

  • Línea 25-26: Importamos S3Client y PutObjectCommand - las herramientas para hablar con S3
  • Línea 29: Creamos el cliente S3 que se conecta al servicio
  • Línea 33: Definimos el nombre del bucket "fotos-perfil-usuarios" (este bucket debe crearse antes)
  • Línea 44-56: Preparamos los parámetros: Bucket (dónde), Key (nombre del archivo), Body (el contenido de la foto), ContentType (tipo de imagen)
  • Línea 59: ¡Guardamos la foto en S3 con s3Cliente.send()!
  • Línea 62: Generamos la URL pública para poder ver la foto después
  • Línea 70: Guardamos la URL en DynamoDB junto con los datos del usuario

🔐 5.3 Política IAM (Permisos para S3 + DynamoDB)

📁 iam-policy.json - Permisos que necesita Lambda
{ "Version": "2012-10-17", "Statement": [ { "Sid": "PermisoParaGuardarEnS3", "Effect": "Allow", "Action": "s3:PutObject", "Resource": "arn:aws:s3:::fotos-perfil-usuarios/*" }, { "Sid": "PermisoParaGuardarEnDynamoDB", "Effect": "Allow", "Action": "dynamodb:PutItem", "Resource": "arn:aws:dynamodb:us-east-1:123456789012:table/UsuariosRegistrados" } ] } // 🔐 Lambda SOLO puede escribir en S3 (subir fotos) y SOLO puede escribir en esa tabla // No puede leer, no puede borrar, no puede acceder a otros recursos

📦 5.4 ¿Cómo se ve la foto guardada en S3?

Cuando el usuario sube una foto, Lambda la guarda en S3 con esta estructura:

Bucket: fotos-perfil-usuarios/
└── 12345678-1704067200000-mi-foto.jpg

URL pública: https://fotos-perfil-usuarios.s3.amazonaws.com/12345678-1704067200000-mi-foto.jpg

✅ Ventajas de guardar fotos en S3:

  • Las fotos no saturan la base de datos (DynamoDB es para datos pequeños)
  • S3 es más económico para archivos grandes (imágenes, videos, PDFs)
  • S3 entrega las fotos rápidamente a los usuarios (CDN integrado)
  • Puedes hacer que las fotos sean públicas o privadas con permisos IAM

📊 PASO 6: Diagrama del Flujo Completo

🔄 ¿Qué pasa cuando el usuario envía el formulario con foto?

PASO 1: Usuario completa el formulario (Cédula, Nombre, Email) y selecciona una foto.
PASO 2: JavaScript convierte la foto a Base64 y envía TODO a API Gateway mediante fetch().
PASO 3: API Gateway recibe la petición y activa Lambda.
PASO 4: Lambda guarda la foto en S3 Bucket y obtiene la URL.
PASO 5: Lambda guarda los datos del usuario + la URL de la foto en DynamoDB.
PASO 6: Lambda responde con éxito y la URL de la foto.
PASO 7: El usuario ve el mensaje: "✅ Usuario registrado exitosamente".
🎯 PUNTOS CLAVE SOBRE S3:
  • S3 es para ARCHIVOS (imágenes, videos, documentos, backups)
  • DynamoDB es para DATOS ESTRUCTURADOS (texto, números, fechas)
  • La URL de la foto se guarda en DynamoDB para saber dónde está cada foto
  • Los ARN de S3 tienen formato: arn:aws:s3:::nombre-del-bucket/ruta
  • S3 es REGIONAL - eliges en qué región se guardan tus archivos
