🔐 AWS IAM

Identity & Access Management

Seguridad granular · Control total
IAM permite administrar usuarios, grupos, roles y políticas de acceso a los recursos AWS de forma segura. Sigue el principio de mínimo privilegio y autenticación multifactor (MFA).
  • Usuarios y grupos – identidades permanentes con credenciales
  • Roles AWS – asume permisos temporales para aplicaciones o servicios
  • Políticas JSON – define qué acciones se permiten o deniegan
  • MFA / SSO – autenticación robusta y federación de identidades

📦 Amazon S3

Simple Storage Service

Almacenamiento objeto · escalable infinito
S3 es el servicio de almacenamiento en la nube más popular del mundo. Ideal para backups, sitios web estáticos, data lakes y contenido multimedia. Ofrece alta durabilidad (11 nueves) y seguridad avanzada.
  • Buckets & Objects – contenedores globales con clave-valor y metadatos
  • Clases de almacenamiento – S3 Standard, Intelligent-Tiering, Glacier para archivado
  • Políticas de bucket & versionado – control de acceso y recuperación ante borrados
  • Eventos y notificaciones – dispara Lambdas o procesos automáticos

⚙️ IAM + S3: Integración Potente

Políticas de bucket y roles para acceso seguro

Combinar IAM y S3 permite un control preciso sobre quién puede leer/escribir objetos. Utiliza políticas de recursos (bucket policies) junto a roles IAM para una postura de seguridad robusta.
Acceso privado por defecto Pre-signed URLs Cargas seguras
  • Roles de servicio EC2 → S3: sin credenciales hardcodeadas
  • Condiciones IAM (IP, MFA): restringir acceso solo desde ubicaciones autorizadas
  • Block Public Access y políticas anti-riesgos
