☁️ Cloud Computing & AWS

Aprende desde cero cómo funciona la nube y cómo crear aplicaciones web

📚 ¿Qué es Cloud Computing?

Cloud Computing = usar computadoras que están en internet, pagando solo por lo que usas.

📌 Ejemplo: En lugar de comprar tu propio servidor (que cuesta $10,000), alquilas uno en AWS por $10 al mes.

🏢 IaaS

Alquilas servidores y almacenamiento. Tú controlas todo.

Ej: EC2

⚙️ PaaS

Alquilas un lugar para poner tu código. Ellos manejan los servidores.

Ej: Elastic Beanstalk

📱 SaaS

Usas aplicaciones que están en internet. Ej: Gmail, Netflix.

Ej: Gmail

🌍 Regiones de AWS (¿Dónde están las computadoras?)

AWS tiene centros de datos en todo el mundo. Cada Región es un lugar físico:

  • 🇺🇸 us-east-1 - Norte de Virginia
  • 🇺🇸 us-west-2 - Oregón
  • 🇪🇺 eu-west-1 - Irlanda
  • 🇧🇷 sa-east-1 - São Paulo
💡 ¿Por qué importa la región?
• Si tus usuarios están en Latinoamérica, usa sa-east-1 (más rápido)
• Los precios cambian según la región
• Cada región tiene 3 centros de datos (Zonas de Disponibilidad)

🏷️ ARN - La "Cédula" de cada recurso

Cada servidor, base de datos o archivo en AWS tiene un identificador único llamado ARN.

arn:aws:s3:::mi-bucket/foto.jpg

🔴 arn: | 🟠 aws: | 🟢 s3: | 🔵 (vacío) | 🟣 cuenta | ⚫ bucket/foto.jpg

⚡ Servicios de AWS que se utilizan con más frecuencia

API Gateway

¿Qué es? Es una "puerta" que recibe peticiones de tu formulario y activa Lambda.

🌐 Genera una URL pública

Lambda

¿Qué es? Es código que se ejecuta SOLO cuando alguien envía el formulario. No hay servidor 24/7.

📝 Se paga por ejecución

S3

¿Qué es? Es un "disco duro en la nube" para guardar fotos, videos, archivos.

🖼️ Guardamos aquí las fotos

DynamoDB

¿Qué es? Es una base de datos rápida para guardar textos y números.

📝 Guardamos aquí cédula, nombre, email

🔑 ¿CÓMO SE OBTIENE LA URL DEL API GATEWAY?

Antes de que el formulario funcione, AWS tiene que generar una URL especial que conecta todo.

PASO 1 Crear la función Lambda
Escribimos el código que procesará los datos del formulario.
PASO 2 Crear API Gateway
En AWS Console, creamos una "API" y la conectamos a nuestra función Lambda.
PASO 3 ¡AWS genera la URL automáticamente!
AWS nos da una URL como esta: https://xxxxx.execute-api.us-east-1.amazonaws.com/prod/registrar
PASO 4 Copiar la URL en nuestro formulario
Ponemos esa URL en nuestro código JavaScript donde dice API_URL.
🎯 ¿Qué es esa URL?
Es como un "número de teléfono" para tu función Lambda. Cuando el formulario llama a esa URL (con fetch), AWS sabe que debe activar tu código. El "xxxxx" es único para tu cuenta de AWS.

📊 RESUMEN: ¿CÓMO FUNCIONA TODO?

2 🌐 JavaScript envía los datos a API Gateway URL
3 ⚡ API Gateway activa AWS Lambda
4 💾 Lambda guarda los datos en DynamoDB
5 ✅ El usuario ve el mensaje "¡Registrado exitosamente!"
🎯 LO MÁS IMPORTANTE PARA RECORDAR:
  • API Gateway genera una URL única que conecta tu formulario con Lambda
  • No necesitas un servidor - Lambda se ejecuta solo cuando alguien envía el formulario
  • Solo pagas cuando usas - si nadie envía el formulario, no pagas nada
  • DynamoDB guarda los datos de forma segura
