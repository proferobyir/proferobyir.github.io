2. Instalación Paso a Paso (Evitando Errores Comunes)

El error de configuración más habitual entre estudiantes es omitir el Kit de Desarrollo (.NET SDK) o descargar la versión clásica del motor. Sigue estrictamente este orden:

Paso 1: Instalar .NET SDK

Godot no compilará código C# sin las herramientas de compilación oficiales de Microsoft. Descarga e instala el SDK correspondiente a tu arquitectura (x64):

Descargar .NET SDK Oficial

Paso 2: Descargar Godot Engine .NET

Entra a la web oficial y baja obligatoriamente la versión identificada como Godot Engine - .NET. La versión estándar NO soporta scripts de C#.

Descargar Godot Engine .NET

⚠️ Regla de Oro para Estudiantes:

Godot es portable (no requiere instalador). Descomprime el archivo ZIP en una ruta raíz limpia de tu disco duro, por ejemplo: C:\Godot\. No lo ejecutes directamente desde el archivo ZIP comprimido o perderás tus configuraciones.

3. Reconocimiento de la Interfaz de Godot

Al abrir el editor por primera vez, te encontrarás con cinco zonas de control indispensables. Entender la distribución evitará confusiones al momento de construir tus niveles:

Mapa Visual de la Interfaz Gráfica

  1. Panel Escena (Arriba Izquierda): Muestra la estructura de árbol de tu nivel actual. Todo elemento en pantalla nace aquí.
  2. Panel de Sistema de Archivos (Abajo Izquierda): Tu explorador de archivos interno. Contiene sprites, escenas (.tscn) y códigos (.cs).
  3. Visor Central (Viewport): El lienzo gráfico donde arrastras, escalas y posicionas tus componentes visuales.
  4. Panel Inferior: Contiene la consola de **Salida**, vital para ver impresiones de depuración mediante código.
  5. Inspector (Derecha): Cambia dinámicamente según el nodo seleccionado, exponiendo campos editables como la posición, rotación o físicas.

4. Proyecto 1: "Hola Mundo" desde la Consola del Motor

Diseñado para familiarizarse con el ciclo de vida de inicialización de un script de C#.

Flujo de Configuración:

  1. Crea un nuevo proyecto en la ventana inicial de Godot (activa la casilla de soporte .NET).
  2. En el panel **Escena**, pulsa el botón **"Escena 2D"**. Creará un nodo raíz básico llamado Node2D.
  3. Renombra este nodo como Main e introduce la combinación de teclas Ctrl + S para guardarlo como main.tscn.
  4. Con el nodo Main seleccionado, presiona el icono del pergamino con un signo más en la parte superior del panel Escena (**Añadir Script**).
  5. Configura los campos: Lenguaje = **C#**, Hereda de = Node2D y presiona Crear.

Código Fuente (Main.cs):

using Godot;
using System;

public partial class Main : Node2D
{
    // Se ejecuta una sola vez cuando el nodo se monta con éxito en el motor
    public override void _Ready()
    {
        // Imprime un mensaje académico en la pestaña de Salida inferior
        GD.Print("¡Inicialización Exitosa! Hola Mundo desde C# en Godot.");
    }
}

Prueba el proyecto pulsando **F5**. Verás el mensaje impreso en el panel inferior.

5. Proyecto 2: Movimiento y Sistema de Colisiones Sólidas

En este bloque aprenderás a capturar las entradas de teclado del alumno y a implementar un sistema de colisiones físicas reales que impida al jugador atravesar muros o salirse de los límites establecidos.

Paso A: Mapeado de Controles del Teclado

  1. Entra en la barra de opciones: Proyecto -> Configuración del Proyecto.
  2. Ve a la pestaña Mapa de Entradas (Input Map).
  3. Escribe mover_derecha en la barra de texto superior y pulsa **Añadir**. Realiza lo mismo para las acciones: mover_izquierda, mover_arriba y mover_abajo.
  4. Presiona el botón **"+"** al lado de cada acción añadida y pulsa la tecla física que desees vincular (ej. Flechas del teclado o teclas W, A, S, D).

Paso B: Construcción de la Escena Física del Jugador

Para procesar colisiones en dos dimensiones, requerimos asociar un cuerpo de física con una forma geométrica rígida definida:

Estructura de Nodos Obligatoria para Físicas

  1. Crea una escena nueva limpia. Añade como nodo raíz un **CharacterBody2D** y nombralo Jugador.
  2. Haz clic derecho sobre Jugador -> **Añadir Nodo Hijo**, e incorpora un **Sprite2D**.
  3. Arrastra el logotipo icon.svg desde tus archivos hasta el recuadro **Texture** dentro del Inspector del Sprite2D.
  4. Haz clic derecho sobre Jugador -> **Añadir Nodo Hijo**, y añade un **CollisionShape2D**. *(Verás un triángulo de advertencia amarillo indicando que falta definir la geometría)*.
  5. Selecciona el CollisionShape2D, ve al Inspector de la derecha, localiza el campo **Shape**, dale clic y elige **`New RectangleShape2D`**.
  6. En el visor central verás un cuadro celeste claro sobre la imagen del logotipo. Estíralo desde sus controladores hasta encuadrar el sprite por completo.
  7. Guarda la escena como jugador.tscn y añádele un Script de **C#** al nodo raíz.

Código de Movimiento Cinemático con Colisiones (Jugador.cs):

using Godot;
using System;

public partial class Jugador : CharacterBody2D
{
    // Expone la propiedad en el Inspector para facilitar su calibración sin tocar código
    [Export]
    public float Velocidad { get; set; } = 350.0f;

    // Se ejecuta de manera sincronizada con el motor de físicas interno de Godot
    public override void _PhysicsProcess(double delta)
    {
        Vector2 direccionEntrada = Vector2.Zero;

        // Evalúa las pulsaciones físicas del teclado configuradas en el Input Map
        if (Input.IsActionPressed("mover_derecha"))     direccionEntrada.X += 1.0f;
        if (Input.IsActionPressed("mover_izquierda"))   direccionEntrada.X -= 1.0f;
        if (Input.IsActionPressed("mover_abajo"))       direccionEntrada.Y += 1.0f;
        if (Input.IsActionPressed("mover_arriba"))      direccionEntrada.Y -= 1.0f;

        // Evita que el movimiento diagonal duplique la velocidad normal del objeto
        if (direccionEntrada != Vector2.Zero)
        {
            direccionEntrada = direccionEntrada.Normalized();
        }

        // Velocity es una palabra clave nativa y reservada de la clase CharacterBody2D
        Velocity = direccionEntrada * Velocidad;

        // MÉTODO CLAVE: Desplaza el objeto calculando de forma automática fricciones, 
        // deslizamientos y choques contra otros límites del entorno sólido.
        MoveAndSlide();
    }
}

Paso C: Creación de Muros o Obstáculos de Colisión

Para comprobar que el código funciona y frena ante un objeto, necesitamos añadir un cuerpo estático en el entorno:

  1. Regresa a tu escena principal main.tscn.
  2. Arrastra el archivo de la escena del personaje (jugador.tscn) desde el Panel de Archivos directamente hacia el interior del Visor Central.
  3. Para crear un obstáculo sólido: Añade un nodo de tipo **StaticBody2D** (ideal para rocas, paredes o suelos) como hijo de Main.
  4. Añádele un **CollisionShape2D** a ese `StaticBody2D` y asígnale un tamaño rectangular grande dentro del mapa.
  5. Presiona **F6** para iniciar la escena. Conduce tu sprite con el teclado hacia la zona del obstáculo invisible; comprobarás que el motor frena el vector de movimiento impidiendo el traspaso de píxeles.
