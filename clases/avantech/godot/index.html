1. El Plano Cartesiano en los Videojuegos

A diferencia de la matemática tradicional, en la mayoría de los motores de videojuegos y desarrollo web, el origen de coordenadas cambia:

  • El punto (0, 0) se encuentra en la esquina superior izquierda de la pantalla.
  • El eje X aumenta hacia la derecha.
  • El eje Y aumenta hacia abajo.
Truco de Diseño
Para que tus objetos aparezcan dentro del recuadro visible del juego, debemos darles valores positivos en el centro del lienzo (por ejemplo, X: 400, Y: 300).

2. Estructura Teatral de Nodos

En Godot, todo se organiza como una obra de teatro: una Escena contiene Nodos (actores) organizados de forma jerárquica (Padres e Hijos).

Para nuestro primer prototipo, tu árbol de nodos debe verse así:

  • Mundo (Node2D - La raíz de todo)
    • Suelo (StaticBody2D - Objeto físico inmóvil)
      • ColorRect (La caja visual verde)
      • CollisionShape2D (La zona sólida de colisión)
    • Jugador (CharacterBody2D - Objeto con físicas controlables)
      • ColorRect (La caja visual roja)
      • CollisionShape2D (La zona sólida de colisión)

3. Ajustes en el Inspector

Para evitar que los objetos se deformen o queden invisibles, asegúrate de configurar las propiedades del panel derecho (Inspector) con las siguientes coordenadas iniciales:

Nodo Padre Transform -> Position Transform -> Scale Hijos (ColorRect / Collision)
Suelo X: 100, Y: 500 X: 1.0, Y: 1.0 Position en X: 0, Y: 0
(Size sugerido: 800 x 60)
Jugador X: 400, Y: 300 X: 1.0, Y: 1.0 Position en X: 0, Y: 0
(Size sugerido: 40 x 40)
¡Cuidado Estudiantes!
Los nodos hijos siempre deben tener su posición local en (0,0). Si mueves un hijo fuera de su padre, la lógica física del juego se romperá.

4. Programando el Movimiento con GDScript

GDScript es el lenguaje nativo de Godot. Es un lenguaje limpio, intuitivo (estilo Python o pseudocódigo) y no requiere tiempos lentos de compilación.

Haz clic derecho en el nodo Jugador, selecciona Añadir Script, asegúrate de elegir el lenguaje GDScript y pega este código de 15 líneas:

extends CharacterBody2D

# Expone la variable al Inspector para cambiarla de forma visual
@export var velocidad: float = 300.0

# El 'Game Loop' físico. Se ejecuta unas 60 veces por segundo de forma infinita
func _physics_process(_delta: float) -> void:
    # Capturamos la dirección usando los mapeos de flechas nativos de Godot
    var direccion := Input.get_vector("ui_left", "ui_right", "ui_up", "ui_down")
    
    if direccion != Vector2.ZERO:
        # Si se presiona una tecla, avanzamos hacia esa dirección
        velocity = direccion * velocidad
    else:
        # Si soltamos las teclas, frenamos suavemente hacia 0
        velocity.x = move_toward(velocity.x, 0, velocidad)
        velocity.y = move_toward(velocity.y, 0, velocidad)

    # Aplica el movimiento y gestiona automáticamente los choques con el suelo
    move_and_slide()

5. Guía de Supervivencia: Primeros Auxilios

Si las cosas no salen como esperabas al presionar F5, usa estos trucos de desarrollador profesional:

¿El personaje desapareció del mapa en el editor?

Selecciona el nodo perdido en el árbol izquierdo y presiona la tecla F. El editor centrará la cámara sobre él automáticamente.

¿El juego arranca pero las teclas no responden?

  1. Asegúrate de haber guardado los cambios del script presionando Ctrl + S.
  2. Cuando se abra la ventana gris del juego, haz un clic izquierdo sobre ella para que el sistema operativo sepa que vas a usar el teclado ahí.

¿Escribiste todo bien y aun así sale una línea roja?

Recuerda que en GDScript las constantes van en mayúsculas sostenidas. Escribir Vector2.Zero dará error; la forma correcta es Vector2.ZERO.
