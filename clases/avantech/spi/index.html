Modelos de Servicio en la Nube

Lectura e Infraestructura Visual: IaaS vs PaaS vs SaaS

Infraestructura

IaaS

Tú controlas: El sistema operativo, los entornos y las aplicaciones.

El proveedor da: Servidores virtuales, redes y almacenamiento.

Ejemplos del Mercado:
Amazon EC2
Compute Engine (GCP)
Plataforma

PaaS

Tú controlas: El código de tu aplicación y las bases de datos.

El proveedor da: El entorno listo, servidores web y actualizaciones.

Ejemplos del Mercado:
Google Firebase
Heroku
Software

SaaS

Tú controlas: Configuraciones básicas de usuario y tus datos de consumo.

El proveedor da: El software completo y funcional desde la web.

Ejemplos del Mercado:
Google Workspace (Gmail)
GitHub Platform

Laboratorio y Análisis de Casos (Interactivo)

Caso 1: PyME de Repuestos Automotrices
Necesitan desplegar una Landing Page informativa rápido y no tienen un administrador de servidores ni ingenieros de redes en su equipo.

👉 Solución sugerida para debate: PaaS (Firebase) porque se abstraen por completo de la gestión de la infraestructura física o virtual.

Caso 2: Sistema de Archivo Nacional con Software Legado
Una empresa de software requiere migrar un sistema antiguo que necesita una versión muy específica de Linux Kernel y configuraciones de red personalizadas.

👉 Solución sugerida para debate: IaaS (AWS EC2) porque requieren control absoluto del entorno de bajo nivel de la máquina virtual.
