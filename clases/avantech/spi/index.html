Infraestructura

IaaS

Tú controlas: El sistema operativo, las aplicaciones, los datos y los entornos de ejecución.

El proveedor te da: Servidores virtuales, redes y almacenamiento en bruto.

Plataforma

PaaS

Tú controlas: Solo tu código fuente y la base de datos de tu aplicación.

El proveedor te da: El entorno listo, servidores web automáticos, actualizaciones y SO preconfigurado.

Software

SaaS

Tú controlas: Únicamente las configuraciones de usuario final y tus propios datos de consumo.

El proveedor te da: El software completo, 100% funcional y mantenido directamente desde el navegador web.

Laboratorio y Análisis de Casos (Interactivo)

Lee en voz alta los siguientes requerimientos del cliente para que la clase debata cuál es la opción ideal basándose en la estructura superior:

Caso 1: PyME de Repuestos Automotrices
Necesitan desplegar una Landing Page informativa rápido y no tienen un administrador de servidores ni ingenieros de redes en su equipo.

👉 Solución sugerida para debate: PaaS (Firebase) porque se abstraen de la infraestructura.

Caso 2: Sistema de Archivo Nacional con Software Legado
Una empresa de software requiere migrar un sistema antiguo que necesita una versión muy específica y vieja de Linux Kernel y configuraciones de red personalizadas.

👉 Solución sugerida para debate: IaaS (AWS EC2) porque requieren control absoluto de la máquina virtual.
