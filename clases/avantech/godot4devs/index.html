🎯 Introducción

Si ya conoces HTML, CSS y JavaScript, programar personajes en Godot con C# te resultará familiar. Este material desglosa cada palabra clave, cada objeto y cada método del código usando analogías del mundo web.

🏷️ HTML + CSS Escena + Nodos (estructura visual)
⚡ JavaScript (eventos) _PhysicsProcess (bucle de juego)
📄 DOM / Document Árbol de escenas (jerarquía)
⌨️ addEventListener Input (objeto global de Godot)

🧩 El Código Fuente

using Godot;
using System;

public partial class Jugador : CharacterBody2D
{
    // [Export] → visible en el Inspector (como las DevTools)
    [Export]
    public float Velocidad = 300.0f;

    // _PhysicsProcess → el "requestAnimationFrame" de la física
    public override void _PhysicsProcess(double delta)
    {
        Vector2 velocidadActual = Velocity;

        Vector2 direccion = Input.GetVector(
            "ui_left", "ui_right", 
            "ui_up", "ui_down"
        );

        if (direccion != Vector2.Zero)
        {
            velocidadActual = direccion * Velocidad;
        }
        else
        {
            velocidadActual.X = Mathf.MoveToward(Velocity.X, 0, Velocidad);
            velocidadActual.Y = Mathf.MoveToward(Velocity.Y, 0, Velocidad);
        }

        Velocity = velocidadActual;
        MoveAndSlide(); // ¡magia de colisiones!
    }
}

🔍 Desglose: cada palabra clave y objeto

📦 1. using Godot; using System; — Bibliotecas

📚 Web → Como <script src="..."> o import en ES6. Traen las herramientas del motor (Godot) y del entorno .NET (System).

🧬 2. public partial class Jugador : CharacterBody2D — Clase del personaje

🏗️ Web → Extiende un elemento base, como class MiDiv extends HTMLElement.
partial → el código puede estar en varios archivos (como CSS+JS separados).
CharacterBody2D → el "div" con superpoderes de movimiento y física.

🎛️ 3. [Export] public float Velocidad = 300.0f; — Propiedad editable

🎨 Web → Como una variable CSS que se puede modificar desde las DevTools.
[Export] → la hace visible en el Inspector de Godot.
float → número decimal (como number en JS).

🔄 4. public override void _PhysicsProcess(double delta) — Bucle de física

🔄 Web → Es el requestAnimationFrame pero especializado en física.
override → reemplaza el método del padre.
delta → tiempo entre frames (como el timestamp de requestAnimationFrame).

📐 5. Vector2 — El tipo de dato de coordenadas

📐 Web → Como un objeto con propiedades x e y en JavaScript.

¿Qué es Vector2? Es un tipo de dato estructurado que Godot proporciona para representar pares ordenados (X, Y). Se usa para:

  • Posiciones → new Vector2(100, 200)
  • Direcciones → new Vector2(1, 0) (derecha)
  • Velocidades → new Vector2(300, 0)
  • Escalas → new Vector2(2, 2) (tamaño doble)
💡 Analogía Web: Vector2 es como un objeto JavaScript: 
const vector = { x: 10, y: 20 };
Pero con métodos integrados como Length(), Normalized(), DistanceTo(), etc.

⚡ 6. Velocity — Propiedad de velocidad

⚡ Web → Como las propiedades style.left y style.top pero en forma de Vector2.

Velocity es una propiedad nativa de CharacterBody2D que almacena la velocidad actual del personaje en píxeles por segundo.

⌨️ 7. Input — El objeto global de entrada

🎯 ¡IMPORTANTE! Input es un objeto global (singleton) de Godot que actúa como el puente entre el hardware (teclado, mouse, joystick) y tu juego.

Analogía Web: Input es como el objeto window en JavaScript, que contiene todos los eventos y métodos para interactuar con el usuario.

📋 Métodos principales de Input:

🎮 Método 🎯 Para qué sirve 📤 Retorna 🌐 Analogía Web
Input.GetVector() Movimiento direccional (WASD/Flechas) Vector2 (-1 a 1) Joystick virtual
Input.IsKeyPressed(Key.Shift) Tecla específica (presionada) bool window.addEventListener('keydown')
Input.IsKeyJustPressed(Key.Space) Tecla presionada en este frame bool Evento keydown (solo una vez)
Input.IsActionPressed("saltar") Acción definida en Project Settings bool Evento personalizado
Input.GetMousePosition() Posición del mouse en la pantalla Vector2 event.clientX/Y
Input.IsMouseButtonPressed(MouseButton.Left) Click del mouse bool window.addEventListener('mousedown')
Input.GetJoyAxis(0, JoyAxis.LeftX) Eje del joystick (control) float (-1 a 1) Gamepad API

⌨️ 8. Teclas UI vs WASD — ¡La confusión más común!

🎯 ¡ATENCIÓN! Por defecto, cuando usamos "ui_left", "ui_right", "ui_up" y "ui_down", Godot viene preconfigurado de fábrica para escuchar solo las flechas del teclado (teclas de la interfaz de usuario).

¿El problema? Los jugadores de PC usamos por instinto las clásicas teclas W, A, S, D. ¡Y por defecto NO funcionan con "ui_left", etc.!

⚡ Tutorial: Cambiar flechas por WASD en 2 minutos

🎯 ¡Sigue estos pasos y tendrás WASD funcionando en menos de 2 minutos!
  1. Paso 1: En el menú superior de Godot, ve a Proyecto → Configuración del Proyecto.
    💡 Alternativa: Usa el atajo Ctrl + Shift + P
  2. Paso 2: En la ventana que se abre, haz clic en la pestaña de arriba que dice Mapa de Entradas (o Input Map en inglés).
    💡 Verás una lista de acciones existentes. Si activas "Mostrar acciones del sistema", verás las de ui_left y compañía.
  3. Paso 3: En la barra de arriba escribe, por ejemplo, mover_izquierda y haz clic en el botón Añadir.
    💡 Repite este paso para las 4 direcciones:
    • mover_izquierda
    • mover_derecha
    • mover_arriba
    • mover_abajo
  4. Paso 4: Al lado de la acción que acabas de crear, haz clic en el botón "+", presiona la tecla A en tu teclado y haz clic en Aceptar.
    💡 Así se asigna la tecla A a la acción mover_izquierda.
  5. Paso 5: Repite el paso 4 para cada dirección:
    • mover_derecha → Tecla D
    • mover_arriba → Tecla W
    • mover_abajo → Tecla S
    💡 Si quieres que también funcionen las flechas, añade más teclas a la misma acción con el botón "+".
  6. Paso 6: ¡Ya está! Ahora en tu código de C#, cambia la línea de Input.GetVector por esto: 
    Vector2 direccion = Input.GetVector(
    "mover_izquierda", "mover_derecha", 
    "mover_arriba", "mover_abajo"
);
    ✅ ¡Listo! Ahora tu personaje se mueve con W, A, S, D.
🎯 Analogía Web: Es como si en una página web los eventos de teclado (addEventListener('keydown')) solo funcionaran con las flechas, pero no con las letras W, A, S, D. Tendrías que configurar manualmente ambos conjuntos de teclas para que funcionen.

💡 Recomendación: Siempre usa acciones personalizadas en el Input Map para que los jugadores puedan usar ambos: flechas Y WASD.

✅ Ventaja: Con este método, tus jugadores podrán usar ambos tipos de teclas y tú tendrás un código más limpio y reutilizable.

✨ 9. MoveAndSlide() — ¡La función mágica!

✨ Web → Imagina un método que mueve el elemento y además detecta colisiones automáticamente.

¿Qué hace exactamente?
1. Calcula la nueva posición basada en Velocity
2. Detecta colisiones con otros cuerpos físicos
3. "Desliza" al personaje a lo largo de las superficies con las que choca
4. Actualiza automáticamente el estado de colisión (suelo, pared, techo)

🛑 10. Mathf.MoveToward(Velocity.X, 0, Velocidad); — Desaceleración suave

🛑 Web → Como una transition: left 0.3s ease-out en CSS.

¿Qué hace? Reduce un valor hacia otro a una velocidad determinada.
Mathf.MoveToward(desde, hasta, paso)
• desde → Valor actual (ej: 150)
• hasta → Valor objetivo (ej: 0)
• paso → Velocidad de reducción (ej: 300)

🌍 Métodos de colisión de CharacterBody2D

🎯 ¡IMPORTANTE! Después de llamar a MoveAndSlide(), Godot actualiza automáticamente el estado de colisión del personaje. Puedes consultar este estado con métodos como IsOnFloor(), IsOnWall(), etc.

🌍 11. IsOnFloor() — ¿Está en el suelo?

🎮 Godot → Método de CharacterBody2D que devuelve true si el personaje está tocando el suelo.

Analogía Web: Como preguntar element.getBoundingClientRect().bottom === containerHeight (si el elemento está en la parte inferior del contenedor).

🎯 Uso típico: Saltar 
if (Input.IsKeyJustPressed(Key.Space) && IsOnFloor())
{
    Velocity = new Vector2(Velocity.X, -500.0f); // Salta hacia arriba
}
📌 Importante: Sin IsOnFloor(), el personaje podría saltar en el aire (salto infinito). ¡Esto evita ese problema!

🧱 12. IsOnWall() — ¿Está contra una pared?

🎮 Godot → Método de CharacterBody2D que devuelve true si el personaje está tocando una pared.

Analogía Web: Como preguntar si un elemento <div> está tocando el borde izquierdo o derecho de la pantalla.

🎯 Uso típico: Rebote / deslizamiento 
if (IsOnWall())
{
    // Cambiar dirección o rebotar
    Velocity = new Vector2(-Velocity.X, Velocity.Y);
}

⬆️ 13. IsOnCeiling() — ¿Está contra el techo?

🎮 Godot → Método de CharacterBody2D que devuelve true si el personaje está tocando el techo.

Analogía Web: Como preguntar si un elemento está tocando la parte superior de la pantalla.

🎯 Uso típico: Limitar salto / rebote en techo 
if (IsOnCeiling())
{
    // Detener movimiento hacia arriba
    Velocity = new Vector2(Velocity.X, 0);
}

📐 14. GetLastSlideCollision() — Información de la última colisión

🎮 Godot → Método de CharacterBody2D que devuelve un objeto KinematicCollision2D con información detallada sobre la última colisión que ocurrió durante MoveAndSlide().

🎯 Uso típico: Obtener el objeto con el que chocamos 
KinematicCollision2D colision = GetLastSlideCollision();
if (colision != null)
{
    Node2D objetoChocado = (Node2D)colision.GetCollider();
    GD.Print("Choqué contra: " + objetoChocado.Name);
    
    // Obtener la normal de la superficie
    Vector2 normal = colision.GetNormal();
}
📌 Propiedades útiles:
• GetCollider() → El objeto con el que chocamos
• GetNormal() → Dirección de la superficie (Vector2)
• GetPosition() → Punto de colisión

📏 Métodos matemáticos importantes

📏 15. Mathf.Clamp(valor, minimo, maximo) — Limitar valores

📏 Web → Como Math.min(Math.max(valor, min), max) en JavaScript.

¿Qué hace? Limita un valor entre un mínimo y un máximo.

🎯 Uso típico: Limitar velocidad 
float maxVel = 600.0f;
velocidadActual.X = Mathf.Clamp(velocidadActual.X, -maxVel, maxVel);
velocidadActual.Y = Mathf.Clamp(velocidadActual.Y, -maxVel, maxVel);

📊 16. Mathf.Lerp(desde, hasta, peso) — Interpolación lineal

📊 Web → Similar a las animaciones CSS o easing en JavaScript.

¿Qué hace? Interpola (mezcla) dos valores usando un peso (0 a 1).
• 0 → devuelve desde
• 1 → devuelve hasta
• 0.5 → punto medio entre ambos

🎯 Uso típico: Movimiento suave de cámara 
Vector2 posicionCamara = GetViewport().GetCamera2D().Position;
Vector2 objetivo = Player.Position;
posicionCamara = posicionCamara.Lerp(objetivo, 0.1f); // Seguimiento suave

🧪 Ejercicio práctico

🎯 Reto completo: Implementa un personaje que:
• Se mueva con WASD y Flechas (usa acciones personalizadas como en el tutorial)
• Haga sprint con Shift (Input.IsKeyPressed)
• Salte con Espacio (Input.IsKeyJustPressed + IsOnFloor)
• Interactúe con E (Input.IsActionJustPressed)
• Dispare con click izquierdo (Input.IsMouseButtonPressed)
• Tenga un límite de velocidad (Mathf.Clamp)
• Muestre la posición del mouse (Input.GetMousePosition)
• Rebote al chocar con paredes (IsOnWall)

💡 Solución completa:

public override void _PhysicsProcess(double delta)
{
    // 1️⃣ Obtener dirección (WASD + Flechas con acciones personalizadas)
    Vector2 direccion = Input.GetVector(
        "mover_izquierda", "mover_derecha", 
        "mover_arriba", "mover_abajo"
    );
    
    // 2️⃣ Sprint con Shift
    float velocidadFinal = Velocidad;
    if (Input.IsKeyPressed(Key.Shift))
    {
        velocidadFinal = Velocidad * 2.0f;
    }
    
    // 3️⃣ Calcular velocidad
    Vector2 velocidadActual = Velocity;
    if (direccion != Vector2.Zero)
    {
        velocidadActual = direccion * velocidadFinal;
    }
    else
    {
        velocidadActual.X = Mathf.MoveToward(Velocity.X, 0, Velocidad);
        velocidadActual.Y = Mathf.MoveToward(Velocity.Y, 0, Velocidad);
    }
    
    // 4️⃣ Límite de velocidad con Clamp
    float maxVel = 600.0f;
    velocidadActual.X = Mathf.Clamp(velocidadActual.X, -maxVel, maxVel);
    velocidadActual.Y = Mathf.Clamp(velocidadActual.Y, -maxVel, maxVel);
    
    // 5️⃣ Salto con Espacio (solo si está en el suelo)
    if (Input.IsKeyJustPressed(Key.Space) && IsOnFloor())
    {
        velocidadActual.Y = -500.0f;
    }
    
    // 6️⃣ Interacción con E (acción personalizada)
    if (Input.IsActionJustPressed("interactuar"))
    {
        GD.Print("¡Interactuando!");
    }
    
    // 7️⃣ Disparar con click izquierdo
    if (Input.IsMouseButtonPressed(MouseButton.Left))
    {
        GD.Print("¡Disparando!");
    }
    
    // 8️⃣ Mostrar posición del mouse (cada 60 frames)
    if (Engine.GetProcessFrames() % 60 == 0)
    {
        Vector2 posMouse = Input.GetMousePosition();
        GD.Print("Mouse: " + posMouse);
    }
    
    // 9️⃣ REBOTE: Si choca contra una pared, invierte la velocidad horizontal
    if (IsOnWall())
    {
        velocidadActual.X = -velocidadActual.X;
    }
    
    // 🔟 Aplicar movimiento
    Velocity = velocidadActual;
    MoveAndSlide();
    
    // 1️⃣1️⃣ Información de colisión (opcional)
    KinematicCollision2D colision = GetLastSlideCollision();
    if (colision != null)
    {
        Node2D objeto = (Node2D)colision.GetCollider();
        // GD.Print("Choqué contra: " + objeto.Name); // Descomentar para debug
    }
}

🎯 Conclusión

Programar un personaje en Godot con C# es como construir una aplicación web interactiva:

  • Estructura → Árbol de nodos (como el DOM)
  • Interacción → Input y señales (como eventos)
  • Actualización → Game loop (como requestAnimationFrame)
  • Visualización → Inspector y escena (como HTML/CSS + DevTools)
  • Colisiones → Detección de bordes y límites (como getBoundingClientRect)
  • Teclas UI → Solo flechas por defecto, hay que configurar WASD manualmente
  • Input Map → Como configurar teclas personalizadas en un archivo de configuración

✨ ¡Tus alumnos verán que los videojuegos y las aplicaciones web comparten los mismos fundamentos!
