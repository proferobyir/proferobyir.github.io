Comandos de Docker

Guía rápida de referencia para la administración de n8n

📥 Preparación e Inicio

# Descarga por adelantado las imágenes del archivo docker-compose

docker-compose pull

# Levanta n8n y Postgres en segundo plano (crea si no existen)

docker-compose up -d

# Fuerza la recreación de contenedores si cambiaste variables de entorno

docker-compose up -d --force-recreate

🚀 Ciclo de Vida habitual

# Reinicia ambos servicios rápidamente (útil ante bloqueos leves)

docker-compose restart

# Apaga los contenedores liberando RAM sin borrar tus datos

docker-compose down

# Apaga BORRANDO volúmenes (¡Resetea la BD por completo!)

docker-compose down -v

🔄 Actualización de Versiones

# Paso 1: Descarga la última versión de la imagen (ej. n8nio/n8n:latest)

docker-compose pull n8n

# Paso 2: Aplica los cambios bajando el contenedor viejo y subiendo el nuevo

docker-compose up -d n8n

🔍 Diagnóstico y Estado

# Muestra el estado actual de tus contenedores y puertos activos

docker-compose ps

# Ver qué está haciendo n8n en tiempo real (mira errores de ejecución)

docker-compose logs -f n8n

# Ver los logs de la base de datos Postgres (errores de conexión)

docker-compose logs -f postgres

# Muestra el consumo de CPU, Memoria y red en vivo

docker stats

🧹 Limpieza y Acceso

# Entrar a la consola interna del contenedor de n8n (para revisar archivos)

docker-compose exec n8n ash

# Elimina imágenes intermedias, contenedores apagados y caché (Libera espacio)

docker system prune -a

💾 Respaldos Rápidos

# Crea una copia de seguridad de la BD en un archivo .sql fuera del contenedor

docker-compose exec -t postgres pg_dumpall -U n8n_user > respaldo_n8n.sql

# Restaura el respaldo .sql (El contenedor debe estar activo)

cat respaldo_n8n.sql | docker-compose exec -T postgres psql -U n8n_user -d n8n_db
