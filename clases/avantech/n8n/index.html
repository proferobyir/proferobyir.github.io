📦 El objeto $json en n8n

En n8n, cada nodo recibe y emite datos en formato JSON. El objeto $json representa los datos del ítem actual que fluye por el workflow. Es la forma de acceder a campos, transformarlos y pasarlos entre nodos.

🔍 ¿Qué es $json?

Es una variable especial que contiene el objeto JavaScript con los datos del ítem que está procesando el nodo. Puedes acceder a propiedades usando $json.propiedad o $json["propiedad anidada"].

{
  "nombre": "Ana",
  "email": "ana@ejemplo.com",
  "datos": { "edad": 25 }
}

Para acceder: $json.nombre → "Ana"
$json.datos.edad → 25

✏️ Manipular con Edit Fields

El nodo Edit Fields permite crear o modificar campos usando expresiones con $json.

mensaje = "Hola {{$json.nombre}}, tu email es {{$json.email}}"
fecha_actual = "{{new Date().toISOString()}}"
telefono_cliente = {{$json.formulario.telefono}}

✅ Puedes usar expresiones dentro de {{ }} para cálculos y concatenación.

🧪 Ejemplo práctico: Webhook → Edit Fields → Telegram

Cuando un webhook recibe datos, estos quedan en $json.body (si es POST). Luego en Edit Fields construyes un mensaje:

mensaje_telegram = "📬 Nuevo contacto: {{$json.body.nombre}} - {{$json.body.email}}"

Y en el nodo Telegram usas {{$json.mensaje_telegram}} como texto.

🎮 Playground $json: Simula datos de entrada y prueba cómo se transforman

💡 La expresión usada es: "📬 NUEVO CONTACTO\n👤 Nombre: {{$json.nombre}}\n📧 Email: {{$json.email}}\n📱 Teléfono: {{$json.telefono}}\n💬 Mensaje: {{$json.mensaje}}"

⚡ Importante: $json siempre contiene el item actual. Si tu workflow maneja múltiples items (por ejemplo, desde una base de datos), n8n itera sobre cada uno y aplica las transformaciones.
