¡IMPORTANTE! Prerrequisitos Obligatorios en el Laboratorio

Para que la arquitectura de contenedores funcione de manera local y offline, es **estrictamente obligatorio** verificar y habilitar los siguientes componentes del sistema operativo antes de intentar correr los scripts:

  • Habilitar WSL (Windows Subsystem for Linux): Docker Desktop requiere el núcleo lúdico de Linux para operar en Windows de forma eficiente. Si no está activo, abre PowerShell como Administrador y ejecuta wsl --install, luego reinicia la computadora.
  • Instalar Docker Desktop: Asegúrate de que las estaciones de trabajo cuenten con el motor de Docker instalado y en ejecución. (Si el proxy no permite la descarga, el instalador ejecutable offline debe ser provisto previamente en un pendrive).
  • Verificar Permisos del Motor: El icono de Docker en la barra de tareas debe mostrarse en color verde (Running) antes de dar doble clic al automatizador.
Fase 1: Preparación en Casa Con Internet Libre

El objetivo es extraer y empaquetar la imagen del repositorio global de Docker Hub para transportarla sin consumo de red.

1
Descarga la última imagen oficial estables de n8n ejecutando en tu terminal: 
docker pull n8nio/n8n:latest
2
Exporta la imagen descargada a un archivo binario plano comprimido tipo .tar: 
docker save -o n8n-latest.tar n8nio/n8n:latest
3
Estructura tu directorio local creando un archivo llamado docker-compose.yml y un script automatizador por lotes Iniciar_Laboratorio.bat en la misma carpeta junto al archivo .tar extraído.
4
Comprime toda la estructura en un archivo .zip y súbelo a tu almacenamiento de Google Drive para la distribución masiva en el aula.
Fase 2: Despliegue en el Aula Entorno Bloqueado

Acción del estudiante para evadir el proxy de manera local interactuando con el software portado.

.
1
Descarga el paquete comprimido usando el enlace provisto de Google Drive y descomprímelo en un directorio local de la máquina (ej: Escritorio).
2
Ejecuta el script Iniciar_Laboratorio.bat mediante doble clic. Este comando forzará al motor local de Docker a ingerir la imagen física sin consultar servidores externos: 
docker load -i n8n-latest.tar
3
Docker Compose levantará el contenedor en segundo plano e instanciará el volumen persistente dentro del directorio datos_n8n.
4
Abre tu navegador de preferencia (Chrome o Edge) e ingresa al ecosistema local: 
http://localhost:5678
