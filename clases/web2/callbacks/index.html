Funciones & Asincronía en JS

Conceptos profundos, arquitectura y uso práctico con addEventListener

1. Funciones Tradicionales

Son bloques de código reutilizables. Pueden definirse como Declaraciones o Expresiones. Tienen su propio contexto de ejecución (this) y sufren de Hoisting (pueden invocarse antes de ser declaradas en el código si usas la sintaxis declarativa).

// Declaración (Tiene hoisting)
function calcularArea(base, altura) {
  return base * altura;
}

// Expresión de Función (Guardada en variable, no tiene hoisting)
const saludar = function(nombre) {
  return `Hola, ${nombre}`;
};

2. Funciones Arrow (Flecha)

Introducidas en ES6 (2015). Ofrecen una sintaxis más corta e implícita. Tienen dos características clave:

  • No tienen su propio this: Heredan el this del contexto léxico superior (donde fueron creadas).
  • Retorno implícito: Si ocupan una sola línea, no requieren las llaves {} ni la palabra return.
// Sintaxis completa
const sumar = (a, b) => {
  return a + b;
};

// Sintaxis corta (Retorno Implícito)
const duplicar = x => x * 2;

3. Funciones Ejecutadoras (Callbacks)

En JS, las funciones son "ciudadanos de primera clase". Esto significa que una función se puede pasar como parámetro a otra función como si fuera una variable cualqueira. La función receptora guardará ese código y lo "ejecutará" más tarde.

// 'callback' guarda el CÓDIGO de la función que le enviamos
function procesarUsuario(id, callback) {
  console.log(`1. Buscando usuario ${id}...`);
  
  // 🔍 SI IMPRIMIMOS 'callback' SIN PARÉNTESIS (), VEREMOS SU CONTENIDO:
  console.log("2. Contenido interno de la variable 'callback':", callback);

  // ⚡ AL PONERLE PARÉNTESIS 'callback(...)', LA EJECUTAMOS:
  callback("Carlos");
}

// Invocación: le pasamos el ID 10 y una función Arrow
procesarUsuario(10, (nombre) => {
  console.log(`3. ¡Resultado recibido! Usuario encontrado: ${nombre}`);
});

4. Promesas (Promises)

Objeto que representa el resultado de una operación asíncrona que puede terminar con éxito (resolve) o con error (reject). Evitan el infierno de los callbacks (Callback Hell).

const obtenerDatos = () => {
  return new Promise((resolve, reject) => {
    const exito = true;
    setTimeout(() => {
      if(exito) resolve("¡Datos cargados con éxito!");
      else reject("Error en el servidor");
    }, 1500);
  });
};

// Consumo con .then() y .catch()
obtenerDatos()
  .then(respuesta => console.log(respuesta))
  .catch(error => console.error(error));

5. Funciones Async / Await

Es una "azúcar sintáctica" construida sobre las Promesas. Hace que el código asíncrono se lea y se comporte de forma parecida al código síncrono, mejorando drásticamente la legibilidad.

  • async: Convierte una función para que siempre devuelva una Promesa.
  • await: Pausa la ejecución de la función hasta que la Promesa se resuelva (solo se puede usar dentro de funciones async).
// Definición async + arrow
const pedirDatosServidor = async () => {
  try {
    // Pausa la ejecución aquí hasta que la promesa responda
    const resultado = await obtenerDatos(); 
    console.log(resultado);
  } catch (error) {
    console.error("Falló:", error);
  }
};

6. Formas de Uso en addEventListener

addEventListener recibe como segundo parámetro una función ejecutadora. Aquí verás las 3 formas principales de pasársela:

const boton = document.querySelector('button');

// FORMA 1: Arrow function anónima directamente
boton.addEventListener('click', (e) => {
  console.log("Clic en:", e.target);
});

// FORMA 2: Pasando la referencia de una función declarada
function miManejador(e) {
  console.log("Evento manejado por función externa");
}
boton.addEventListener('click', miManejador); // SIN paréntesis ()

// FORMA 3: Ejecutadora Async para procesos asíncronos
boton.addEventListener('click', async (e) => {
  e.target.disabled = true;
  await consultarAPI();
  e.target.disabled = false;
});
