1. Funciones Tradicionales

💡 En resumen: Son bloques de código que puedes reutilizar. Si las declaras con function, puedes llamarlas incluso antes de haberlas escrito (a esto se le llama Hoisting). 
// 1. Declaración (¡Tiene hoisting! Puedes usarla antes en el código)
function calcularArea(base, altura) {
  return base * altura;
}

// 2. Expresión (Guardada en variable, NO tiene hoisting)
const saludar = function(nombre) {
  return `Hola, ${nombre}`;
};

2. Funciones Arrow (Flecha) =>

💡 En resumen: Una forma moderna y rápida de escribir funciones.
  • Más cortas: Si solo hacen una cosa, no necesitan return ni llaves {}.
  • Comparten el contexto: No tienen su propio this (usan el del lugar donde fueron creadas).
// Sintaxis Completa
const sumar = (a, b) => {
  return a + b;
};

// Sintaxis Corta (Retorno implícito automático)
const duplicar = x => x * 2;

3. Callbacks (Funciones de Respuesta)

💡 En resumen: Pasar una función dentro de otra como si fuera una variable. Es como darle una receta a un amigo para que la prepare después. 
function procesarUsuario(id, miCallback) {
  console.log(`1. Buscando usuario ${id}...`);

  // Al agregar paréntesis (), ejecutamos la función guardada:
  miCallback("Carlos");
}

// Pasamos una función Arrow como parámetro
procesarUsuario(10, (nombre) => {
  console.log(`2. ¡Usuario encontrado: ${nombre}!`);
});

4. Promesas (Promises)

💡 En resumen: Como un ticket de pedido en un restaurante. Promete entregarte comida (éxito) o avisarte si se agotó (error) en el futuro. 
const obtenerDatos = () => {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve("✅ ¡Datos cargados!");
    }, 1500);
  });
};

// Consumo con .then()
obtenerDatos().then(respuesta => console.log(respuesta));

5. Async / Await

💡 En resumen: La forma moderna de leer Promesas. Hace que el código asíncrono parezca ordenado y secuencial (de arriba a abajo).
  • async: Marca una función para que trabaje con asincronía.
  • await: Pausa el código hasta que la Promesa responda.
const pedirServidor = async () => {
  try {
    // Pausa aquí hasta que los datos lleguen
    const resultado = await obtenerDatos(); 
    console.log(resultado);
  } catch (error) {
    console.error("Falló:", error);
  }
};

6. Uso Real en addEventListener

💡 En resumen: Escuchadores de eventos que ejecutan una función cuando el usuario interactúa. 
const boton = document.querySelector('button');

// Opción A: Con Arrow Function anónima
boton.addEventListener('click', (e) => {
  console.log("Clic registrado");
});

// Opción B: Pasando el nombre de la función (SIN paréntesis)
function miManejador() { console.log("Clic externo"); }
boton.addEventListener('click', miManejador); 

// Opción C: Función Async para llamadas a APIS
boton.addEventListener('click', async () => {
  await consultarAPI();
});
