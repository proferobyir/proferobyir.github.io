Paso 1 El Concepto de Asincronía

En JavaScript, operaciones como consultar un servidor web tardan tiempo en completarse. Para evitar que la aplicación web se congele mientras espera la respuesta, utilizamos funciones asíncronas que devuelven una Promesa (un objeto que representa un valor que estará disponible ahora, en el futuro o nunca).

Paso 2 La anatomía de Async / Await

La combinación async/await es una forma elegante y legible de trabajar con promesas, permitiendo escribir código asíncrono que se lee como si fuera síncrono (línea por línea).

  • async: Declara que una función contiene procesos asíncronos. Siempre retorna una promesa.
  • await: Pausa la ejecución de la función de forma interna hasta que la promesa se resuelva (con éxito o error).

Paso 3 Analizando el código de la API

Para conectarnos a la API de Dragon Ball y traer los datos de los personajes, estructuramos nuestra función utilizando un bloque try...catch para capturar posibles errores de red:

async function obtenerPersonajes() {
    try {
        // 1. Enviamos la petición HTTP GET
        const respuesta = await fetch('https://dragonball-api.com/api/characters');
        
        // 2. Convertimos el flujo de datos a formato JSON
        const datos = await respuesta.json();
        
        // 3. Retornamos la estructura de los datos (el arreglo viene en la propiedad 'items')
        return datos.items; 
    } catch (error) {
        console.error("Error en la comunicación con la API:", error);
    }
}
