📚 Teoría: HTML Semántico

Las etiquetas HTML se dividen en dos categorías según su significado (semántica):

🔹 Etiquetas Semánticas

Son aquellas que describen con precisión su propósito y el tipo de contenido que contienen. Por ejemplo:

<h1>Encabezado principal</h1>

En este código, podemos identificar claramente que el contenido dentro de <h1> es un encabezado.

Ejemplos comunes: <h1>-<h6>, <form>, <table>, <main>, etc.

🔸 Etiquetas No Semánticas

No tienen un significado o propósito específico. Se usan como contenedores generales sin proporcionar contexto adicional. Por ejemplo:

<span>Algún texto</span>

No podemos extraer el significado del texto, ya que este código podría pertenecer a cualquier sección del documento.

Ejemplos comunes: <div>, <span>, etc.

📋 Tabla de Etiquetas Semánticas

Etiqueta Descripción
<p> Define un párrafo de un documento
<header> Define el encabezado de un documento o sección
<footer> Define el pie de página de un documento o sección
<nav> Define una sección con enlaces de navegación
<article> Contenido independiente y autónomo
<aside> Contenido relacionado (como una barra lateral)
<main> Especifica el contenido principal del documento
<section> Define una sección temática de un documento
<details> / <summary> Detalles adicionales que el usuario puede ver/ocultar

❓ ¿Por qué usar HTML semántico?

📖

Código más legible

Facilita la comprensión y el mantenimiento del código, especialmente en proyectos grandes.

♿

Mejor accesibilidad

Ayuda a tecnologías asistivas (lectores de pantalla) a interpretar y navegar el contenido.

🔍

Mejor SEO

Los motores de búsqueda indexan mejor el contenido y entienden su jerarquía.

📝 Ejemplo comparativo

✅ Con etiquetas semánticas

<body> <header> <h1>Encabezado</h1> </header> <section> <main> <article> Contenido principal </article> </main> <aside>Información extra</aside> </section> <footer>Copyright 2026</footer> </body>

✨ Código claro y fácil de entender

❌ Con <div> (no semántico)

<body> <div> <h1>Encabezado</h1> </div> <div> <div> <div> Contenido principal </div> </div> <div>Información extra</div> </div> <div>Copyright 2026</div> </body>

⚠️ Difícil de leer y mantener

👉 Ambos ejemplos producen el mismo resultado visual, pero el primero es mucho más fácil de entender para desarrolladores, motores de búsqueda y tecnologías asistivas.

📖 Conceptos fundamentales

Definición

SEO

Search Engine Optimization: conjunto de técnicas para posicionar una web en los resultados de Google.

🎯 Las etiquetas semánticas ayudan al SEO porque los buscadores entienden mejor tu contenido.
Definición

Crawler

Un crawler (araña web) es un robot que recorren internet analizando el código de las páginas.

🕷️ Googlebot lee tus etiquetas semánticas para indexar tu sitio correctamente.

🏷️ Etiquetas semánticas en acción

<header>

header

Cabecera del sitio. Contiene la marca, navegación y elementos introductorios.

📈 Ayuda al crawler a identificar la zona de marca y el contenido principal de la cabecera.
<nav>

nav

Menú de navegación principal o secundaria del sitio.

🔗 Los crawlers descubren enlaces importantes y la estructura de navegación.
<main>

main

Contenido principal y único de la página. Solo debe haber uno por documento.

⭐ Google prioriza el texto dentro de main como el contenido más relevante.
<article>

article

Contenido independiente y autónomo como noticias, posts o entradas de blog.

📰 Ideal para contenido que puede distribuirse o sindicarse de forma independiente.
<section>

section

Agrupa contenido temático relacionado dentro de un documento.

📚 Organiza el documento en capítulos lógicos que los crawlers pueden interpretar.
<aside> <footer>

aside / footer

Contenido complementario (aside) y pie de página (footer).

🧩 Clarifica contenido secundario y proporciona contexto adicional a los buscadores.

📚 Resumen de la clase:

  • HTML Semántico: Etiquetas que describen su propósito (header, main, article, etc.)
  • No semántico: div y span son contenedores genéricos sin significado.
  • Beneficios: Código más legible, mejor accesibilidad y mejor SEO.
  • SEO: Posicionamiento en buscadores. Crawler: Robot que lee tu web.
  • Clase CSS (.): Reutilizable, se aplica con class="nombre".
  • ID CSS (#): Único por página, más específico, se aplica con id="nombre".
  • Especificidad: ID (100) > Clase (10) > Etiqueta (1).
