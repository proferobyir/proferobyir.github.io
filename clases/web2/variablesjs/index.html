📦 1. Variables y Tipos de Datos

Los programas necesitan guardar datos. Usamos let para cajas reutilizables cuyo valor puede variar, y const para valores que deben quedar fijos y protegidos.

let nombre = "Juan"; // Tipo: String (Texto)
const edad = 28;     // Tipo: Number (Número) - Inmutable

🔍 2. Scope de Bloque (Alcance)

El scope define los límites donde vive una variable. Observa cómo los "paréntesis" de color delimitan dónde empieza y dónde termina cada entorno:

let global = "Disponible en todo el archivo";

if (true) {
let local = "Solo existo aquí adentro";
console.log(local); // ✅ Éxito
}
console.log(local); // ❌ ¡ERROR! Fuera del paréntesis rosa la variable local ha muerto.

🚦 3. Sentencias Condicionales (Flujos Excluyentes)

Las condicionales dividen los caminos de tu código. Al ejecutarse, el navegador **solo entrará a un único paréntesis de color** (el primero que se cumpla):

let semaforo = "Amarillo";

if (semaforo === "Rojo") {
console.log("🛑 ¡Detenerse por completo!");
} else if (semaforo === "Amarillo") {
console.log("⚠️ ¡Precaución! El paréntesis del medio se activa.");
} else {
console.log("🍏 ¡Avanzar de forma segura!");
}

🔁 4. Ciclos (Loops)

El bucle for repite todo el código que se encuentra atrapado dentro de su paréntesis cian tantas veces como lo dicte su condición de parada:

for (let i = 1; i <= 3; i++) {
// Todo lo que está dentro de este paréntesis curvo se repetirá 3 veces
console.log("Vuelta número: " + i);
}

🛠️ 5. Funciones Tradicionales y Funciones Flecha

Bloques de código reutilizables estructurados de forma clásica o mediante la sintaxis moderna:

Sintaxis Tradicional:

function calcularDoble(numero) {
    return numero * 2;
}

Sintaxis Flecha (Arrow Function):

const calcularDobleFlecha = (numero) => {
    return numero * 2;
};

🔌 6. Buenas Prácticas: ¿Dónde incluir JavaScript?

JavaScript se puede incrustar en múltiples partes de tu documento HTML utilizando las etiquetas <script>:

A. Scripts Internos (Directamente en el cuerpo del HTML)

La buena práctica dicta agruparlos preferiblemente al final del <body> para asegurar que el contenido visual ya exista.

<body>
    <h1>Mi Sitio</h1>

    <script>
        let nombre = "Juan"; // Código embebido directamente
        console.log(nombre);
    </script>
</body>

B. Scripts Externos (En el Head mediante Defer y Async)

Para proyectos reales, separamos el código en archivos `.js` externos optimizando el rendimiento desde el head:

<!-- RECOMENDADO: Descarga asíncrona y ejecución automática al final de la carga del HTML -->
<script src="app.js" defer></script>

<!-- Descarga asíncrona pero interrumpe al HTML para ejecutarse de inmediato -->
<script src="analiticas.js" async></script>
