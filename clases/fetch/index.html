Aprendiendo Async/Await, .map() y CSS Grid para organizar Pokémon en 2 columnas

📐 ¿Cómo organizamos los Pokémon?

Usamos CSS Grid con grid-template-columns: 1fr 1fr para que las tarjetas se muestren SIEMPRE en 2 columnas (excepto en móvil). El método .map() solo genera el HTML, el Grid se encarga del layout.

/* CSS: Contenedor con Grid de 2 columnas */ .pokemon-grid-2cols { display: grid; grid-template-columns: 1fr 1fr; /* 🔥 2 columnas exactas */ gap: 1rem; } /* JavaScript: .map() genera las tarjetas */ const tarjetasHTML = pokemones.map(pokemon => { return `
...
`; }).join(''); // join() solo concatena strings // El contenedor tiene la clase .pokemon-grid-2cols contenedor.innerHTML = `
${tarjetasHTML}
`;
💡 Ventaja: El CSS Grid controla el diseño (2 columnas, responsivo). .map() + join() solo generan el contenido. ¡Separación de responsabilidades!

📁 1. .map() + Grid (Datos locales)

Transformamos un arreglo JSON de 5 Pokémon en una cuadrícula de 2 columnas usando CSS Grid.

// Datos locales (5 Pokémon) const pokemonLocal = [ { id: 25, nombre: "Pikachu", tipo: "eléctrico", imagen: "https://raw.githubusercontent.com/.../25.png" }, { id: 4, nombre: "Charmander", tipo: "fuego", imagen: "https://raw.githubusercontent.com/.../4.png" } // ... 3 más ]; // 🔥 .map() genera las tarjetas const tarjetas = pokemonLocal.map(p => ` <div class="pokemon-card"> <img src="${p.imagen}"> <strong>${p.nombre}</strong> <small>${p.tipo}</small> </div> `).join(''); // 🎨 CSS Grid organiza en 2 columnas contenedor.innerHTML = ` <div class="pokemon-grid-2cols"> ${tarjetas} </div> `;
⚡ Haz clic para ver cómo .map() + Grid 1fr 1fr organizan los Pokémon.

🌐 2. PokéAPI + map() + Grid 2 columnas

Consumimos la PokéAPI, transformamos con .map() y mostramos en 2 columnas.

// Arreglo de nombres const nombres = ["pikachu", "charizard", "mewtwo", "eevee", "snorlax", "bulbasaur"]; // .map() para crear promesas const promesas = nombres.map(n => fetch(`https://pokeapi.co/api/v2/pokemon/${n}`).then(r => r.json()) ); const resultados = await Promise.all(promesas); // .map() para generar HTML (sin preocuparse del layout) const tarjetas = resultados.map(p => ` <div class="pokemon-card"> <img src="${p.sprites.front_default}"> <strong>${p.name}</strong> </div> `).join(''); // El CSS Grid aplica automáticamente 2 columnas contenedor.innerHTML = `
${tarjetas}
´;
🌟 .map() genera el HTML, CSS Grid organiza en 2 columnas.

🔄 3. Grid 1fr 1fr vs Flexbox

Con Grid forzamos EXACTAMENTE 2 columnas. Con Flexbox las tarjetas se ajustan según el ancho.

/* 🔥 CSS Grid: 2 columnas exactas */ .grid-2cols { display: grid; grid-template-columns: 1fr 1fr; /* Siempre 2 columnas */ gap: 1rem; } /* 🔄 Flexbox: se ajusta automáticamente */ .flex-wrap { display: flex; flex-wrap: wrap; gap: 1rem; } .flex-wrap .pokemon-card { flex: 1 1 200px; /* Ancho variable */ }
💡 Muestra la diferencia entre Grid (2 columnas fijas) y Flexbox (responsive fluido).

📱 4. Grid Responsivo con Media Query

Usamos media query para que en móvil las tarjetas se muestren en 1 columna y en desktop en 2 columnas.

/* CSS responsivo */ .pokemon-grid-2cols { display: grid; grid-template-columns: 1fr 1fr; /* Desktop: 2 columnas */ gap: 1rem; } @media (max-width: 640px) { .pokemon-grid-2cols { grid-template-columns: 1fr; /* Móvil: 1 columna */ } } // El mismo .map() funciona para ambos casos const tarjetas = pokemones.map(p => `
...
`).join('');
🖥️ Redimensiona tu navegador: en escritorio verás 2 columnas, en móvil 1 columna.

🎨 5. Grid avanzado: tarjetas con imágenes más grandes

Usamos Grid de 2 columnas pero con tarjetas más vistosas (imagen de 100px).

// Estilo mejorado para las tarjetas .pokemon-card-grande { background: linear-gradient(135deg, #fffbeb, #fef3c7); border-radius: 1.5rem; padding: 1.2rem; text-align: center; transition: transform 0.2s; } .pokemon-card-grande img { width: 100px; height: 100px; } // .map() genera el HTML const tarjetas = pokemones.map(p => ` <div class="pokemon-card-grande"> <img src="${p.imagen}"> <h3>${p.nombre}</h3> <span class="badge">#${p.id}</span> </div> `).join('');
🎯 Grid 1fr 1fr con tarjetas más grandes y detalles adicionales.
