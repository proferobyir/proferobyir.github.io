🏗️ Estructura Básica de una Página Web

El esqueleto de toda página HTML Estructura

Toda página web sigue esta estructura fundamental. Cada parte tiene un propósito específico.

▶ Vista previa de la estructura
<!DOCTYPE html>
<head>
🧠 Metadatos
• <meta charset="UTF-8">
• <title>Mi página</title>
• <link rel="stylesheet" href="style.css">
• <link rel="icon" href="favicon.ico">
</head>
<body>
👁️ Contenido visible
• <header> ... </header>
• <nav> ... </nav>
• <main> ... </main>
• <footer> ... </footer>
</body>
</html>
📄 <!DOCTYPE html>

Declara que es HTML5 (siempre al inicio).

🧠 <head>

Contiene metadatos (info sobre la página): título, codificación, enlaces a CSS, favicon, etc. No se ve en el navegador.

👁️ <body>

Contiene todo el contenido visible: textos, imágenes, formularios, etc. Es lo que ve el usuario.

<!DOCTYPE html> <html lang="es"> <head> <meta charset="UTF-8" /> <meta name="viewport" content="width=device-width, initial-scale=1.0" /> <title>Mi página web</title> <link rel="stylesheet" href="estilos.css" /> <link rel="icon" href="favicon.ico" /> </head> <body> <!-- Todo el contenido visible va aquí --> <h1>Bienvenidos</h1> <p>Este es el contenido de mi página.</p> </body> </html>

📦 Box Model (Modelo de Caja)

¿Cómo se compone cada elemento HTML? Box Model

Todos los elementos HTML son cajas. El Box Model define cómo se calcula el espacio que ocupa cada elemento, compuesto por 4 capas:

▶ Diagrama del Box Model
Contenido
Margin (exterior) Border (borde) Padding (interno) Content (contenido)
Margin (Margen)

Espacio externo entre el elemento y los demás. Separa cajas.

Border (Borde)

Línea que rodea el padding y el contenido. Puede tener color, grosor y estilo.

Padding (Relleno)

Espacio interno entre el borde y el contenido. Da "aire" al contenido.

Content (Contenido)

Es el texto, imagen u otro elemento dentro de la caja. El corazón del elemento.

✏️ Ejemplo práctico: Párrafo con Margin y Padding

🔴 Sin estilos (por defecto)

Sin margin ni padding adicionales

Este párrafo no tiene margin ni padding personalizados. Solo ocupa el espacio necesario.

Margin: 0px (por defecto)
Padding: 0px (por defecto)
Border: 2px sólido

🟢 Con Margin y Padding

margin: 30px 20px · padding: 25px 20px

✨ Este párrafo tiene margin y padding
El margin (rojo) separa este párrafo de los elementos exteriores.
El padding (azul) da espacio interno entre el borde y el texto.

Margin: 30px arriba/abajo · 20px izquierda/derecha
Padding: 25px arriba/abajo · 20px izquierda/derecha
Border: 3px sólido azul
💡 ¿Ves la diferencia? El párrafo con margin tiene más separación con los bordes de la tarjeta, y el padding hace que el texto no esté pegado al borde azul.
/* Párrafo CON margin y padding */ .parrafo-con-estilos { background: #e8f4fd; border: 3px solid #3498DB; border-radius: 8px; margin: 30px 20px 30px 20px; padding: 25px 20px 25px 20px; }

📏 Unidades de Medida en CSS

Absolutas vs Relativas Unidades

En CSS existen diferentes unidades para definir tamaños. Las absolutas son fijas, mientras que las relativas se adaptan a otros elementos o al viewport.

📌 Absolutas Fijas

No cambian. Útiles para tamaños exactos que no deben variar.

px Píxeles. Unidad más común. 16px
pt Puntos. Usado en impresión. 12pt
cm Centímetros. Para impresión. 2cm
mm Milímetros. Para impresión. 5mm
in Pulgadas. 1in = 2.54cm. 1in
pc Picas. 1pc = 12pt. 2pc

🔄 Relativas Adaptables

Se adaptan a otros elementos. ¡Las más recomendadas para responsive!

% Porcentaje del elemento padre. 50%
em Relativo al tamaño de fuente del elemento padre. 2em
rem Relativo al tamaño de fuente raíz (<html>). 1.5rem
vw 1% del ancho del viewport (pantalla). 50vw
vh 1% del alto del viewport (pantalla). 50vh
vmin 1% del lado más pequeño del viewport. 10vmin
vmax 1% del lado más grande del viewport. 10vmax

👁️ Demostración visual de unidades

▶ Comparativa en tiempo real
px (fijo)
100px
Siempre mide 100px
% (relativo)
50%
Mitad del contenedor
rem (raíz)
10rem
10 × 16px = 160px
em (padre)
20em
20 × tamaño fuente padre
vw (viewport)
30vw
30% del ancho de pantalla
vh (viewport)
10vh
10% del alto de pantalla
🎯 ¿Cuál usar?
  • px → Para bordes, sombras y elementos que necesitan tamaño exacto.
  • rem → ¡La mejor opción! Para fuentes, márgenes y paddings. Escala con la configuración del usuario.
  • % → Para layouts fluidos (anchos de columnas, imágenes).
  • vw/vh → Para elementos que deben ocupar un porcentaje de la pantalla (hero sections, modales).
  • em → Útil para componentes que deben escalar juntos (cards, botones).
/* Ejemplos de uso de unidades */ .caja-px { width: 100px; border: 2px solid #333; } .caja-rem { width: 10rem; padding: 1.5rem; } .caja-porcentaje { width: 50%; } .caja-vw { width: 30vw; } html { font-size: 16px; }

🔤 Fuentes Web: WOFF / WOFF2

@font-face y Google Fonts Fuentes

Las fuentes web permiten usar tipografías personalizadas en tu sitio. Los formatos WOFF y WOFF2 son los más modernos y eficientes.

📁 @font-face WOFF/WOFF2

Carga archivos de fuentes locales. Ideal para fuentes propias o comerciales.

Esta es mi fuente personalizada MiFuentePersonalizada (WOFF2)
Versión en negrita MiFuentePersonalizada Bold (WOFF2)
📂 Estructura de carpetas:
/css/
/fonts/
  ├── mi-fuente.woff2
  ├── mi-fuente.woff
  ├── mi-fuente-bold.woff2
  └── mi-fuente-bold.woff

🌐 Google Fonts CDN

Fuentes gratuitas y optimizadas. Solo necesitas un enlace en el <head>.

Open Sans - Google Fonts Open Sans (400 / 700)
Open Sans Light Open Sans (300)
💡 Ventajas: No necesitas descargar archivos, actualizaciones automáticas, gran variedad de fuentes.

👁️ Comparativa de fuentes

▶ Diferentes tipos de fuentes
🔹 Mi fuente personalizada (WOFF2) @font-face
🔹 Open Sans (Google Fonts) Google Fonts
🔹 Fuente del sistema (sans-serif) system-ui
🔹 Fuente Serif (Georgia) Georgia
🔹 Fuente Monoespaciada Courier New

📝 Código completo

Opción 1: @font-face (fuentes locales)
/* 1. DEFINIR LA FUENTE CON @FONT-FACE */ @font-face { font-family: 'MiFuentePersonalizada'; src: url('../fonts/mi-fuente.woff2') format('woff2'), url('../fonts/mi-fuente.woff') format('woff'); font-weight: normal; font-style: normal; font-display: swap; } /* Versión en negrita */ @font-face { font-family: 'MiFuentePersonalizada'; src: url('../fonts/mi-fuente-bold.woff2') format('woff2'), url('../fonts/mi-fuente-bold.woff') format('woff'); font-weight: bold; font-style: normal; font-display: swap; }
Opción 2: Google Fonts (CDN)
<!-- En el <head> del HTML --> <link rel="preconnect" href="https://fonts.googleapis.com" /> <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin /> <link href="https://fonts.googleapis.com/css2?family=Open+Sans:wght@300;400;700&display=swap" rel="stylesheet" /> /* En CSS */ body { font-family: 'Open Sans', sans-serif; }
🎯 Recomendaciones:
  • WOFF2 → ¡El mejor formato! Mayor compresión y rendimiento.
  • WOFF → Buena compatibilidad con navegadores antiguos.
  • font-display: swap → Muestra una fuente de respaldo mientras carga la personalizada.
  • Google Fonts → Ideal para proyectos rápidos y sin necesidad de hosting de archivos.
  • @font-face → Perfecto para fuentes propias, comerciales o con licencias específicas.
🔧 Herramientas útiles:
  • Google Fonts → Catálogo de fuentes gratuitas
  • Transfonter → Convierte tus fuentes a WOFF/WOFF2
  • Font Squirrel Generator → Genera todos los formatos necesarios

🎯 Selectores CSS

Cómo seleccionar elementos para aplicar estilos Selectores

Los selectores son el puente entre HTML y CSS. Permiten elegir qué elementos van a recibir los estilos.

🏷️ Selector de tipo Etiqueta

Selecciona todas las etiquetas de un tipo específico.

p { color: blue; }
Aplica a todos los <p>

📌 Selector de clase .clase

Selecciona elementos con la clase especificada. Reutilizable.

.destacado { background: yellow; }
<div class="destacado"> ... </div>

🆔 Selector de ID #id

Selecciona un elemento único con el ID especificado.

#header { font-size: 2rem; }
<header id="header"> ... </header>

📂 Selector descendente espacio

Selecciona elementos dentro de otros (cualquier nivel).

.menu a { color: red; }
Todos los <a> dentro de .menu

➡️ Hijo directo >

Selecciona solo los hijos directos (primer nivel).

.contenido > p { margin: 0; }
Solo <p> hijos directos de .contenido

➕ Adyacente +

Selecciona el elemento que sigue inmediatamente a otro.

h2 + p { font-size: 1.2rem; }
El <p> que sigue a un <h2>

🔗 Selector de atributo [attr]

Selecciona elementos por sus atributos.

[type="text"] { border: 1px solid #ccc; }
<input type="text">

🎭 Pseudo-clases :hover

Seleccionan elementos en estados específicos.

a:hover { color: red; }
:hover, :focus, :active, :first-child, :last-child, :nth-child()

✨ Pseudo-elementos ::before

Añaden contenido antes o después del elemento.

.destacado::before { content: "⭐ "; }
::before, ::after, ::first-line, ::first-letter

🌟 Selector universal *

Selecciona todos los elementos.

* { box-sizing: border-box; }
Útil para resetear estilos globales

⭐ Favicon: El ícono de tu página

El pequeño ícono que identifica tu sitio Favicon

El favicon es el ícono que aparece en la pestaña del navegador, en favoritos y en la barra de direcciones. Mejora el branding y la experiencia del usuario.

▶ ¿Dónde se ve el favicon?
🌐
Pestaña del navegador Aparece junto al título
⭐
Favoritos / Marcadores Ícono en la lista de favoritos
📱
Apple Touch Icon Ícono para iOS (pantalla de inicio)
🔍
Resultados de búsqueda Aparece en buscadores como Google
📁 Formatos soportados
  • .ico → Tradicional, compatible con todos
  • .png → Moderno, buena calidad
  • .svg → Vectorial, se escala perfectamente
  • .jpg → No recomendado (pérdida de calidad)
📐 Tamaños recomendados
  • 16×16 → Barra de direcciones
  • 32×32 → Pestaña del navegador
  • 180×180 → Apple Touch Icon
  • 512×512 → PWA (Progressive Web App)

📝 Código para agregar favicon

<!-- Favicon básico (formato ICO) --> <link rel="icon" type="image/x-icon" href="favicon.ico" /> <!-- Favicon PNG (moderno) --> <link rel="icon" type="image/png" sizes="32x32" href="favicon-32x32.png" /> <link rel="icon" type="image/png" sizes="16x16" href="favicon-16x16.png" /> <!-- Apple Touch Icon (iOS) --> <link rel="apple-touch-icon" sizes="180x180" href="apple-touch-icon.png" /> <!-- Manifest para PWA (opcional) --> <link rel="manifest" href="site.webmanifest" /> <!-- SVG (moderno, escala perfecto) --> <link rel="icon" type="image/svg+xml" href="favicon.svg" />
🎯 Recomendaciones para favicon:
  • Usa PNG o SVG → Mejor calidad que ICO en pantallas modernas.
  • Proporciona múltiples tamaños → Para diferentes dispositivos y contextos.
  • Diseño simple y reconocible → Se ve pequeño, usa pocos detalles.
  • Fondo transparente → Se adapta a temas claros y oscuros del navegador.
  • Herramienta: favicon.io o RealFaviconGenerator
📂 Estructura recomendada:
/assets/
  /favicon/
    ├── favicon.ico
    ├── favicon-16x16.png
    ├── favicon-32x32.png
    ├── favicon.svg
    ├── apple-touch-icon.png
    └── site.webmanifest

🎨 3 Formas de usar CSS

En línea · Interno · Externo CSS

Elige la forma según el alcance que necesites: específico, página única o global.

🔹 En Línea

Atributo style

Aplicado directamente en la etiqueta.

<p style="color: blue; font-size: 20px;" >Texto</p>

🔹 Interno

Etiqueta <style>

Dentro del <head>, para una sola página.

<head> <style> h1 { color: green; } </style> </head>

🔹 Externo

Archivo .css

Vinculado con <link>. ¡La mejor práctica!

<head> <link rel="stylesheet" href="estilos.css" /> </head>

🧩 Variables en CSS

Colores y sombras con var(--) Custom Properties

Se definen con --nombre y se usan con var(--nombre). Cambia una vez y actualiza todo.

▶ Vista previa: variables en acción
Color primario
--color-secundario
Sombra suave
--sombra-suave
Sombra fuerte
--sombra-media
/* Definición */ :root { --color-primario: #2C3E50; --color-secundario: #3498DB; --sombra-suave: 0 4px 6px rgba(0,0,0,0.1); --sombra-media: 0 8px 16px rgba(0,0,0,0.15); } /* Uso */ .boton { background: var(--color-secundario); box-shadow: var(--sombra-media); }

💡 Ventaja: Si cambias --color-secundario, todos los elementos que lo usan se actualizan automáticamente.
