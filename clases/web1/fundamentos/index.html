Estructura Cliente-Servidor

Para entender la web, debemos entender que no es un sistema aislado. Se basa en una conversación constante.

Explicación: El cliente (tu navegador) pide permiso o archivos al servidor (una computadora remota), y este responde enviando el sitio web.

Principios de Diseño Gráfico

Un sitio web no solo debe funcionar, debe comunicar de forma efectiva a través de:

  • Tipografía: Elegir letras legibles y con personalidad.
  • Composición: El arte de organizar elementos en el espacio.
  • Jerarquía Visual: Decidir qué es lo más importante para que el ojo lo vea primero.

Armonías del Color

Usamos la rueda del color para encontrar combinaciones que no cansen la vista y se vean profesionales.

Complementaria

Colores opuestos (Máximo contraste).

Análoga

Colores vecinos (Armonía suave).

Tríada

Tres colores equidistantes (Equilibrio).

Monocromática

Un solo tono, distintos brillos.

Estructura vs Estilo

En el desarrollo moderno usamos:

HTML5

Es el esqueleto. Define qué es un título, qué es un párrafo y qué es una imagen.

CSS3

Es la apariencia. Define colores, tamaños y posiciones usando Flexbox y Grid.

Experiencia de Usuario (UX)

La UX es el proceso de diseñar productos que sean útiles y fáciles de usar. La Jerarquía de Contenidos es vital para esto.

Dato Didáctico: Un usuario decide si quedarse en un sitio en menos de 3 segundos. Por eso, lo más importante debe estar arriba y ser visualmente llamativo.

Publicación y Estándares

Para que el mundo vea tu trabajo, debes subirlo a un servidor (como GitHub Pages).

Accesibilidad Web (A11y)

Las WCAG (Web Content Accessibility Guidelines) son las Pautas de Accesibilidad para el Contenido Web. Desarrolladas por el World Wide Web Consortium (W3C), son el estándar internacional para garantizar que los sitios web y las aplicaciones sean accesibles para personas con discapacidades.

💡 Nota para el estudiante: ¿Por qué se abrevia "a11y"?

En el mundo del software es muy común usar numerónimos para acortar palabras extremadamente largas. Observa cómo funciona:

A 11 y

Se toma la letra A inicial, se cuentan las 11 letras intermedias de la palabra en inglés "accessibility" y se coloca la última letra y. ¡Es así de sencillo!

Esto incluye prácticas obligatorias de desarrollo como:

  • Poner texto alternativo a las imágenes (`alt="..."`).
  • Asegurar que los colores tengan suficiente contraste.
  • Que el sitio se pueda navegar por completo usando solo el teclado.
