Introducción al desarrollo web: estructura cliente-servidor

El desarrollo web se fundamenta en el modelo Cliente-Servidor. La interacción en la web ocurre mediante peticiones (Requests) enviadas por el cliente y respuestas (Responses) devueltas por el servidor a través del protocolo HTTP/HTTPS.

  • Cliente: Es el dispositivo del usuario (computadora, smartphone) ejecutando un navegador web (Chrome, Firefox, Safari).
  • Servidor: Computadora remota optimizada que almacena los archivos del sitio web y responde a las solicitudes globales.

Principios de diseño gráfico y Armonías del Color

La estética web no es accidental, responde a pilares fundamentales que dictan cómo los humanos procesamos la información visual:

Tipografía y Color

La tipografía define la legibilidad. El color evoca emociones y establece el contraste necesario.

Composición y Jerarquía

Organiza los elementos en el espacio de la pantalla e indica al ojo qué mirar primero.

Círculos Cromáticos y Tipos de Armonía de Color

Cada tipo de armonía crea un esquema visualmente atractivo utilizando el círculo cromático como base, donde una figura geométrica delineada une los colores elegidos.

1. Complementaria

Combina dos colores opuestos en el círculo. Ofrece el máximo contraste y vibración, ideal para destacar elementos de acción (CTAs).

2. Análoga

Utiliza tres colores adyacentes en el círculo. Crea esquemas relajantes, serenos y muy coherentes, naturales a la vista.

3. Tríada

Esquema de tres colores equidistantes, formando un triángulo equilátero. Ofrece gran vibración y equilibrio cromático.

4. Monocromática

Utiliza variaciones de saturación y brillo de un solo tono de color. Muy elegante, limpia y fácil de aplicar en la web.

Uso de HTML y CSS

La separación de responsabilidades es clave en la web: HTML dota de significado y estructura al documento, mientras que CSS define las capas de estilo y presentación visual.

Diseño Visual, UX y Jerarquía de Contenidos

La Experiencia de Usuario (UX) analiza el comportamiento, las necesidades y las emociones del usuario interactuando con la interfaz. Diseñar interfaces eficaces requiere planificar patrones de escaneo visual fluidos.

Publicación de sitios web en servidores

Una vez completado el diseño del sitio localmente, la web debe transferirse a un entorno de producción accesible a través de Internet.

Servidores Locales

Herramientas en tu computadora (ej. XAMPP, Live Server) que procesan código simulando un servidor, exclusivo para pruebas.

Servidores Gratuitos y Hosting Cloud

Plataformas modernas (ej. GitHub Pages, Netlify) que permiten subir el código estático directamente desde la nube.

Accesibilidad en diseño web

La accesibilidad web implica construir sitios inclusivos que puedan ser utilizados por cualquier persona, independientemente de sus capacidades físicas, cognitivas o técnicas.

¿Cómo medimos la accesibilidad? Para garantizar esto, los desarrolladores nos guiamos por las WCAG (Web Content Accessibility Guidelines), que se traducen como las Pautas de Accesibilidad para el Contenido Web. Desarrolladas por el World Wide Web Consortium (W3C), funcionan como el estándar internacional oficial para asegurar que los sitios web y las aplicaciones sean plenamente accesibles para personas con discapacidades.

Requisitos Básicos de Accesibilidad:

  • Uso correcto de etiquetas semánticas estructurales.
  • Contraste óptimo de colores entre texto y fondos.
  • Navegación e interacción íntegra a través del teclado.
