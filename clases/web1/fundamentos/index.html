Introducción al desarrollo web: estructura cliente-servidor

El desarrollo web se fundamenta en el modelo Cliente-Servidor. La interacción en la web ocurre mediante peticiones (Requests) enviadas por el cliente y respuestas (Responses) devueltas por el servidor a través del protocolo HTTP/HTTPS.

  • Cliente: Es el dispositivo del usuario (computadora, smartphone) ejecutando un navegador web (Chrome, Firefox, Safari).
  • Servidor: Computadora remota optimizada que almacena los archivos del sitio web y responde a las solicitudes globales.

Principios de diseño gráfico en la web

La estética web no es accidental, responde a cuatro pilares fundamentales que dictan cómo los humanos procesamos la información visual:

Tipografía

Define la legibilidad y la identidad visual de la página. El uso de fuentes Sans-Serif (como Arial, Segoe UI o Helvetica) es preferido en entornos digitales por su lectura fluida en pantallas.

Composición y Jerarquía

La composición organiza los elementos en el espacio dinámico de la pantalla. La jerarquía le indica al ojo del usuario qué mirar primero reduciendo la carga de lectura.

Teoría del Color: Rueda y Armonías

El color es una de las herramientas de comunicación más poderosas en el diseño web. No solo define la estética, sino que influye en las emociones del usuario y en la usabilidad del sitio.

La Rueda del Color (Círculo Cromático)

Es una representación organizada y circular de los colores. Se compone de 12 tonos clave distribuidos en círculo, que nos ayudan a entender cómo se relacionan los colores entre sí.

Tipos de Armonías de Color

Las armonías son fórmulas o reglas de diseño que utilizamos para combinar colores de manera agradable y equilibrada. A continuación se presentan las cuatro armonías fundamentales aplicadas en la web:

1. Armonía Complementaria

Consiste en elegir dos colores que están en extremos opuestos del círculo cromático (como el Azul y el Naranja). Ofrece un contraste muy alto y dinámico, ideal para destacar botones de acción importantes.

2. Armonía Análoga

Se crean combinando colores que están uno al lado del otro en el círculo cromático (como el Amarillo, el Amarillo-Verde y el Verde). Ofrece una sensación muy natural, suave y relajante a la vista.

3. Armonía de Tríada

Utiliza tres colores equidistantes entre sí en la rueda, formando un triángulo equilátero perfecto (como el Rojo, el Amarillo y el Azul). Es una combinación muy vibrante y llena de vida.

4. Armonía Monocromática

Usa un solo color de base (por ejemplo, el Azul) y juega únicamente con sus variaciones de brillo, saturación y luminosidad. Es la combinación más segura, limpia y elegante para interfaces corporativas.

Uso de HTML y CSS

La separación de responsabilidades es clave en la web: HTML dota de significado y estructura al documento, mientras que CSS define las capas de estilo y presentación visual de la interfaz.

Diseño Visual, UX y Jerarquía de Contenidos

La Experiencia de Usuario (UX) analiza el comportamiento, las necesidades y las emociones del usuario interactuando con la interfaz. Diseñar interfaces eficaces requiere planificar patrones de escaneo visual fluidos.

Publicación de sitios web en servidores

Una vez completado el diseño del sitio localmente, la web debe transferirse a un entorno de producción accesible a través de Internet.

Servidores Locales

Herramientas en tu computadora (ej. Live Server en VS Code o ambientes como XAMPP) que simulan un servidor en tu PC. Es el espacio ideal de pruebas privadas.

Servidores Gratuitos (Hosting Cloud)

Plataformas que nos permiten alojar gratis y publicar nuestro código estático en la nube para que cualquiera acceda (ej. GitHub Pages, Netlify o Vercel).

Accesibilidad en diseño web

La accesibilidad web implica construir sitios inclusivos que puedan ser utilizados por cualquier persona, independientemente de sus capacidades físicas, cognitivas o técnicas.

¿Cómo medimos la accesibilidad? Para garantizar esto, los desarrolladores nos guiamos por las WCAG (Web Content Accessibility Guidelines), que se traducen como las Pautas de Accesibilidad para el Contenido Web. Desarrolladas por el World Wide Web Consortium (W3C), funcionan como el estándar internacional oficial para asegurar que los sitios web y las aplicaciones sean plenamente accesibles para personas con discapacidades.

Requisitos Básicos de Accesibilidad:

  • Uso correcto de etiquetas semánticas estructurales (`<main>`, `<nav>`, `<article>`).
  • Contraste óptimo de colores entre texto y fondos para facilitar la lectura.
  • Navegación e interacción íntegra a través del teclado (sin depender del ratón).
