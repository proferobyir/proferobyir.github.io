📚 Anatomía de funciones: .map() , fetch() y Promesas

🔁 .map() → Transforma arrays Método de array

// .map() recorre CADA elemento y devuelve NUEVO array
const original = [1, 2, 3];
const doble = original.map( (numero) => numero * 2 );
// Resultado: [2, 4, 6]
┌─────────────────────────────────────────────────────────┐
│ arrayOriginal ──.map( callback )──► nuevoArray │
│ │
│ callback recibe: (elemento, índice, array) │
│ → SIEMPRE retorna el elemento transformado │
│ → El array resultante tiene la MISMA longitud │
└─────────────────────────────────────────────────────────┘
✨ Uso práctico: Convertir datos en HTML, modificar objetos, formatear texto.

🌐 fetch() + Promesas Asincronía

// fetch() devuelve una PROMESA (estado pendiente inicialmente)
fetch('https://api.ejemplo.com/datos')
  .then( respuesta => respuesta.json() )  // transforma a JSON
  .then( datos => console.log(datos) )  // trabajas con datos
  .catch( error => manejarError(error) );  // captura errores
┌──────────┐ ┌──────────────┐ ┌─────────────────┐
│ fetch() │ ──► │ Promesa │ ──► │ Respuesta │
│ petición │ │ (pendiente) │ │ (resuelta/OK) │
└──────────┘ └──────────────┘ └─────────────────┘
│ │ │
▼ ▼ ▼
[ .then() ] [ .catch() ] [ .finally() ]
(encadenar) (errores) (siempre ejecuta)
✅ Ventaja: Las promesas evitan el "callback hell" y permiten manejar operaciones asíncronas de forma legible.

⚡ map() + Promise.all (múltiples peticiones) Paralelismo

// 1. Arreglo de nombres o URLs
const nombres = ['pikachu', 'charizard', 'bulbasaur'];
// 2. .map() crea un array de promesas
const promesas = nombres.map( nombre =>
  fetch(`https://pokeapi.co/api/v2/pokemon/${nombre}`).then(res => res.json())
);
// 3. Promise.all espera a que TODAS terminen
const resultados = await Promise.all(promesas);
// 4. resultados contiene los datos de TODAS las peticiones
[ url1, url2, url3 ]
│
▼
.map( fetch ) → [ promesa1, promesa2, promesa3 ]
│
▼
Promise.all( [promesa1, promesa2, promesa3] )
│
▼
[ data1, data2, data3 ] ← ¡Todas en paralelo, sin esperar secuencialmente!
🚀 Ideal para: Cargar múltiples recursos simultáneamente (Pokémon, usuarios, productos) y mejorar el rendimiento.
💡 Resumen visual: .map() transforma arrays → fetch() lanza peticiones asíncronas → Promesas manejan resultados → ¡Grid organiza sin floats!

📁 1. .map() + Grid (local)

Transformamos datos locales en HTML con .map() y CSS Grid (2 columnas fijas).

// Datos locales
const pokemones = [ {id:25, nombre:"Pikachu"}, ... ];
// .map() genera tarjetas HTML
const tarjetas = pokemones.map(p => `<div class="pokemon-card">...</div>`).join('');
// Grid 1fr 1fr aplica 2 columnas automáticas

🔄 3. Grid 1fr 1fr vs Flexbox

Comparativa: Grid fuerza 2 columnas exactas, Flex se adapta al ancho.

.grid-2cols { display: grid; grid-template-columns: 1fr 1fr; }
.flex-wrap { display: flex; flex-wrap: wrap; gap:1rem; }
Comparativa clara sin floats.
