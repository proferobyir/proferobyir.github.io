Guía Interactiva: ECMAScript

Preparación clave usando la API de JSONPlaceholder

Bienvenido al entorno interactivo de nivelación. Lee los conceptos de cada módulo y haz clic en "Cargar en el Editor" para probar y modificar el código directamente en tiempo real.

Módulo 1: Ámbito de variables con Let y Const

Para declarar las variables no usaremos var. En Node.js, para evitar mutaciones accidentales en peticiones concurrentes, usamos un entorno controlado:

  • const: Para valores que no cambiarán de identificador.
  • let: Para variables limitadas al bloque de código donde se declaran.

Módulo 2: Desestructuración (Destructuring)

Es la herramienta más usada en Node.js para extraer propiedades de un objeto de forma directa. Muy Importante cuando manejamos respuestas HTTP (req.body o req.query).

Módulo 3: Plantillas Literales e Inmutabilidad

Concatenar cadenas con el operador + funciona pero es propenso a errores; en su lugar usamos las comillas invertidas (`) para interpolar de manera legible.

⚠️ El peligro de la Referencia en JavaScript:
A diferencia de PHP, si en JavaScript haces const copia = objetoOriginal, no estás duplicando el objeto. Ambos identificadores apuntan al mismo espacio físico de memoria. Si modificas copia, destruirás el original de forma colateral.

Para lograr la Inmutabilidad (crear un objeto nuevo basado en uno viejo sin alterar el original), usamos el operador Spread (...). Esto es vital en Node.js para mantener la integridad de los datos entre peticiones HTTP.

Módulo 4: Consumo de Promesas (fetch)

Una Promesa representa un proceso asíncrono. En lugar de simularlo, usaremos fetch() para consultar usuarios reales en internet y procesar la respuesta usando .then().

Módulo 5: Async / Await moderno

Es la forma estándar de la industria para resolver Promesas de manera lineal. Estructuramos el flujo con await y controlamos los posibles errores de red usando bloques try/catch.
