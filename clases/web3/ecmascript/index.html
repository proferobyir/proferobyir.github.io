Modulo 1: Ambito de variables con Let y Const

En JavaScript moderno, usamos const y let en lugar de var para tener mayor control sobre nuestras variables.

  • const: Para valores que no van a cambiar. Ejemplo: const PI = 3.1416;
  • let: Para variables que pueden cambiar dentro de un bloque especifico.

✅ Ventaja: Esto evita errores accidentales cuando trabajamos con peticiones en Node.js.

Modulo 2: Desestructuracion (Destructuring)

Es una forma elegante de extraer datos de objetos y arrays. En Node.js es muy util cuando recibimos datos de una API.

  • Objetos: const { nombre, edad } = persona;
  • Arrays: const [primero, segundo] = [1, 2];

🔥 Uso tipico: Para extraer req.body o req.query en Express.

Modulo 3: Template Literals e Inmutabilidad

Los Template Literals (usando comillas invertidas `) nos permiten insertar variables directamente en strings:

  • `Hola ${nombre}` en lugar de "Hola " + nombre
⚠️ Peligro de la referencia en JavaScript:
Cuando haces const copia = objetoOriginal, ambas variables apuntan al mismo objeto en memoria. Si modificas copia, tambien cambias objetoOriginal.

Para crear una copia independiente, usamos el Spread Operator (...):

  • const copia = { ...original }; - Crea un nuevo objeto con las mismas propiedades.

Modulo 4: Promesas con fetch

Una Promesa es un objeto que representa un valor que puede estar disponible ahora, en el futuro o nunca. Es la base del codigo asincrono en JavaScript.

Estructura de una Promesa:

  • Pendiente (pending): Estado inicial.
  • Cumplida (fulfilled): La operacion fue exitosa.
  • Rechazada (rejected): La operacion fallo.

Usamos .then() para manejar el exito y .catch() para manejar los errores:

  • fetch(url).then(response => response.json()).then(data => console.log(data))

Modulo 5: Async / Await

Async/Await es una forma mas legible de trabajar con Promesas. Hace que el codigo asincrono parezca sincrono.

  • async: Indica que una funcion devuelve una Promesa.
  • await: Pausa la ejecucion hasta que la Promesa se resuelva.

Siempre debemos usar try/catch para manejar errores:

  • try { const data = await fetch(url); } catch (error) { console.error(error); }
