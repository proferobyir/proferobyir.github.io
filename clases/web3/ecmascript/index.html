ECMAScript

Guía completa paso a paso para estudiantes de Informática y Sistemas

🎯 Objetivo: Aprender los fundamentos de ECMAScript que usarás todos los días en Node.js.

Bienvenido al entorno interactivo. Lee cada módulo y haz clic en "Cargar en el Editor" para ver el código en acción.

📦 Módulo 1: Let y Const

Reemplazan a var y solucionan sus problemas:

  • const: Valor constante que no puede reasignarse.
  • let: Variable con ámbito de bloque (dentro de {}).
💡 Regla práctica: Usa const por defecto. Solo usa let si sabes que el valor va a cambiar.

🎯 Módulo 2: Destructuring

Extrae datos de objetos y arrays de forma rápida y legible:

  • Objetos: const { nombre, edad } = persona;
  • Arrays: const [primero, segundo] = [1, 2];
  • Anidado: const { address: { city } } = user;

🔥 Uso común: Extraer datos de APIs como req.body en Express.

🧩 Módulo 3: Template Literals y Spread

Template Literals (con comillas invertidas `) permiten insertar variables:

  • `Hola ${nombre}` en lugar de "Hola " + nombre
⚠️ ¡Cuidado con la referencia!
const copia = original no crea una copia nueva. Ambas apuntan al mismo objeto.

Spread Operator ...

  • const copiaReal = { ...original }; - Copia independiente
  • const nuevoArray = [...viejo, 4, 5]; - Copia y expande arrays

⚡ Módulo 4: Promesas con fetch

Una Promesa representa un valor futuro. Estados:

  • ⏳ Pending: En proceso
  • ✅ Fulfilled: Completado con éxito
  • ❌ Rejected: Falló

Encadenamos con .then() para éxito y .catch() para errores:

  • fetch(url).then(res => res.json()).then(data => console.log(data))

🔄 Módulo 5: Async / Await

Hace el código asíncrono más legible, como si fuera síncrono:

  • async: Declara que la función devuelve una Promesa
  • await: Pausa hasta que la Promesa se resuelva
🛡️ Mejores prácticas:
- Usa try/catch para manejar errores
- Promise.all() para ejecutar múltiples peticiones en paralelo

🧰 Módulo 6: Métodos de Array (map, filter, reduce)

Los métodos funcionales son esenciales para procesar datos:

  • map() - Transforma cada elemento → nuevo array
  • filter() - Filtra elementos que cumplen condición
  • reduce() - Acumula todos los elementos en un solo valor

💡 Tip: Estos métodos son inmutables, no modifican el array original.

📦 Módulo 7: Módulos (import/export)

ECMAScript 2015 introdujo módulos nativos:

  • export: export const PI = 3.1416;
  • import: import { PI } from './math.js';
  • export default: Exporta un valor principal por módulo
📌 En Node.js: Necesitas "type": "module" en package.json o usar extensión .mjs.

🔗 Módulo 8: Optional Chaining (?.)

Accede a propiedades anidadas sin temor a errores:

  • usuario?.direccion?.ciudad - Si algo es null/undefined, devuelve undefined
  • array?.[0] - Acceso seguro a arrays
  • func?.() - Llama solo si la función existe
🚀 Ventaja: Elimina la necesidad de && y if para verificar nulos.

🛡️ Módulo 9: Nullish Coalescing (??)

Provee un valor por defecto solo cuando el valor es null o undefined:

  • const nombre = usuario.nombre ?? "Anónimo";

⚠️ Diferencia con ||: ?? no asigna default a valores falsos como 0 o ''.
