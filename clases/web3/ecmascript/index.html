🚀 ¿Cómo crear tu primer proyecto Node.js en Windows?

Paso 1: Abre la terminal de Windows (puedes presionar Win + X y seleccionar Terminal o PowerShell). Crea la carpeta de tu laboratorio y entra en ella:

mkdir mi-proyecto-node
cd mi-proyecto-node

Paso 2: Inicializa el gestor de paquetes de Node.js (NPM). Esto preparará tu entorno:

npm init -y
💡 Tip para la clase: El parámetro -y (yes) le dice a NPM que configure todo de forma automática con los valores por defecto. Si lo omites, te hará preguntas como el nombre del proyecto o la versión.

Paso 3: El comando anterior generará un archivo clave llamado package.json en tu directorio. Es la cédula de identidad de tu software.

Paso 4: Crea el script principal desde la consola de Windows. **Nota:** En Linux se usa touch, pero en Windows utilizaremos comandos nativos de la consola de comandos:

⚙️ Comando Nativo de Windows:
Si estás en PowerShell, usa:
New-Item app.js

Si estás en la **Símbolo del Sistema (CMD)** tradicional, usa:
type nul > app.js

Paso 5: Abre el archivo en tu editor (como VS Code) escribiendo code ., escribe tu lógica y ejecútala desde tu terminal mediante el motor runtime de Node:

node app.js
⚠️ Configuración Clave para ES Modules: Para poder usar las instrucciones de este módulo (como import y export), debes indicarle a Node.js que trabaje con módulos modernos. Abre tu package.json y agrega la siguiente propiedad: 
"type": "module"

📦 Módulo 1: Ámbito de Variables (Let vs. Const)

Olvídate del viejo var. El estándar moderno nos obliga a proteger la memoria y el ciclo de vida de los datos mediante ámbitos de bloque (todo lo que viva dentro de unas llaves {}).

💡 Analogía pedagógica: Piensa en const como una caja fuerte cerrada con llave: guardas algo y nadie lo puede cambiar. Piensa en let como una pizarra escolar: puedes escribir un valor, borrarlo más tarde y anotar uno nuevo.
  • const: Declara referencias inmutables. Ideal para importar librerías, conexiones a bases de datos o configuraciones fijas.
  • let: Variables limitadas estrictamente al bloque de código donde se declararon (estructuras if, bucles for, etc.).
const PUERTO = 3000; // No cambiará
let contador = 0;    // Cambiará con cada ciclo
contador++;

console.log(`Puerto: ${PUERTO}`);
console.log(`Contador: ${contador}`);

// Demostración de ámbito de bloque
if (true) {
    let mensaje = "Solo visible dentro del if";
    const PI = 3.1416;
    console.log(mensaje);
    console.log(`PI: ${PI}`);
}
// console.log(mensaje); // Error: mensaje no está definido fuera del bloque

🎯 Módulo 2: Desestructuración (Destructuring)

Es una técnica que permite "desempacar" valores de arreglos o propiedades de objetos directamente en variables individuales con una sintaxis limpia.

💡 Analogía pedagógica: Imagina que te entregan una caja de herramientas (objeto). En lugar de escribir cada vez herramientas.martillo, usas destructuring para sacar el martillo y la pinza y colocarlos directo sobre tu mesa de trabajo para usarlos por su nombre.
  • Objetos: Extracción por el nombre exacto de la clave: const { nombre, rol } = usuario;
  • Arrays: Extracción secuencial por la posición del índice: const [primero, segundo] = listaNumeros;
🔥 Uso real en backend: Cuando manejas solicitudes HTTP en arquitecturas como Express.js, lo usarás constantemente para parsear JSON: const { username, password } = req.body; 
// Desestructuración de objetos
const usuario = {
    nombre: 'Ana',
    edad: 28,
    rol: 'desarrolladora',
    direccion: {
        ciudad: 'Madrid',
        pais: 'España'
    }
};

// Extraemos propiedades específicas
const { nombre, edad, rol } = usuario;
console.log(`Nombre: ${nombre}, Edad: ${edad}, Rol: ${rol}`);

// Desestructuración anidada
const { direccion: { ciudad, pais } } = usuario;
console.log(`Ciudad: ${ciudad}, País: ${pais}`);

// Desestructuración de arrays
const numeros = [10, 20, 30, 40, 50];
const [primero, segundo, ...resto] = numeros;
console.log(`Primero: ${primero}, Segundo: ${segundo}`);
console.log(`Resto: ${resto}`);

// Valores por defecto
const { telefono = 'No disponible' } = usuario;
console.log(`Teléfono: ${telefono}`);

🧩 Módulo 3: Template Literals y Spread Operator

Template Literals (Plantillas de Texto)

Olvídate de concatenar con el molesto operador suma (+). Usando las comillas invertidas o backticks (`) puedes inyectar expresiones de JavaScript directo en el flujo de texto usando ${expresión}.

Spread Operator (Operador de Propagación ...)

Permite expandir o "esparcir" los elementos de un array o las propiedades de un objeto.

⚠️ ¡Peligro con la asignación por referencia!
En JS, si haces const copia = original; no estás duplicando el objeto, estás creando un nuevo puntero al mismo espacio de memoria. Si cambias la copia, destruirás el original. El operador spread soluciona esto clonando el contenido de forma segura: 
const copiaReal = { ...original };
// Template Literals
const nombre = 'Carlos';
const edad = 25;
const ciudad = 'Barcelona';

// String tradicional con concatenación
const mensaje1 = 'Hola, ' + nombre + '. Tienes ' + edad + ' años y vives en ' + ciudad + '.';
console.log(mensaje1);

// Template Literal
const mensaje2 = `Hola, ${nombre}. Tienes ${edad} años y vives en ${ciudad}.`;
console.log(mensaje2);

// Expresiones dentro de template literals
const precio = 150;
const iva = 0.21;
console.log(`Precio total: ${precio * (1 + iva)}€`);

// Spread Operator con arrays
const paisesEuropeos = ['España', 'Francia', 'Italia'];
const paisesAmericanos = ['México', 'Brasil', 'Argentina'];
const todosLosPaises = [...paisesEuropeos, ...paisesAmericanos];
console.log('Todos los países:', todosLosPaises);

// Spread Operator con objetos
const datosPersonales = { nombre, edad, ciudad };
const datosProfesionales = { rol: 'programador', empresa: 'TechCorp' };
const perfilCompleto = { ...datosPersonales, ...datosProfesionales };
console.log('Perfil completo:', perfilCompleto);

// Clonación segura de objetos
const original = { a: 1, b: 2 };
const copia = { ...original };
copia.a = 99;
console.log('Original:', original);  // { a: 1, b: 2 }
console.log('Copia:', copia);        // { a: 99, b: 2 }

⚡ Módulo 4: Asincronía y Promesas (fetch)

JavaScript es asíncrono y corre sobre un solo hilo. Para operaciones pesadas (como consultar una API pública en internet o una base de datos), utiliza **Promesas**, que representan procesos que se resolverán en el futuro.

💡 Analogía pedagógica: Cuando pides comida rápida, te dan un "beeper" o vibrador electrónico. Ese aparato representa una **Promesa**: está en estado **pendiente**. Si la comida está lista, el aparato vibra (**resuelta con éxito**). Si la cocina se quedó sin ingredientes, se activa una luz roja (**rechazada**).

Las promesas manejan tres estados técnicos fundamentales:

  • ⏳ Pending (Pendiente): La operación asíncrona aún se está ejecutando en segundo plano.
  • ✅ Fulfilled (Cumplida): Completada con éxito. Capturamos los datos devueltos usando el método .then().
  • ❌ Rejected (Rechazada): La operación falló (error de red, caída del servidor). Capturamos el fallo con .catch().
// Simulación de una promesa que resuelve después de 2 segundos
const promesaSimulada = new Promise((resolve, reject) => {
    console.log('⏳ Promesa en estado pendiente...');
    setTimeout(() => {
        const exito = true; // Cambiar a false para probar el rechazo
        if (exito) {
            resolve('✅ Promesa resuelta con éxito');
        } else {
            reject('❌ Promesa rechazada');
        }
    }, 2000);
});

// Consumir la promesa con .then() y .catch()
promesaSimulada
    .then(resultado => {
        console.log(resultado);
        return 'Procesando resultado...';
    })
    .then(mensaje => {
        console.log(mensaje);
    })
    .catch(error => {
        console.log('Error capturado:', error);
    })
    .finally(() => {
        console.log('🔄 La promesa ha finalizado (éxito o error)');
    });

// Ejemplo práctico: Simular fetch a una API
const obtenerUsuarios = () => {
    return new Promise((resolve) => {
        setTimeout(() => {
            resolve([
                { id: 1, nombre: 'María', email: 'maria@email.com' },
                { id: 2, nombre: 'Juan', email: 'juan@email.com' }
            ]);
        }, 1500);
    });
};

obtenerUsuarios()
    .then(usuarios => {
        console.log('📋 Usuarios obtenidos:');
        usuarios.forEach(usuario => {
            console.log(`- ${usuario.nombre} (${usuario.email})`);
        });
    });

🔄 Módulo 5: Async / Await

Es una envoltura sintáctica ("sugar syntax") construida sobre las promesas. No cambia cómo funcionan internamente, pero hace que tu código asíncrono se lea de forma secuencial y limpia, simulando el comportamiento de un bloque síncrono.

  • async: Modificador obligatorio que se coloca antes de declarar la función. Indica que la función retornará, sí o sí, una promesa.
  • await: Pausa la ejecución de la función de manera asíncrona hasta que la promesa externa se resuelva. **Solo puede usarse dentro de funciones marcadas como async**.
🛡️ Estructura recomendada: Para interceptar errores en arquitecturas asíncronas limpias, siempre envuelve tus operaciones con bloques tradicionales de control de excepciones: try { ... } catch (error) { ... }. 
// Función asíncrona que simula una operación con delay
const delay = (ms) => new Promise(resolve => setTimeout(resolve, ms));

// Función que simula obtener datos de una API
const obtenerDatosAPI = async () => {
    console.log('🔄 Iniciando petición a la API...');
    await delay(2000); // Simula tiempo de red
    return {
        usuarios: ['Ana', 'Luis', 'Marta'],
        timestamp: new Date().toISOString()
    };
};

// Función principal con async/await
const procesarDatos = async () => {
    try {
        console.log('🚀 Iniciando proceso...');
        
        // Obtenemos datos con await
        const datos = await obtenerDatosAPI();
        console.log('✅ Datos recibidos:', datos);
        
        // Procesamos los datos
        const usuarios = datos.usuarios;
        console.log('📋 Lista de usuarios:');
        usuarios.forEach((usuario, index) => {
            console.log(`${index + 1}. ${usuario}`);
        });
        
        // Simulamos otra operación asíncrona
        console.log('🔄 Guardando información...');
        await delay(1000);
        console.log('💾 Información guardada exitosamente');
        
        return 'Proceso completado';
    } catch (error) {
        console.error('❌ Error en el proceso:', error);
        throw error; // Re-lanzamos el error para manejo externo
    }
};

// Ejecutamos la función principal
procesarDatos()
    .then(resultado => console.log('✨', resultado))
    .catch(error => console.error('💥 Error fatal:', error));

// Ejemplo con múltiples operaciones en paralelo
const obtenerMultiplesDatos = async () => {
    console.log('🔄 Obteniendo datos en paralelo...');
    
    // Usamos Promise.all para ejecutar promesas en paralelo
    const [usuarios, productos] = await Promise.all([
        delay(1000).then(() => ['Usuario1', 'Usuario2']),
        delay(1500).then(() => ['ProductoA', 'ProductoB'])
    ]);
    
    console.log('Usuarios:', usuarios);
    console.log('Productos:', productos);
};

obtenerMultiplesDatos();

🧰 Módulo 6: Métodos de Arreglos Immutables

Herramientas fundamentales de programación funcional para manipular colecciones de datos sin modificar el arreglo original en memoria (Inmutabilidad).

  • map(): Recorre el array y transforma cada elemento uno a uno, retornando un **nuevo array** con la misma longitud.
  • filter(): Evalúa cada elemento bajo una condición lógica. Retorna un nuevo array que contiene únicamente los elementos que devolvieron true.
  • reduce(): Procesa la colección completa para acumularla en un único valor final (como la suma total de precios de un carrito).
// Datos de ejemplo: lista de estudiantes
const estudiantes = [
    { nombre: 'Carlos', edad: 22, calificacion: 85 },
    { nombre: 'María', edad: 20, calificacion: 92 },
    { nombre: 'Luis', edad: 23, calificacion: 78 },
    { nombre: 'Ana', edad: 21, calificacion: 95 },
    { nombre: 'Jorge', edad: 22, calificacion: 70 }
];

console.log('📋 Lista original de estudiantes:');
console.log(estudiantes);

// 1. map() - Transformar cada elemento
const nombresMayusculas = estudiantes.map(est => est.nombre.toUpperCase());
console.log('📝 Nombres en mayúsculas:', nombresMayusculas);

// 2. map() - Crear un nuevo array con información adicional
const estudiantesConEstado = estudiantes.map(est => ({
    ...est,
    estado: est.calificacion >= 80 ? 'Aprobado' : 'Reprobado'
}));
console.log('📊 Estudiantes con estado:');
console.log(estudiantesConEstado);

// 3. filter() - Filtrar por condición
const aprobados = estudiantes.filter(est => est.calificacion >= 80);
console.log('✅ Estudiantes aprobados:', aprobados);

const reprobados = estudiantes.filter(est => est.calificacion < 80);
console.log('❌ Estudiantes reprobados:', reprobados);

// 4. reduce() - Acumular valores
const sumaCalificaciones = estudiantes.reduce((acumulador, est) => acumulador + est.calificacion, 0);
const promedioCalificaciones = sumaCalificaciones / estudiantes.length;
console.log(`📈 Promedio de calificaciones: ${promedioCalificaciones.toFixed(2)}`);

// 5. reduce() - Agrupar por categoría
const estudiantesPorEdad = estudiantes.reduce((grupos, est) => {
    const edad = est.edad;
    if (!grupos[edad]) {
        grupos[edad] = [];
    }
    grupos[edad].push(est);
    return grupos;
}, {});
console.log('👥 Estudiantes agrupados por edad:');
console.log(estudiantesPorEdad);

// 6. Combinar métodos para análisis complejos
const analisis = estudiantes
    .filter(est => est.calificacion >= 80)
    .map(est => ({
        nombre: est.nombre,
        calificacion: est.calificacion,
        nivel: est.calificacion >= 90 ? 'Excelente' : 'Bueno'
    }))
    .sort((a, b) => b.calificacion - a.calificacion);

console.log('🏆 Ranking de aprobados:');
console.log(analisis);

🔗 Módulo 7: Optional Chaining (?.) y Nullish Coalescing (??)

Optional Chaining (Encadenamiento Opcional)

Permite leer propiedades profundamente anidadas dentro de un árbol de objetos sin peligro de que la aplicación colapse si una propiedad intermedia es null o undefined.

Nullish Coalescing (Operador de Fusión de Nulos)

Es un operador lógico que retorna el valor del lado derecho únicamente cuando el valor del lado izquierdo es estrictamente null o undefined.

⚠️ Diferencia Crítica en Exámenes (Diferencia con ||): El operador OR lógico (||) asignará el valor por defecto si el operando izquierdo es cualquier valor falso (como el número 0 o un string vacío ""). En cambio, ?? respeta al 0 y a los textos vacíos como valores válidos del usuario. 
// Datos de ejemplo con propiedades anidadas y valores nulos
const cliente1 = {
    nombre: 'Laura',
    direccion: {
        calle: 'Calle Mayor',
        ciudad: 'Valencia',
        codigoPostal: '46001'
    },
    preferencias: {
        notificaciones: false,
        temas: ['oscuro', 'claro']
    }
};

const cliente2 = {
    nombre: 'Pedro',
    direccion: null,
    preferencias: null
};

const cliente3 = {
    nombre: 'Sofía',
    direccion: {
        calle: 'Avenida Central'
        // ciudad no está definida
    },
    preferencias: {
        notificaciones: true,
        temas: []
    }
};

// 1. Optional Chaining (?.) - Acceso seguro a propiedades anidadas
console.log('📋 Cliente 1:');
console.log('Ciudad:', cliente1?.direccion?.ciudad); // 'Valencia'
console.log('Temas:', cliente1?.preferencias?.temas); // ['oscuro', 'claro']

console.log('📋 Cliente 2:');
console.log('Ciudad:', cliente2?.direccion?.ciudad); // undefined (no rompe)
console.log('Temas:', cliente2?.preferencias?.temas); // undefined

console.log('📋 Cliente 3:');
console.log('Ciudad:', cliente3?.direccion?.ciudad); // undefined
console.log('Temas:', cliente3?.preferencias?.temas); // []

// 2. Nullish Coalescing (??) - Valores por defecto seguros
const ciudad1 = cliente1?.direccion?.ciudad ?? 'Ciudad no disponible';
console.log('Ciudad Cliente 1:', ciudad1);

const ciudad2 = cliente2?.direccion?.ciudad ?? 'Ciudad no disponible';
console.log('Ciudad Cliente 2:', ciudad2);

// 3. Diferencia entre ?? y || (ejemplo crítico)
const configuracion = {
    puerto: 0,
    nombre: '',
    debug: false,
    timeout: null
};

// Usando || (OR) - considera 0, '', false como falsos
const puertoOR = configuracion.puerto || 3000;  // 3000 (pierde el 0)
const nombreOR = configuracion.nombre || 'default';  // 'default' (pierde el string vacío)
console.log('Con ||:', puertoOR, nombreOR);

// Usando ?? (Nullish) - solo considera null/undefined
const puertoNullish = configuracion.puerto ?? 3000;  // 0 (preserva el 0)
const nombreNullish = configuracion.nombre ?? 'default';  // '' (preserva el string vacío)
const timeoutNullish = configuracion.timeout ?? 5000;  // 5000 (porque timeout es null)
console.log('Con ??:', puertoNullish, nombreNullish, timeoutNullish);

// 4. Combinación de ?. y ??
const mensajeBienvenida = cliente1?.nombre ?? 'Usuario invitado';
console.log('👋 Bienvenido:', mensajeBienvenida);

const mensajeBienvenida2 = cliente2?.nombre ?? 'Usuario invitado';
console.log('👋 Bienvenido:', mensajeBienvenida2);

// 5. Optional Chaining con métodos
const clienteConMetodo = {
    nombre: 'Javier',
    obtenerEmail() {
        return 'javier@email.com';
    }
};

const clienteSinMetodo = {
    nombre: 'Elena'
};

console.log('Email:', clienteConMetodo.obtenerEmail?.());
console.log('Email:', clienteSinMetodo.obtenerEmail?.()); // undefined
