1. Configuración de Entorno en el Servidor

📂

Ruta Obligatoria de Almacenamiento:
Los siguientes dos archivos deben guardarse estrictamente en el directorio de tu perfil de usuario en Windows:
C:\Users\TU_USUARIO\.aws\

Archivo A: credentials (Crear sin extensión .txt)

credentials 
[default]
aws_access_key_id = test
aws_secret_access_key = test

Archivo B: config (Crear sin extensión .txt)

config 
[default]
region = us-east-1
output = json

2. Capa de Datos: Creación de Tabla en DynamoDB

Se inicializa la persistencia. Definimos la clave primaria limpia como email (sin el prefijo 'txt') de tipo String (S).

Windows CMD 
aws dynamodb create-table --table-name estudiantes --attribute-definitions AttributeName=email,AttributeType=S --key-schema AttributeName=email,KeyType=HASH --provisioned-throughput ReadCapacityUnits=5,WriteCapacityUnits=5 --endpoint-url http://localhost:4566

3. Capa de Lógica: El Código de la Función (index.js)

💻

Ubicación del Espacio de Trabajo:
Crea y guarda el archivo index.js exactamente dentro del directorio local del usuario:
C:\Users\TU_USUARIO\lab\index.js
Asegúrate de que tu consola CMD esté abierta apuntando a esta ruta (cd C:\Users\TU_USUARIO\lab) antes de continuar.

Este procesador toma los datos mnemotécnicos enviados por tus alumnos y los mapea de forma limpia antes de guardarlos en DynamoDB:

index.js (Node.js) 
const { DynamoDBClient } = require("@aws-sdk/client-dynamodb");
const { DynamoDBDocumentClient, PutCommand } = require("@aws-sdk/lib-dynamodb");

const client = new DynamoDBClient({ endpoint: process.env.AWS_ENDPOINT_URL || "http://localhost:4566" });
const docClient = DynamoDBDocumentClient.from(client);

exports.handler = async (event) => {
    const headers = {
        "Access-Control-Allow-Origin": "*",
        "Access-Control-Allow-Headers": "Content-Type",
        "Access-Control-Allow-Methods": "OPTIONS,POST"
    };

    if (event.requestContext.http.method === "OPTIONS") {
        return { statusCode: 200, headers, body: "" };
    }

    try {
        const data = JSON.parse(event.body);
        
        if (!data.txtemail || !data.txtnombre || !data.txtphone) {
            return {
                statusCode: 400,
                headers,
                body: JSON.stringify({ message: "Todos los campos son obligatorios" })
            };
        }

        // Mapeo: Recibe mnemotécnicos (txt) -> Guarda nombres limpios en la BD
        const params = {
            TableName: "estudiantes",
            Item: {
                email: data.txtemail,
                nombre: data.txtnombre,
                telefono: data.txtphone,
                fecha_registro: new Date().toISOString()
            }
        };

        await docClient.send(new PutCommand(params));

        return {
            statusCode: 200,
            headers,
            body: JSON.stringify({ message: "Estudiante registrado con éxito en DynamoDB (Campos normalizados)" })
        };
    } catch (error) {
        return {
            statusCode: 500,
            headers,
            body: JSON.stringify({ error: error.message })
        };
    }
};

4. Capa de Lógica: Compresión y Carga a MiniStack

Empaqueta el archivo index.js que se encuentra en tu directorio \lab en un archivo binario `.zip` usando comandos de Windows y súbelo al contenedor Docker:

A. Comprimir usando comandos nativos (Ejecutar parados dentro de \lab):

Windows CMD 
tar -a -c -f funcion-lambda.zip index.js

B. Crear el recurso funcional en MiniStack:

Windows CMD 
aws lambda create-function --function-name procesarEstudiantes --runtime nodejs18.x --role arn:aws:iam::000000000000:role/lambda-ex --handler index.handler --zip-file fileb://funcion-lambda.zip --endpoint-url http://localhost:4566

5. Capa de Integración: Configurar API Gateway

Expón la infraestructura mediante HTTP común ejecutando estas líneas en orden secuencial:

A. Crear API REST:

Windows CMD 
aws apigateway create-rest-api --name "API-Estudiantes" --endpoint-url http://localhost:4566

B. Obtener ID del Recurso Raíz:

Windows CMD 
aws apigateway get-resources --rest-api-id TU_API_ID --endpoint-url http://localhost:4566

C. Crear recurso secundario /estudiantes:

Windows CMD 
aws apigateway create-resource --rest-api-id TU_API_ID --parent-id TU_PARENT_ID --path-part estudiantes --endpoint-url http://localhost:4566

D. Métodos e Integración Proxy con la Lambda:

Windows CMD (Método POST) 
aws apigateway put-method --rest-api-id TU_API_ID --resource-id RESOURCE_ID --http-method POST --authorization-type "NONE" --endpoint-url http://localhost:4566
Windows CMD (Enlace) 
aws apigateway put-integration --rest-api-id TU_API_ID --resource-id RESOURCE_ID --http-method POST --type AWS_PROXY --integration-http-method POST --uri arn:aws:apigateway:us-east-1:lambda:path/2015-03-31/functions/arn:aws:lambda:us-east-1:000000000000:function:procesarEstudiantes/submissions --endpoint-url http://localhost:4566

E. Publicar Despliegue (prod):

Windows CMD 
aws apigateway create-deployment --rest-api-id TU_API_ID --stage-name prod --endpoint-url http://localhost:4566

6. Interfaz del Estudiante (Código que pican en el Laboratorio)

Este es el HTML minimalista sin CSS que escribirán tus muchachos en la pizarra. Forzarás la interacción directa con el DOM y el entendimiento del objeto payload:

estudiante.html 
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Práctica Cloud - Datos Puros</title>
</head>
<body>

    <h2>Formulario de Registro Cloud</h2>
    <p id="responseMessage" style="font-weight: bold; color: blue;"></p>

    <form id="studentForm">
        <div>
            <label>Nombre:</label>
            <input type="text" id="txtnombre" required>
        </div><br>
        <div>
            <label>Teléfono:</label>
            <input type="tel" id="txtphone" required>
        </div><br>
        <div>
            <label>Correo:</label>
            <input type="email" id="txtemail" required>
        </div><br>
        <button type="submit">Enviar a AWS MiniStack</button>
    </form>

    <script>
        document.getElementById('studentForm').addEventListener('submit', async (e) => {
            e.preventDefault();
            
            const msgCtrl = document.getElementById('responseMessage');
            msgCtrl.innerText = "Conectando con el servidor en EEUU...";

            const payload = {
                txtnombre: document.getElementById('txtnombre').value,
                txtphone: document.getElementById('txtphone').value,
                txtemail: document.getElementById('txtemail').value
            };

            // URL del API Gateway generada por el profesor en la sección 5
            const API_ENDPOINT = "http://awslocal.ddns.net:4566/restapis/TU_API_ID/prod/_user_request_/estudiantes";

            try {
                const response = await fetch(API_ENDPOINT, {
                    method: 'POST',
                    headers: { 'Content-Type': 'application/json' },
                    body: JSON.stringify(payload)
                });

                const result = await response.json();

                if (response.ok) {
                    msgCtrl.style.color = "green";
                    msgCtrl.innerText = result.message;
                } else {
                    msgCtrl.style.color = "red";
                    msgCtrl.innerText = "Error en el backend: " + (result.error || "Datos inválidos");
                }
            } catch (error) {
                msgCtrl.style.color = "red";
                msgCtrl.innerText = "Error de red: El proxy bloqueó la petición.";
                console.error(error);
            }
        });
    </script>
</body>
</html>
