1. Crear la Tabla en DynamoDB

Crea la tabla persistente utilizando comandos puros de una sola línea compatibles con el CMD de Windows para evitar errores de salto de línea.

Windows CMD 
aws dynamodb create-table --table-name estudiantes --attribute-definitions AttributeName=email,AttributeType=S --key-schema AttributeName=email,KeyType=HASH --provisioned-throughput ReadCapacityUnits=5,WriteCapacityUnits=5 --endpoint-url http://localhost:4566

2. Empaquetar y Desplegar la Función Lambda

Genera el archivo comprimido sin depender de software externo usando el empaquetador nativo de Windows y súbelo a MiniStack:

Windows CMD (Paso A) 
tar -a -c -f funcion-lambda.zip index.js
Windows CMD (Paso B) 
aws lambda create-function --function-name procesarEstudiantes --runtime nodejs18.x --role arn:aws:iam::000000000000:role/lambda-ex --handler index.handler --zip-file fileb://funcion-lambda.zip --endpoint-url http://localhost:4566

3. Crear y Enlazar el API Gateway

Ejecuta la secuencia de comandos secuenciales para exponer públicamente tu microservicio HTTP:

A. Crear la API REST (Guarda el ID retornado):

Windows CMD 
aws apigateway create-rest-api --name "API-Estudiantes" --endpoint-url http://localhost:4566

B. Obtener el ID del recurso raíz (/):

Windows CMD 
aws apigateway get-resources --rest-api-id TU_API_ID --endpoint-url http://localhost:4566

C. Crear el recurso /estudiantes (Usa el Parent ID anterior):

Windows CMD 
aws apigateway create-resource --rest-api-id TU_API_ID --parent-id TU_PARENT_ID --path-part estudiantes --endpoint-url http://localhost:4566

D. Crear el método POST y mapearlo a la Lambda:

Windows CMD (Método) 
aws apigateway put-method --rest-api-id TU_API_ID --resource-id RESOURCE_ID --http-method POST --authorization-type "NONE" --endpoint-url http://localhost:4566
Windows CMD (Integración) 
aws apigateway put-integration --rest-api-id TU_API_ID --resource-id RESOURCE_ID --http-method POST --type AWS_PROXY --integration-http-method POST --uri arn:aws:apigateway:us-east-1:lambda:path/2015-03-31/functions/arn:aws:lambda:us-east-1:000000000000:function:procesarEstudiantes/submissions --endpoint-url http://localhost:4566

E. Desplegar la API a Producción:

Windows CMD 
aws apigateway create-deployment --rest-api-id TU_API_ID --stage-name prod --endpoint-url http://localhost:4566

4. Estructura de la URL del Laboratorio

Esta es la dirección base que debes escribir en la pizarra reemplazando tu identificador único para que los alumnos configuren el fetch de JavaScript:
