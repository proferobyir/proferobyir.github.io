0. Inicializacion de la Infraestructura Emulada

[REQUISITOS]

REQUISITOS PREVIOS OBLIGATORIOS EN WINDOWS:

1. Habilitar WSL (Windows Subsystem for Linux): Docker Desktop requiere de la arquitectura Linux interna de Windows para operar. Si no esta instalado, abre una consola de comandos (CMD) como Administrador y ejecuta:
wsl --install
*(Nota: Si la maquina lo requiere, reinicia el equipo tras completar la instalacion).*

2. Ejecutar Docker Desktop: Una vez activo WSL, abre la aplicacion Docker Desktop. Certifica que el icono inferior de la barra de tareas cambie a color verde (Running). Si el motor de Docker no esta activo en segundo plano, la consola fallara al intentar inicializar los contenedores.

Crea un archivo llamado docker-compose.yml en la raiz de tu proyecto para orquestar el contenedor que simulera los servidores de Amazon Web Services (AWS) de forma local:

docker-compose.yml 
version: '3.8'

services:
  ministack:
    image: ministackorg/ministack:full
    container_name: ministack_practica
    ports:
      - "4566:4566"           # Puerto principal de los servicios de AWS
      - "4510-4559:4510-4559" # Puertos que usa MiniStack internamente para las Lambdas de Docker
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock # Crucial para que funcionen las Lambdas
    environment:
      - AWS_DEFAULT_REGION=us-east-1

Comando para encender los servidores desde la consola:

Windows CMD 
docker-compose up -d

1. Instalacion del Cliente AWS y Configuracion de Entorno

Para poder enviar comandos a la infraestructura emulada, la maquina de desarrollo requiere de la interfaz de linea de comandos oficial de Amazon.

Descarga del AWS CLI:
Para sistemas Windows de 64 bits, descargar e instalar el instalador oficial MSI desde el siguiente enlace:
Instalador Directo AWS CLI v2 (.msi)

A. Verificar la instalacion del cliente:

Abre una terminal de comandos nueva (CMD) y ejecuta la siguiente linea para certificar que el sistema reconoce la herramienta:

Windows CMD 
aws --version
[RUTA]

Ruta Obligatoria de Credenciales:
Una vez verificado el cliente, los siguientes dos archivos de configuracion deben guardarse estrictamente en el directorio del perfil de usuario de Windows:
C:\Users\TU_USUARIO\.aws\

Archivo A: credentials

credentials 
[default]
aws_access_key_id = test
aws_secret_access_key = test

Archivo B: config

config 
[default]
region = us-east-1
output = json

2. Capa de Datos: Creacion de Tabla en DynamoDB

Se inicializa la persistencia de datos. Definimos la clave primaria limpia como email de tipo String (S).

Windows CMD 
aws dynamodb create-table --table-name estudiantes --attribute-definitions AttributeName=email,AttributeType=S --key-schema AttributeName=email,KeyType=HASH --provisioned-throughput ReadCapacityUnits=5,WriteCapacityUnits=5 --endpoint-url http://localhost:4566

3. Capa de Logica: El Codigo Adaptado (index.js)

[UBICACION]

Ubicacion del Espacio de Trabajo:
Crea una carpeta llamada lab dentro de tu carpeta de usuario y guarda el archivo alli:
C:\Users\TU_USUARIO\lab\index.js
Tu consola CMD debe operar apuntando a este directorio (cd %USERPROFILE%\lab).

Este codigo maneja dos metodos HTTP: POST para guardar estudiantes y GET para listarlos, utilizando endpoints separados en API Gateway:

index.js (Node.js) 
const { DynamoDBClient } = require("@aws-sdk/client-dynamodb");
const { DynamoDBDocumentClient, PutCommand, ScanCommand } = require("@aws-sdk/lib-dynamodb");

const client = new DynamoDBClient({ endpoint: process.env.AWS_ENDPOINT_URL || "http://localhost:4566" });
const docClient = DynamoDBDocumentClient.from(client);

exports.handler = async (event) => {
    console.log("Evento recibido:", JSON.stringify(event));

    const headers = {
        "Access-Control-Allow-Origin": "*",
        "Access-Control-Allow-Headers": "Content-Type",
        "Access-Control-Allow-Methods": "OPTIONS,POST,GET"
    };

    const method = event.httpMethod || (event.requestContext && event.requestContext.http && event.requestContext.http.method);

    if (method === "OPTIONS") {
        return { statusCode: 200, headers, body: "" };
    }

    try {
        // GET: Listar todos los estudiantes
        if (method === "GET") {
            const params = { TableName: "estudiantes" };
            const result = await docClient.send(new ScanCommand(params));
            return {
                statusCode: 200,
                headers,
                body: JSON.stringify(result.Items || [])
            };
        }

        // POST: Guardar un estudiante
        if (method === "POST") {
            const data = JSON.parse(event.body);
            
            if (!data.txtemail || !data.txtnombre || !data.txtphone) {
                return {
                    statusCode: 400,
                    headers,
                    body: JSON.stringify({ message: "Todos los campos son obligatorios" })
                };
            }

            const params = {
                TableName: "estudiantes",
                Item: {
                    email: data.txtemail,
                    nombre: data.txtnombre,
                    telefono: data.txtphone,
                    fecha_registro: new Date().toISOString()
                }
            };

            await docClient.send(new PutCommand(params));

            return {
                statusCode: 200,
                headers,
                body: JSON.stringify({ message: "Estudiante registrado con exito en DynamoDB" })
            };
        }

        return {
            statusCode: 405,
            headers,
            body: JSON.stringify({ error: "Metodo no permitido. Use GET o POST" })
        };

    } catch (error) {
        return {
            statusCode: 500,
            headers,
            body: JSON.stringify({ error: error.message })
        };
    }
};

4. Capa de Logica: Gestion de Dependencias e Inyeccion Completa

Dado que el entorno de ejecucion requiere el SDK utilitario de DynamoDB, es obligatorio inicializar el paquete localmente e incluir la carpeta node_modules dentro de la compresion binaria:

A. Descargar dependencias necesarias (Ejecutar parados dentro de \lab):

Windows CMD 
npm init -y
npm install @aws-sdk/lib-dynamodb

B. Comprimir de forma compuesta (Codigo + Modulos):

Windows CMD 
tar -a -c -f funcion-lambda.zip index.js node_modules package.json

C. Crear el recurso funcional en MiniStack:

Windows CMD 
aws lambda create-function --function-name procesarEstudiantes --runtime nodejs18.x --role arn:aws:iam::000000000000:role/lambda-ex --handler index.handler --zip-file fileb://funcion-lambda.zip --endpoint-url http://localhost:4566

5. Capa de Integracion: Configurar API Gateway con Dos Endpoints

Se crean dos recursos independientes: /estudiantes para POST y /listar para GET. Ejecuta estos comandos en orden secuencial:

A. Instanciar la API REST en el contenedor:

Windows CMD 
aws apigateway create-rest-api --name "API-Estudiantes" --endpoint-url http://localhost:4566
[SALIDA]

Salida JSON Esperada en Consola (Paso A):
La terminal te arrojara la metadata de creacion. Identifica el campo id de la API:
{ "id": "294eb1e6", "name": "API-Estudiantes" }
Este valor es tu API_ID (Ejemplo: 294eb1e6).


B. Consultar el identificador del recurso raiz (/):

Windows CMD 
aws apigateway get-resources --rest-api-id TU_API_ID --endpoint-url http://localhost:4566
[SALIDA]

Salida JSON Esperada (Paso B):
Captura el ID del path "/":
{ "items": [ { "id": "aac51348", "path": "/" } ] }
Este es tu ROOT_ID (Ejemplo: aac51348).


C. Crear el endpoint POST /estudiantes:

Windows CMD 
aws apigateway create-resource --rest-api-id TU_API_ID --parent-id ROOT_ID --path-part estudiantes --endpoint-url http://localhost:4566
[SALIDA]

Salida JSON Esperada (Paso C):
Captura el ID del recurso /estudiantes:
{ "id": "d4f56816", "path": "/estudiantes" }
Este es tu ESTUDIANTES_ID.


D. Configurar metodo POST en /estudiantes:

Windows CMD 
aws apigateway put-method --rest-api-id TU_API_ID --resource-id ESTUDIANTES_ID --http-method POST --authorization-type "NONE" --endpoint-url http://localhost:4566

aws apigateway put-integration --rest-api-id TU_API_ID --resource-id ESTUDIANTES_ID --http-method POST --type AWS_PROXY --integration-http-method POST --uri arn:aws:apigateway:us-east-1:lambda:path/2015-03-31/functions/arn:aws:lambda:us-east-1:000000000000:function:procesarEstudiantes/submissions --endpoint-url http://localhost:4566

E. Crear el endpoint GET /listar:

Windows CMD 
aws apigateway create-resource --rest-api-id TU_API_ID --parent-id ROOT_ID --path-part listar --endpoint-url http://localhost:4566
[SALIDA]

Salida JSON Esperada (Paso E):
Captura el ID del recurso /listar:
{ "id": "f7g68179", "path": "/listar" }
Este es tu LISTAR_ID.


F. Configurar metodo GET en /listar:

Windows CMD 
aws apigateway put-method --rest-api-id TU_API_ID --resource-id LISTAR_ID --http-method GET --authorization-type "NONE" --endpoint-url http://localhost:4566

aws apigateway put-integration --rest-api-id TU_API_ID --resource-id LISTAR_ID --http-method GET --type AWS_PROXY --integration-http-method POST --uri arn:aws:apigateway:us-east-1:lambda:path/2015-03-31/functions/arn:aws:lambda:us-east-1:000000000000:function:procesarEstudiantes/submissions --endpoint-url http://localhost:4566

5B. Capa de Seguridad de Red: Soporte CORS

Los navegadores web ejecutan una validacion previa de seguridad (peticion OPTIONS). Es necesario habilitar CORS en ambos endpoints:

Para /estudiantes (POST):

Windows CMD 
aws apigateway put-method --rest-api-id TU_API_ID --resource-id ESTUDIANTES_ID --http-method OPTIONS --authorization-type "NONE" --endpoint-url http://localhost:4566

aws apigateway put-integration --rest-api-id TU_API_ID --resource-id ESTUDIANTES_ID --http-method OPTIONS --type AWS_PROXY --integration-http-method POST --uri arn:aws:apigateway:us-east-1:lambda:path/2015-03-31/functions/arn:aws:lambda:us-east-1:000000000000:function:procesarEstudiantes/submissions --endpoint-url http://localhost:4566

Para /listar (GET):

Windows CMD 
aws apigateway put-method --rest-api-id TU_API_ID --resource-id LISTAR_ID --http-method OPTIONS --authorization-type "NONE" --endpoint-url http://localhost:4566

aws apigateway put-integration --rest-api-id TU_API_ID --resource-id LISTAR_ID --http-method OPTIONS --type AWS_PROXY --integration-http-method POST --uri arn:aws:apigateway:us-east-1:lambda:path/2015-03-31/functions/arn:aws:lambda:us-east-1:000000000000:function:procesarEstudiantes/submissions --endpoint-url http://localhost:4566

Despliegue en produccion (OBLIGATORIO):

Windows CMD 
aws apigateway create-deployment --rest-api-id TU_API_ID --stage-name prod --endpoint-url http://localhost:4566

    <script>
        document.getElementById('studentForm').addEventListener('submit', async (e) => {
            e.preventDefault();
            
            const msgCtrl = document.getElementById('responseMessage');
            msgCtrl.innerText = "Conectando con el servidor...";

            const payload = {
                txtnombre: document.getElementById('txtnombre').value,
                txtphone: document.getElementById('txtphone').value,
                txtemail: document.getElementById('txtemail').value
            };

            const API_ENDPOINT = "http://awslocal.ddns.net:4566/restapis/TU_API_ID/prod/_user_request_/estudiantes";

            try {
                const response = await fetch(API_ENDPOINT, {
                    method: 'POST',
                    headers: { 'Content-Type': 'application/json' },
                    body: JSON.stringify(payload)
                });

                const result = await response.json();

                if (response.ok) {
                    msgCtrl.style.color = "green";
                    msgCtrl.innerText = result.message;
                } else {
                    msgCtrl.style.color = "red";
                    msgCtrl.innerText = "Error en el backend: " + (result.error || "Datos invalidos");
                }
            } catch (error) {
                msgCtrl.style.color = "red";
                msgCtrl.innerText = "Error de red: Conexion fallida.";
                console.error(error);
            }
        });
    </script>
</body>
</html>

        async function fetchData() {
            const statusBox = document.getElementById('statusBox');
            const tableBody = document.getElementById('tableBody');
            statusBox.className = "status-message loading";
            statusBox.style.display = "block";
            statusBox.innerText = "Consultando base de datos NoSQL...";

                statusBox.style.display = "none";

                    row.innerHTML = `
                        <td><strong>${estudiante.nombre || "N/A"}</strong></td>
                        <td>${estudiante.email || "N/A"}</td>
                        <td>${estudiante.telefono || "N/A"}</td>
                        <td><small>${estudiante.fecha_registro || "N/A"}</small></td>
                    `;
                    tableBody.appendChild(row);
                });
            } catch (error) {
                statusBox.innerText = "Error al conectar con MiniStack.";
                console.error(error);
            }
        }
        document.addEventListener('DOMContentLoaded', fetchData);
    </script>
