2. Capa de Datos: Creación de Tabla en DynamoDB

Se inicializa la persistencia. Definimos la clave primaria limpia como email de tipo String (S).

Windows CMD 
aws dynamodb create-table --table-name estudiantes --attribute-definitions AttributeName=email,AttributeType=S --key-schema AttributeName=email,KeyType=HASH --provisioned-throughput ReadCapacityUnits=5,WriteCapacityUnits=5 --endpoint-url http://localhost:4566

3. Capa de Lógica: El Código de la Función (index.js)

💻

Ubicación del Espacio de Trabajo:
Guarda el archivo index.js dentro de tu directorio de usuario:
C:\Users\TU_USUARIO\lab\index.js
Tu consola CMD debe estar abierta apuntando a esta ruta (cd %USERPROFILE%\lab).

index.js (Node.js) 
const { DynamoDBClient } = require("@aws-sdk/client-dynamodb");
const { DynamoDBDocumentClient, PutCommand } = require("@aws-sdk/lib-dynamodb");

const client = new DynamoDBClient({ endpoint: process.env.AWS_ENDPOINT_URL || "http://localhost:4566" });
const docClient = DynamoDBDocumentClient.from(client);

exports.handler = async (event) => {
    const headers = {
        "Access-Control-Allow-Origin": "*",
        "Access-Control-Allow-Headers": "Content-Type",
        "Access-Control-Allow-Methods": "OPTIONS,POST"
    };

    if (event.requestContext.http.method === "OPTIONS") {
        return { statusCode: 200, headers, body: "" };
    }

    try {
        const data = JSON.parse(event.body);
        
        if (!data.txtemail || !data.txtnombre || !data.txtphone) {
            return {
                statusCode: 400,
                headers,
                body: JSON.stringify({ message: "Todos los campos son obligatorios" })
            };
        }

        const params = {
            TableName: "estudiantes",
            Item: {
                email: data.txtemail,
                nombre: data.txtnombre,
                telefono: data.txtphone,
                fecha_registro: new Date().toISOString()
            }
        };

        await docClient.send(new PutCommand(params));

        return {
            statusCode: 200,
            headers,
            body: JSON.stringify({ message: "Estudiante registrado con éxito en DynamoDB (Campos normalizados)" })
        };
    } catch (error) {
        return {
            statusCode: 500,
            headers,
            body: JSON.stringify({ error: error.message })
        };
    }
};

4. Capa de Lógica: Compresión y Carga a MiniStack

Empaqueta el archivo y súbelo al contenedor Docker:

Windows CMD (Comprimir) 
tar -a -c -f funcion-lambda.zip index.js
Windows CMD (Subir Lambda) 
aws lambda create-function --function-name procesarEstudiantes --runtime nodejs18.x --role arn:aws:iam::000000000000:role/lambda-ex --handler index.handler --zip-file fileb://funcion-lambda.zip --endpoint-url http://localhost:4566

5. Capa de Integración: Configurar API Gateway con Trazabilidad de IDs

Mapea las peticiones HTTP externas ejecutando estas líneas en orden cronológico estricto. Presta mucha atención a las salidas JSON del servidor:

A. Instanciar la API REST:

Windows CMD 
aws apigateway create-rest-api --name "API-Estudiantes" --endpoint-url http://localhost:4566
🔍

Salida esperada en Consola (JSON de Creación):
La consola te devolverá una estructura como esta. Extrae el valor de "id" de la API:
{ "items": [{ "id": "05dd485d", "name": "API-Estudiantes", ... }] }
➔ En este ejemplo, el ID extraído es 05dd485d (Este valor es tu TU_API_ID).


B. Consultar la raíz de la API para obtener el Parent ID:

Usa el ID del paso anterior para consultar el recurso base / de la infraestructura:

Windows CMD 
aws apigateway get-resources --rest-api-id 05dd485d --endpoint-url http://localhost:4566
🔍

Salida esperada en Consola (JSON de Recursos):
Al ejecutar la línea anterior, el servidor te devolverá los recursos base. Captura el "id" asociado al path raíz "/":
{ "items": [{ "id": "aac51348", "path": "/", "resourceMethods": {} }] }
➔ En este ejemplo, el ID de la raíz es aac51348 (Este valor es tu TU_PARENT_ID).


C. Crear la ruta secundaria /estudiantes:

Combina ambos identificadores capturados para instanciar el nuevo endpoint de la base de datos:

Windows CMD 
aws apigateway create-resource --rest-api-id 05dd485d --parent-id aac51348 --path-part estudiantes --endpoint-url http://localhost:4566
⚠️

¡ÚLTIMA ALERTA DE CAPTURA!
Este comando "C" te generará un tercer JSON en la consola que contiene los datos del sub-recurso. Copia el "id" de esa respuesta (por ejemplo: 987654xx). Ese valor final representa tu RESOURCE_ID y lo necesitarás en los pasos "D" inferiores.


D. Habilitar método POST y enlazarlo con la Lambda:

Windows CMD (Método) 
aws apigateway put-method --rest-api-id 05dd485d --resource-id RESOURCE_ID --http-method POST --authorization-type "NONE" --endpoint-url http://localhost:4566
Windows CMD (Integración proxy) 
aws apigateway put-integration --rest-api-id 05dd485d --resource-id RESOURCE_ID --http-method POST --type AWS_PROXY --integration-http-method POST --uri arn:aws:apigateway:us-east-1:lambda:path/2015-03-31/functions/arn:aws:lambda:us-east-1:000000000000:function:procesarEstudiantes/submissions --endpoint-url http://localhost:4566

E. Publicar el despliegue al entorno "prod":

Windows CMD 
aws apigateway create-deployment --rest-api-id 05dd485d --stage-name prod --endpoint-url http://localhost:4566

6. Interfaz del Estudiante (Código que pican en el Laboratorio)

estudiante.html 
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Práctica Cloud - Datos Puros</title>
</head>
<body>

    <h2>Formulario de Registro Cloud</h2>
    <p id="responseMessage" style="font-weight: bold; color: blue;"></p>

    <form id="studentForm">
        <div>
            <label>Nombre:</label>
            <input type="text" id="txtnombre" required>
        </div><br>
        <div>
            <label>Teléfono:</label>
            <input type="tel" id="txtphone" required>
        </div><br>
        <div>
            <label>Correo:</label>
            <input type="email" id="txtemail" required>
        </div><br>
        <button type="submit">Enviar a AWS MiniStack</button>
    </form>

    <script>
        document.getElementById('studentForm').addEventListener('submit', async (e) => {
            e.preventDefault();
            
            const msgCtrl = document.getElementById('responseMessage');
            msgCtrl.innerText = "Conectando con el servidor...";

            const payload = {
                txtnombre: document.getElementById('txtnombre').value,
                txtphone: document.getElementById('txtphone').value,
                txtemail: document.getElementById('txtemail').value
            };

            const API_ENDPOINT = "http://awslocal.ddns.net:4566/restapis/05dd485d/prod/_user_request_/estudiantes";

            try {
                const response = await fetch(API_ENDPOINT, {
                    method: 'POST',
                    headers: { 'Content-Type': 'application/json' },
                    body: JSON.stringify(payload)
                });

                const result = await response.json();

                if (response.ok) {
                    msgCtrl.style.color = "green";
                    msgCtrl.innerText = result.message;
                } else {
                    msgCtrl.style.color = "red";
                    msgCtrl.innerText = "Error en el backend: " + (result.error || "Datos inválidos");
                }
            } catch (error) {
                msgCtrl.style.color = "red";
                msgCtrl.innerText = "Error de red: Conexión fallida.";
                console.error(error);
            }
        });
    </script>
</body>
</html>
