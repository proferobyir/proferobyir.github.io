0. Inicialización de la Infraestructura Emulada

⚙️

REQUISITOS PREVIOS OBLIGATORIOS EN WINDOWS:

1. Habilitar WSL (Windows Subsystem for Linux): Docker Desktop requiere de la arquitectura Linux interna de Windows para operar. Si no está instalado, abre una consola de comandos (CMD) como Administrador y ejecuta:
wsl --install
*(Nota: Si la máquina lo requiere, reinicia el equipo tras completar la instalación).*

2. Ejecutar Docker Desktop: Una vez activo WSL, abre la aplicación Docker Desktop. Certifica que el icono inferior de la barra de tareas cambie a color verde (Running). Si el motor de Docker no está activo en segundo plano, la consola fallará al intentar inicializar los contenedores.

Crea un archivo llamado docker-compose.yml en la raíz de tu proyecto para orquestar el contenedor que simulará los servidores de Amazon Web Services (AWS) de forma local:

docker-compose.yml 
version: '3.8'

services:
  ministack:
    image: ministackorg/ministack:full
    container_name: ministack_practica
    ports:
      - "4566:4566"           # Puerto principal de los servicios de AWS
      - "4510-4559:4510-4559" # Puertos que usa MiniStack internamente para las Lambdas de Docker
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock # Crucial para que funcionen las Lambdas
    environment:
      - AWS_DEFAULT_REGION=us-east-1

Comando para encender los servidores desde la consola:

Windows CMD 
docker-compose up -d

1. Instalación del Cliente AWS y Configuración de Entorno

Para poder enviar comandos a la infraestructura emulada, la máquina de desarrollo requiere de la interfaz de línea de comandos oficial de Amazon.

📥 Descarga del AWS CLI:
Para sistemas Windows de 64 bits, descargar e instalar el instalador oficial MSI desde el siguiente enlace:
👉 Instalador Directo AWS CLI v2 (.msi)

A. Verificar la instalación del cliente:

Abre una terminal de comandos nueva (CMD) y ejecuta la siguiente línea para certificar que el sistema reconoce la herramienta:

Windows CMD 
aws --version
📂

Ruta Obligatoria de Credenciales:
Una vez verificado el cliente, los siguientes dos archivos de configuración deben guardarse estrictamente en el directorio del perfil de usuario de Windows:
C:\Users\TU_USUARIO\.aws\

Archivo A: credentials

credentials 
[default]
aws_access_key_id = test
aws_secret_access_key = test

Archivo B: config

config 
[default]
region = us-east-1
output = json

2. Capa de Datos: Creación de Tabla en DynamoDB

Se inicializa la persistencia de datos. Definimos la clave primaria limpia como email de tipo String (S).

Windows CMD 
aws dynamodb create-table --table-name estudiantes --attribute-definitions AttributeName=email,AttributeType=S --key-schema AttributeName=email,KeyType=HASH --provisioned-throughput ReadCapacityUnits=5,WriteCapacityUnits=5 --endpoint-url http://localhost:4566

3. Capa de Lógica: El Código Adaptado (index.js)

💻

Ubicación del Espacio de Trabajo:
Crea una carpeta llamada lab dentro de tu carpeta de usuario y guarda el archivo allí:
C:\Users\TU_USUARIO\lab\index.js
Tu consola CMD debe operar apuntando a este directorio (cd %USERPROFILE%\lab).

Este código está normalizado para soportar el mapeo de eventos clásico de API Gateway REST APIs (capturando event.httpMethod) evitando colapsos de propiedades indefinidas en tiempo de ejecución:

index.js (Node.js) 
const { DynamoDBClient } = require("@aws-sdk/client-dynamodb");
const { DynamoDBDocumentClient, PutCommand } = require("@aws-sdk/lib-dynamodb");

const client = new DynamoDBClient({ endpoint: process.env.AWS_ENDPOINT_URL || "http://localhost:4566" });
const docClient = DynamoDBDocumentClient.from(client);

exports.handler = async (event) => {
    console.log("Evento recibido:", JSON.stringify(event));

    // Cabeceras CORS mandatorias para navegadores web
    const headers = {
        "Access-Control-Allow-Origin": "*",
        "Access-Control-Allow-Headers": "Content-Type",
        "Access-Control-Allow-Methods": "OPTIONS,POST,GET"
    };

    const method = event.httpMethod || (event.requestContext && event.requestContext.http && event.requestContext.http.method);

    if (method === "OPTIONS") {
        return { statusCode: 200, headers, body: "" };
    }

    try {
        const data = JSON.parse(event.body);
        
        if (!data.txtemail || !data.txtnombre || !data.txtphone) {
            return {
                statusCode: 400,
                headers,
                body: JSON.stringify({ message: "Todos los campos son obligatorios" })
            };
        }

        const params = {
            TableName: "estudiantes",
            Item: {
                email: data.txtemail,
                nombre: data.txtnombre,
                telefono: data.txtphone,
                fecha_registro: new Date().toISOString()
            }
        };

        await docClient.send(new PutCommand(params));

        return {
            statusCode: 200,
            headers,
            body: JSON.stringify({ message: "Estudiante registrado con éxito en DynamoDB (Campos normalizados)" })
        };
    } catch (error) {
        return {
            statusCode: 500,
            headers,
            body: JSON.stringify({ error: error.message })
        };
    }
};

4. Capa de Lógica: Gestión de Dependencias e Inyección Completa

Dado que el entorno de ejecución requiere el SDK utilitario de DynamoDB, es obligatorio inicializar el paquete localmente e incluir la carpeta node_modules dentro de la compresión binaria:

A. Descargar dependencias necesarias (Ejecutar parados dentro de \lab):

Windows CMD 
npm init -y
npm install @aws-sdk/lib-dynamodb

B. Comprimir de forma compuesta (Código + Módulos):

Windows CMD 
tar -a -c -f funcion-lambda.zip index.js node_modules package.json

C. Crear el recurso funcional en MiniStack:

Windows CMD 
aws lambda create-function --function-name procesarEstudiantes --runtime nodejs18.x --role arn:aws:iam::000000000000:role/lambda-ex --handler index.handler --zip-file fileb://funcion-lambda.zip --endpoint-url http://localhost:4566

5. Capa de Integración: Configurar API Gateway con Trazabilidad de IDs

Mapea las peticiones HTTP externas ejecutando estas líneas en orden secuencial estricto, prestando atención a los flujos de salida del servidor:

A. Instanciar la API REST en el contenedor:

Windows CMD 
aws apigateway create-rest-api --name "API-Estudiantes" --endpoint-url http://localhost:4566
🔍

Salida JSON Esperada en Consola (Paso A):
La terminal te arrojará la metadata de creación. Identifica de inmediato el campo de ID único de la API:
{ "items": [ { "id": "05dd485d", "name": "API-Estudiantes", "endpointConfiguration": { "types": ["REGIONAL"] } } ] }
➔ El valor capturado representa tu TU_API_ID (Ejemplo real: 05dd485d).


B. Consultar el identificador del recurso raíz (/):

Reemplaza tu ID capturado para extraer la ruta raíz jerárquica del servidor:

Windows CMD 
aws apigateway get-resources --rest-api-id 05dd485d --endpoint-url http://localhost:4566
🔍

Salida JSON Esperada en Consola (Paso B):
El servidor te responderá el mapeo base. Captura el ID enlazado al path principal "/":
{ "items": [ { "id": "aac51348", "path": "/", "resourceMethods": {} } ] }
➔ El valor capturado representa tu TU_PARENT_ID (Ejemplo real: aac51348).


C. Crear la ruta secundaria (/estudiantes):

Utiliza ambos parámetros extraídos para instanciar el nuevo recurso de red:

Windows CMD 
aws apigateway create-resource --rest-api-id 05dd485d --parent-id aac51348 --path-part estudiantes --endpoint-url http://localhost:4566
🔍

Salida JSON Esperada en Consola (Paso C):
Este comando registrará la ruta del endpoint. Captura el ID asignado a la subruta:
{ "id": "d4f56816", "parentId": "aac51348", "pathPart": "estudiantes", "path": "/estudiantes" }
➔ El valor capturado representa tu RESOURCE_ID (Ejemplo real: d4f56816).


D. Abrir método POST e inyectar el enlace Proxy con la Lambda:

Windows CMD (Habilitar Método) 
aws apigateway put-method --rest-api-id 05dd485d --resource-id d4f56816 --http-method POST --authorization-type "NONE" --endpoint-url http://localhost:4566
Windows CMD (Soldar Integración) 
aws apigateway put-integration --rest-api-id 05dd485d --resource-id d4f56816 --http-method POST --type AWS_PROXY --integration-http-method POST --uri arn:aws:apigateway:us-east-1:lambda:path/2015-03-31/functions/arn:aws:lambda:us-east-1:000000000000:function:procesarEstudiantes/submissions --endpoint-url http://localhost:4566

5B. Capa de Seguridad de Red: Soporte de Tráfico Cruzado (CORS)

Los navegadores web ejecutan una validación previa de seguridad (petición OPTIONS) antes de enviar el JSON. Si no abres este canal, la interfaz web arrojará "Error de red: Conexión fallida".

A. Habilitar el método OPTIONS en la subruta:

Windows CMD 
aws apigateway put-method --rest-api-id 05dd485d --resource-id d4f56816 --http-method OPTIONS --authorization-type "NONE" --endpoint-url http://localhost:4566

B. Enlazar OPTIONS con el flujo Proxy de la Lambda:

Windows CMD 
aws apigateway put-integration --rest-api-id 05dd485d --resource-id d4f56816 --http-method OPTIONS --type AWS_PROXY --integration-http-method POST --uri arn:aws:apigateway:us-east-1:lambda:path/2015-03-31/functions/arn:aws:lambda:us-east-1:000000000000:function:procesarEstudiantes/submissions --endpoint-url http://localhost:4566

C. Publicar/Congelar el despliegue en producción (OBLIGATORIO):

Windows CMD 
aws apigateway create-deployment --rest-api-id 05dd485d --stage-name prod --endpoint-url http://localhost:4566

6. Interfaz de Envío del Estudiante (estudiante.html)

Este es el formulario estructurado que los estudiantes utilizarán para enviar los datos hacia la nube emulada. Incluye un enlace explícito para navegar al visor de registros.

estudiante.html 
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Práctica Cloud - Registro de Estudiantes</title>
</head>
<body>

    <h2>Formulario de Registro Cloud</h2>
    <p id="responseMessage" style="font-weight: bold; color: blue;"></p>

    <form id="studentForm">
        <div>
            <label>Nombre:</label>
            <input type="text" id="txtnombre" required>
        </div><br>
        <div>
            <label>Teléfono:</label>
            <input type="tel" id="txtphone" required>
        </div><br>
        <div>
            <label>Correo:</label>
            <input type="email" id="txtemail" required>
        </div><br>
        <button type="submit">Enviar a AWS MiniStack</button>
    </form>

    <br><hr><br>
    <!-- ENLACE DE REDIRECCIÓN MANDATORIO -->
    <a href="registros.html" style="font-weight: bold; color: green; font-size: 1.1rem; text-decoration: none;">📊 Ver datos guardados (Ir al Panel)</a>

    <script>
        document.getElementById('studentForm').addEventListener('submit', async (e) => {
            e.preventDefault();
            
            const msgCtrl = document.getElementById('responseMessage');
            msgCtrl.innerText = "Conectando con el servidor...";

            const payload = {
                txtnombre: document.getElementById('txtnombre').value,
                txtphone: document.getElementById('txtphone').value,
                txtemail: document.getElementById('txtemail').value
            };

            const API_ENDPOINT = "http://awslocal.ddns.net:4566/restapis/05dd485d/prod/_user_request_/estudiantes";

            try {
                const response = await fetch(API_ENDPOINT, {
                    method: 'POST',
                    headers: { 'Content-Type': 'application/json' },
                    body: JSON.stringify(payload)
                });

                const result = await response.json();

                if (response.ok) {
                    msgCtrl.style.color = "green";
                    msgCtrl.innerText = result.message;
                } else {
                    msgCtrl.style.color = "red";
                    msgCtrl.innerText = "Error en el backend: " + (result.error || "Datos inválidos");
                }
            } catch (error) {
                msgCtrl.style.color = "red";
                msgCtrl.innerText = "Error de red: Conexión fallida.";
                console.error(error);
            }
        });
    </script>
</body>
</html>

        const API_ENDPOINT = "http://awslocal.ddns.net:4566/restapis/05dd485d/prod/_user_request_/estudiantes";

        async function fetchData() {
            const statusBox = document.getElementById('statusBox');
            const tableBody = document.getElementById('tableBody');
            statusBox.className = "status-message loading";
            statusBox.style.display = "block";
            tableBody.innerHTML = "";

            try {
                const response = await fetch(API_ENDPOINT, { method: 'GET' });
                if (!response.ok) throw new Error("Error de respuesta");
                
                const result = await response.json();
                const items = result.Items || result || [];

                if (items.length === 0) {
                    statusBox.innerText = "La tabla está vacía.";
                    return;
                }

                statusBox.style.display = "none";
                items.forEach(estudiante => {
                    const row = document.createElement('tr');
                    row.innerHTML = `
                        <td><strong>${estudiante.nombre || "N/A"}</strong></td>
                        <td>${estudiante.email || "N/A"}</td>
                        <td>${estudiante.telefono || "N/A"}</td>
                        <td><small>${estudiante.fecha_registro || "N/A"}</small></td>
                    `;
                    tableBody.appendChild(row);
                });
            } catch (error) {
                statusBox.innerText = "Error al conectar con MiniStack.";
                console.error(error);
            }
        }
        document.addEventListener('DOMContentLoaded', fetchData);
    </script>
</body>
