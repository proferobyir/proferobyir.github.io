Guía de Despliegue Secuencial: AWS MiniStack (Windows)

Estructura lógica paso a paso para la preparación de laboratorios

Pasos Secuenciales

  • 1. Configuración Local
  • 2. Capa de Datos (DynamoDB)
  • 3. Capa de Lógica (Lambda)
  • 4. Integración (API Gateway)
  • 5. Endpoint del Laboratorio

1. Configuración de Entorno en el Servidor

Antes de lanzar servicios, la AWS CLI necesita sus archivos de identidad de simulacro creados en la ruta C:\Users\TU_USUARIO\.aws\.

Archivo A: credentials (Crear sin extensión)

credentials 
[default]
aws_access_key_id = test
aws_secret_access_key = test

Archivo B: config (Crear sin extensión)

config 
[default]
region = us-east-1
output = json

2. Capa de Datos: Creación de Tabla en DynamoDB

Se inicializa el almacenamiento persistente. Definimos la clave primaria limpia como email de tipo String (S).

Windows CMD 
aws dynamodb create-table --table-name estudiantes --attribute-definitions AttributeName=email,AttributeType=S --key-schema AttributeName=email,KeyType=HASH --provisioned-throughput ReadCapacityUnits=5,WriteCapacityUnits=5 --endpoint-url http://localhost:4566

3. Capa de Lógica: Compresión y Carga de la Lambda

Empaqueta el archivo index.js que traduce los nombres mnemotécnicos (txt...) a campos limpios y despliega el backend:

A. Empaquetar usando tar nativo de Windows:

Windows CMD 
tar -a -c -f funcion-lambda.zip index.js

B. Crear la función en MiniStack:

Windows CMD 
aws lambda create-function --function-name procesarEstudiantes --runtime nodejs18.x --role arn:aws:iam::000000000000:role/lambda-ex --handler index.handler --zip-file fileb://funcion-lambda.zip --endpoint-url http://localhost:4566

4. Capa de Integración: Configurar API Gateway

Sigue la cadena de comandos en estricto orden para mapear las peticiones HTTP externas hacia la Lambda interna:

A. Instanciar la API REST:

Windows CMD 
aws apigateway create-rest-api --name "API-Estudiantes" --endpoint-url http://localhost:4566

B. Consultar el ID del recurso raíz (/):

Windows CMD 
aws apigateway get-resources --rest-api-id TU_API_ID --endpoint-url http://localhost:4566

C. Crear la ruta secundaria /estudiantes:

Windows CMD 
aws apigateway create-resource --rest-api-id TU_API_ID --parent-id TU_PARENT_ID --path-part estudiantes --endpoint-url http://localhost:4566

D. Habilitar el método POST y enlazarlo con la Lambda:

Windows CMD (Método) 
aws apigateway put-method --rest-api-id TU_API_ID --resource-id RESOURCE_ID --http-method POST --authorization-type "NONE" --endpoint-url http://localhost:4566
Windows CMD (Integración proxy) 
aws apigateway put-integration --rest-api-id TU_API_ID --resource-id RESOURCE_ID --http-method POST --type AWS_PROXY --integration-http-method POST --uri arn:aws:apigateway:us-east-1:lambda:path/2015-03-31/functions/arn:aws:lambda:us-east-1:000000000000:function:procesarEstudiantes/submissions --endpoint-url http://localhost:4566

E. Publicar el despliegue al entorno "prod":

Windows CMD 
aws apigateway create-deployment --rest-api-id TU_API_ID --stage-name prod --endpoint-url http://localhost:4566

5. URL de Conexión para los Estudiantes

Con la arquitectura integrada, esta es la ruta pública final que tus alumnos llamarán mediante el JavaScript de sus archivos locales:

🔗

Ruta del Laboratorio:
http://awslocal.ddns.net:4566/restapis/TU_API_ID/prod/_user_request_/estudiantes
