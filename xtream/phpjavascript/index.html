⚡ Guía Rápida: De HTML Tradicional a Peticiones Asíncronas

Conceptos esenciales de JavaScript y PHP explicados visualmente punto por punto.

1. Formato de Datos: Objetos JS & JSON

JS

Objeto en JavaScript

Es una estructura de tipo llave: valor que vive en la memoria de la aplicación mientras el navegador está ejecutando el código.

💡 Uso: Manipular datos dinámicamente dentro de la página. 
const usuario = {
    nombre: "Carlos",
    correo: "carlos@email.com",
    telefono: "04121234567"
};
Formato

JSON (JavaScript Object Notation)

Es simplemente texto plano con formato estandarizado. Se usa para enviar y recibir información entre lenguajes (ej: de PHP a JS).

💡 Regla: Todas las llaves deben llevar comillas dobles " ". 
{
    "nombre": "Carlos",
    "correo": "carlos@email.com",
    "telefono": "04121234567"
}

2. Fundamentos de JavaScript para la Red

JS

Escuchadores & Callbacks

  • addEventListener: Queda "escuchando" cuando un evento ocurre (ej: submit, click).
  • Callback: Es una función que se pasa como argumento a otra función para ejecutarse únicamente cuando la acción principal termina.
// La función anónima es el 'Callback'
formulario.addEventListener('submit', (event) => {
    event.preventDefault(); // Frena el recargo
    console.log("¡Formulario interceptado!");
});
JS

Asincronía: Async & Await

Las peticiones web toman tiempo en responder. Usamos async/await para pausar la ejecución dentro de la función sin congelar el resto de la interfaz del usuario.

async function guardarDatos() {
    // 'await' espera a que la red responda
    const respuesta = await fetch('procesar.php');
    const datos = await respuesta.json();
}

3. Fundamentos de PHP para Backend

PHP

Arreglos Asociativos & JSON

En PHP los arreglos asociativos usan claves personalizadas en lugar de índices numéricos. Son el equivalente perfecto a un objeto JSON.

// 1. Arreglo Asociativo
$respuesta = [
    "status" => "success",
    "message" => "Registro guardado"
];

// 2. Convertir a texto JSON
echo json_encode($respuesta);
PHP

Base de Datos Procedimental (MySQLi)

Uso directo de funciones mysqli para validar e insertar datos recibidos en el servidor.

// Crear conexión por procedimientos
$conexion = mysqli_connect("localhost", "root", "", "mi_bd");

// Ejecutar inserción básica
$sql = "INSERT INTO usuarios VALUES (NULL, '$nom', '$cor', '$tel')";
$resultado = mysqli_query($conexion, $sql);

4. Comunicación entre Cliente y Servidor

Red

Códigos de Estado HTTP

El servidor comunica qué ocurrió mediante un código estándar:

Código Significado
200 / 201 OK / Creado exitosamente
400 Petición mala (Faltan campos)
409 Conflicto (Registro duplicado)
500 Error interno del Servidor/Base de Datos
Seguridad

¿Qué es CORS y cómo configurarlo?

CORS (Cross-Origin Resource Sharing) es una regla de seguridad de los navegadores que bloquea peticiones si el Front-End y el Back-End están en dominios o puertos distintos.

💡 Solución en PHP: Agregar cabeceras al inicio del archivo backend. 
// Permitir accesos desde cualquier origen
header("Access-Control-Allow-Origin: *");
header("Access-Control-Allow-Methods: POST, GET");
header("Content-Type: application/json; charset=UTF-8");

5. Laboratorio Práctico Integrado

Este formulario utiliza fetch(), intercepta el submit, maneja estados de carga y procesa la respuesta sin refrescar la ventana.

PHP

Código de referencia completo para: procesar.php

Este es el archivo que debe residir en la misma carpeta que procesa los datos y devuelve el JSON.

<?php
// 1. Cabeceras CORS y Formato de Respuesta
header("Access-Control-Allow-Origin: *");
header("Content-Type: application/json; charset=UTF-8");

// 2. Recepción de variables por POST
$nombre   = $_POST['nombre'] ?? '';
$correo   = $_POST['correo'] ?? '';
$telefono = $_POST['telefono'] ?? '';

// 3. Validación básica de backend
if (empty($nombre) || empty($correo) || empty($telefono)) {
    http_response_code(400);
    echo json_encode([
        "status"  => "error",
        "message" => "Todos los campos son obligatorios."
    ]);
    exit;
}

// 4. Conexión a Base de Datos (Procedimental)
$conexion = @mysqli_connect("localhost", "root", "", "mi_base_datos");

if (!$conexion) {
    http_response_code(500);
    echo json_encode([
        "status"  => "error",
        "message" => "Error de conexión con la Base de Datos."
    ]);
    exit;
}

// 5. Intento de Inserción
$sql = "INSERT INTO usuarios (nombre, correo, telefono) VALUES ('$nombre', '$correo', '$telefono')";
$query = mysqli_query($conexion, $sql);

if ($query) {
    http_response_code(200);
    echo json_encode([
        "status"  => "success",
        "message" => "¡Usuario guardado con éxito!"
    ]);
} else {
    // Capturar error de duplicado (Código SQL 1062)
    $errorCode = mysqli_errno($conexion);
    http_response_code($errorCode == 1062 ? 409 : 500);
    
    echo json_encode([
        "status"  => "error",
        "code"    => $errorCode,
        "message" => $errorCode == 1062 ? "El correo ya está registrado." : "Error al insertar el registro."
    ]);
}

mysqli_close($conexion);
?>
