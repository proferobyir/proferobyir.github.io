Firebase Emulator Suite

Desarrolla, prueba y depura sin salir de tu computadora · Sin costos · Sin internet · Sin riesgos

100% entorno local · Firestore · Auth · Functions · Storage · Hosting
1
Prepara tu estación de trabajo

Necesitarás Node.js (v18 o superior) y npm instalados globalmente. Firebase CLI se ejecuta sobre Node.

Verifica tu entorno: Abre tu terminal y ejecuta: 
node --version && npm --version
Si no los tienes, descárgalos desde nodejs.org.

También necesitarás un proyecto de Firebase (puede ser nuevo). Ve a Firebase Console y crea un proyecto (modo "sandbox" o gratuito). No es obligatorio tener recursos en la nube para usar emuladores, solo sirve para las configuraciones.

2
Instalar Firebase CLI globalmente

Ejecuta este comando en tu terminal favorita (PowerShell, bash, zsh):

npm install -g firebase-tools

Para confirmar que se instaló correctamente:

firebase --version
Si tienes problemas de permisos en Linux/Mac, usa sudo npm install -g firebase-tools o configura npm global sin sudo.
3
Inicializar Firebase en tu carpeta local

Crea una carpeta para tu proyecto y entra en ella:

mkdir mi-app-firebase-local
cd mi-app-firebase-local

Ejecuta el comando de inicialización:

firebase init

Sigue los pasos interactivos:

  • Selecciona Firestore, Authentication, Functions, Hosting, Storage (puedes elegir solo los que necesites).
  • Elige el proyecto que creaste en Firebase Console (o crea uno nuevo desde CLI).
  • Acepta las rutas por defecto (firestore.rules, firestore.indexes.json, etc).
  • Para Functions: elige TypeScript o JavaScript (como prefieras).
  • Para Hosting: escribe "public" como directorio y sí a SPA.
Tip: No necesitas tener las APIs habilitadas en la nube, el emulador las simula completamente.
4
Configurar & habilitar emuladores

Ahora ejecuta el siguiente comando para iniciar la configuración de emuladores:

firebase init emulators

Marca los servicios que quieres emular localmente:

Firestore Authentication Storage Functions Hosting Pub/Sub

Asigna puertos (los default son perfectos: Firestore 8080, Auth 9099, Storage 9199, Functions 5001, Hosting 5000). Elige "descargar emuladores" si es primera vez.

Esto generará el archivo firebase.json con la sección "emulators". Ahí puedes ajustar puertos manualmente si lo deseas.
5
🔥 Iniciar los emuladores locales

¡Llegó el momento! Ejecuta el comando mágico:

firebase emulators:start

Verás en la terminal una salida similar a:

┌─────────────────────────────────────────────────────────────┐
│ ✔  All emulators ready! It is now safe to connect your app. │
│ i  View Emulator UI at http://localhost:4000                │
└─────────────────────────────────────────────────────────────┘

Abre tu navegador y visita http://localhost:4000 — ahí tendrás una interfaz gráfica para ver datos, logs y estado de cada emulador.

La UI te permite modificar documentos de Firestore, crear usuarios de prueba, revisar funciones, ¡todo en tiempo real!
6
Apuntar tu aplicación a los emuladores

En el código de tu app (web, React, Node, etc.) debes redirigir los SDKs al localhost. Ejemplo para una app web con Firebase v9+:

JavaScript - Conexión a emuladores 
import { initializeApp } from "firebase/app";
import { getFirestore, connectFirestoreEmulator } from "firebase/firestore";
import { getAuth, connectAuthEmulator } from "firebase/auth";
import { getStorage, connectStorageEmulator } from "firebase/storage";
import { getFunctions, connectFunctionsEmulator } from "firebase/functions";

// Tu configuración de Firebase (puedes usar variables locales, no importan los valores reales)
const firebaseConfig = {
  apiKey: "demo-key-local",
  authDomain: "localhost",
  projectId: "tu-id-proyecto",
  storageBucket: "tu-bucket",
  messagingSenderId: "000000",
  appId: "1:000000:web:demo",
};

const app = initializeApp(firebaseConfig);
const db = getFirestore(app);
const auth = getAuth(app);
const storage = getStorage(app);
const functions = getFunctions(app);

// 🔌 Conectar a emuladores locales (solo desarrollo)
connectFirestoreEmulator(db, 'localhost', 8080);
connectAuthEmulator(auth, 'http://localhost:9099');
connectStorageEmulator(storage, 'localhost', 9199);
connectFunctionsEmulator(functions, 'localhost', 5001);

console.log("✅ App apuntando a Firebase Emulator Suite");
Asegúrate de que los puertos coincidan con los definidos en el paso 4. Para hosting emulado, tu app servida desde http://localhost:5000 también tendrá emulación completa.
7
Poblar datos y depurar localmente

Ahora puedes usar la UI (localhost:4000) para crear colecciones y documentos, o mediante código. La magia es que cualquier cambio es local y efímero.

Comandos útiles para exportar/importar datos: puedes persistir el estado entre sesiones.

# Exportar datos emulados a una carpeta
firebase emulators:export ./emulator-data

# Iniciar emuladores con datos guardados previamente
firebase emulators:start --import=./emulator-data
Debugging profesional: Los logs de funciones en tiempo real aparecen en la terminal y en la UI. Además puedes usar breakpoints si trabajas con Functions locales.
📘
Buenas prácticas + workflow profesional

Agrega scripts en tu package.json para facilitar tu flujo:

{
  "scripts": {
    "emulators": "firebase emulators:start --import=./seed-data --export-on-exit",
    "emulators:clean": "firebase emulators:start",
    "seed": "node scripts/seedFirestore.js"
  }
}

✅ Ventajas clave que obtienes:

  • Sin límite de lectura/escritura ni costos sorpresa.
  • Pruebas automatizadas con jest + emuladores (entorno aislado).
  • Colaboración: Cada miembro del equipo corre su propia instancia.
  • Las reglas de seguridad (Firestore Rules) se prueban inmediatamente.
Recuerda: cuando despliegues a producción (firebase deploy), los emuladores no intervienen. Todo tu código apuntará a la nube si no usas condicionales. Maneja process.env.NODE_ENV para cambiar entre emuladores y producción automáticamente.

Preguntas frecuentes sobre emuladores

¿Necesito conexión a internet para usarlos?

No. Una vez instalados, los emuladores funcionan offline al 100%. Solo ocupas internet la primera vez para descargar los binarios de los emuladores.

¿Puedo usar Firebase Admin SDK con emuladores?

Sí, apunta el Admin SDK a localhost y los puertos respectivos. Perfecto para cloud functions locales o scripts de backend.

¿Los datos se borran al cerrar los emuladores?

Por defecto sí, pero usando --import/--export-on-exit puedes guardar snapshots y reutilizarlos.
