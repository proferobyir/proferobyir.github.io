🎣 CLASE 1 · Primer Workflow Proyecto: Lead magnet automático

Tu primera automatización: Captador de leads con cálculo instantáneo

EJEMPLO REAL:

Imagina que tienes un negocio de entrenamiento online. Un prospecto llena un formulario con su nombre y nivel de condición física (1 al 10). n8n captura esos datos, calcula automáticamente un "plan de urgencia" (puntaje * 5 minutos de ejercicio recomendado) y te envía un mensaje a Telegram con el resumen. ¡Todo en menos de 2 segundos sin que tú hagas nada!

🔹 Trigger (Disparador):

¿Qué es? El evento que inicia TODO el flujo. Sin trigger, nada se mueve.

🔸 Ejemplo aquí: Usamos el nodo Webhook o Form Trigger. Cuando alguien envía el formulario → se dispara la automatización.

📌 Analogía: Como el sensor de movimiento que enciende una luz. El movimiento (formulario) activa todo.
🔹 Nodo de Datos (Edit Fields):

¿Qué es? Transforma, calcula o modifica los datos que vienen del trigger.

🔸 Ejemplo aquí: Añadimos un campo nuevo llamado "recomendacion" cuyo valor es $json.nivel_fisico * 5. Así creamos información nueva que antes no existía.

📌 Analogía: Como una máquina que toma patatas (datos brutos) y las convierte en patatas fritas (dato procesado).
🔹 Acción (Action):

¿Qué es? Lo que hace n8n con los datos ya procesados: enviar un mensaje, guardar en base de datos, llamar a una API.

🔸 Ejemplo aquí: Nodo de Telegram o Email. Enviamos un mensaje al dueño del negocio con el nombre y la recomendación calculada.

📌 Analogía: Es como el repartidor que lleva el pedido ya cocinado a la casa del cliente.
Resultado final del workflow: El lead se siente atendido al instante, tú recibes una alerta con un cálculo personalizado y puedes hacer seguimiento en minutos.

🤖 CLASE 2 · IA + APIs Proyecto: Clasificador inteligente de correos

Conecta n8n con ChatGPT y toma decisiones automáticas

EJEMPLO REAL:

Tu negocio recibe 100 correos al día de soporte. Conectas n8n a Gmail. Cada nuevo correo se lo envías a OpenAI pidiendo: "Clasifica este mensaje como POSITIVO, NEUTRO o NEGATIVO". Si la IA dice NEGATIVO, n8n automáticamente: ① genera un borrador de disculpa en Gmail y ② te envía una alerta a Slack para que priorices ese caso. Si es POSITIVO, solo lo guarda en una hoja de cálculo. ¡Automatización con criterio humano!

Nodo OpenAI

¿Qué hace? Envía un texto a ChatGPT y recibe una respuesta.

Ejemplo prompt: "El cliente dice: '$json.cuerpo_correo'. ¿Es positivo, negativo o neutro? Responde solo una palabra."

Nodo IF / Switch

¿Qué hace? Actúa como un semáforo: si se cumple X, va por un camino; si no, por otro.

Ejemplo: Si la IA responde "NEGATIVO" → rama roja (alerta urgente). Si no → rama verde (solo registro).

Concepto extra: API

¿Qué es una API? Un "mensajero" que permite que dos programas (n8n y OpenAI) se entiendan. n8n usa la API de OpenAI para enviarle el correo y recibir la clasificación.

📌 Analogía: Como cuando pides comida por teléfono (tú = n8n, restaurante = API, la comida = clasificación).
Dato clave: El nodo condicional (IF) es lo que permite que n8n "piense". Sin él, todas las respuestas recibirían el mismo tratamiento.
Flujo con IA
1. Gmail Trigger→ Nuevo correo
2. OpenAI (IA)→ Clasifica
3. Nodo IF→ ¿Negativo?
Sí → Slack alerta + Borrador disculpa
No → Guardar en Google Sheets
🎯 Valor agregado: Automatizas el triaje de atención al cliente 24/7.
📊 CLASE 3 · REPORTES AUTOMÁTICOS Proyecto: Sincronizador nocturno

Base de datos + Schedule Trigger: tus datos trabajan mientras duermes

EJEMPLO REAL:

Imagina que tienes una tienda online. Durante el día, cada venta se guarda automáticamente en una hoja de Google Sheets (producto, precio, cliente). Luego, todos los días a las 11:59 PM, n8n lee esa hoja, calcula el total de ventas del día, los productos más vendidos y te envía un correo con el reporte listo para que lo revises en el desayuno. Sin horarios extra, sin errores manuales.

Schedule Trigger (Disparador por tiempo)

¿Qué es? Un trigger que no espera un evento externo, sino que se activa a una hora específica o cada X minutos/horas/días.

Ejemplo: Configuramos "Every day at 23:59" para que el reporte nocturno se genere justo al terminar el día.

📌 Analogía: Como un despertador que suena siempre a la misma hora, sin que nadie lo toque.
Nodo Google Sheets / Notion (Base de datos simple)

¿Qué es? Almacenar datos de forma estructurada para consultarlos después. No necesitas saber SQL.

Ejemplo: Cada venta nueva se agrega como una fila en Sheets. Luego el schedule trigger lee todas las filas para calcular totales.

Flujo completo: Trigger diario → Leer Sheets → Agrupar por producto (nodo Aggregate) → Enviar email con HTML bonito.
Workflow programado
Schedule Trigger (23:59 hs)
Obtener datos de Google Sheets (ventas del día)
Calcular totales (nodo Aggregator)
Enviar reporte por Email
💡 Beneficio estrella: Tomas decisiones basadas en datos sin dedicarle 1 hora diaria a hacer informes manuales.

Glosario visual de conceptos n8n (con ejemplos claros)

Trigger:El "por qué" empieza. Ej: llega un email, un pago, un webhook.
Nodo:Cada paso o bloque en tu workflow. Ej: nodo HTTP, nodo OpenAI, nodo Set.
Workflow:El conjunto completo de nodos conectados. Ej: la receta completa de cocina.
Webhook:Una URL que recibe datos externos. Ej: cuando Typeform envía un resultado a n8n.
API:Puente entre n8n y otros servicios. Ej: n8n ↔ Google Sheets.
Nodo IF:Toma decisiones. Ej: Si el correo tiene "urgencia" → alerta roja.
🧠 Recuerda: En n8n, todo es un nodo. Aprende a conectar nodos → aprendes a automatizar el mundo.
